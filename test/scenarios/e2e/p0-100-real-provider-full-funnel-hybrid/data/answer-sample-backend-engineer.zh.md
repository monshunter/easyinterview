# 作答样例：资深后端工程师

本样例可在人工 UAT 的 Practice 步骤中使用。内容为合成材料，不含真实 PII。

我主导过一次把单体任务处理服务迁移到事件驱动后端平台的项目。最大的风险是迁移过程中要保持幂等性，同时把报告生成、通知派发和前端轮询都迁移到异步 worker。

我的做法是先定义统一的 job 状态机，再按领域逐个迁移，并用兼容性测试守住既有流程。所有有副作用的接口都增加幂等键，用户可见的状态变化保存在 PostgreSQL，只把稳定的 jobId 暴露给前端。迁移过程中我补了重试次数、死信数量和延迟直方图，让产品和支持团队能判断一次延迟是可处理排队，还是需要人工介入。

结果是高峰期超时工单减少了约 60%，同时产品流程更容易解释：每个长耗时动作都有可观测 job 状态和确定性的重试边界。
