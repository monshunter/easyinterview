公司：云启数据 CloudPivot Data

团队：平台与基础设施

## 关于我们

云启数据为中大型企业提供实时数据集成与分析平台，日均处理数百亿条事件。我们的后端以 Go 微服务为主，运行在 Kubernetes 上，依赖 PostgreSQL、Kafka 与对象存储构建高吞吐、可观测、可回放的数据管线。我们正在扩张平台团队，寻找能独立负责核心服务设计与稳定性的资深工程师。

## 岗位职责

- 负责数据接入与编排核心服务的设计、开发与演进，保障高并发下的正确性与稳定性。
- 设计幂等、可重试、可观测的异步任务与事件流水线（Kafka / outbox / 重试与补偿）。
- owns 关键链路的容量、延迟与错误预算；建立 SLO 与告警，主导线上问题排查与复盘。
- 与产品、数据和前端团队对齐 API 契约（OpenAPI），推动跨团队接口稳定演进。
- 把控数据隐私与合规红线，确保敏感字段不进入日志与可观测面。
- 指导初中级工程师，参与代码评审与技术方案评审，沉淀工程规范。

## 任职要求（硬性）

- 5 年以上后端开发经验，3 年以上 Go 生产级服务经验。
- 扎实的分布式系统基础：一致性、幂等、并发控制、消息可靠投递。
- 熟练 PostgreSQL：索引、事务隔离、查询优化与 schema 演进。
- 熟悉消息队列（Kafka / NATS 之一）与异步任务编排模式。
- 具备可观测性实践经验：结构化日志、指标、链路追踪。
- 良好的 API 契约意识，能基于 OpenAPI 做前后端协作。

## 加分项

- 有数据平台 / 实时计算 / CDC 相关经验。
- 熟悉 Kubernetes 与云原生部署、灰度发布。
- 有从 0 到 1 设计核心服务、主导稳定性治理的经历。
- 关注成本与性能权衡，有大规模降本增效实践。

## 技术栈

Go、PostgreSQL、Kafka、Redis、Kubernetes、gRPC、OpenAPI、OpenTelemetry、对象存储（S3 兼容）。
