# Bug 模板

## 1 Bug 记录模板

```markdown
# BUG-NNNN: [一句话标题]

> 模块: xxx | 严重度: medium | 状态: investigating
> 发现日期: YYYY-MM-DD | 关联 Commit: `-` 或 `type(scope): subject`
> 关联日志: [YYYY-MM-DD](../work-journal/YYYY-MM-DD.md) 或 `-`

## 1 现象

- 触发条件 / 发现方式
- 可观察到的错误表现
- 影响范围

## 2 诊断过程

### 假设 1：[描述]
- 验证方法：...
- 结果：排除 / 确认

### 假设 2：[描述]
- 验证方法：...
- 结果：排除 / 确认

## 3 根因分析

- 根本原因
- 涉及的代码路径（文件:行号）
- 为什么之前没有发现

## 4 修复方案

- 选择的方案及理由
- 考虑过但放弃的替代方案
- 修改的文件列表

## 5 验证

- 如何确认修复有效
- 回归测试（如有）

## 6 经验教训

- 通用化的检查模式（可提取到 PATTERNS.md）
- 架构/设计层面的改进建议
```
