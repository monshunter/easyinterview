# 报告模板

## 1 代码审查模板

```markdown
# ${Subject} 代码审查

> **日期**: YYYY-MM-DD
> **审查人**: xxx

## 1 审查范围

描述本次审查覆盖的代码范围。

## 2 发现的问题

### 2.1 严重问题

- 问题描述
  - **位置**：文件路径:行号
  - **建议**：修复建议

### 2.2 一般问题

- 问题描述

### 2.3 建议改进

- 改进建议

## 3 总结

总体评价和结论。
```

## 2 实施评估 / 交付复盘模板

```markdown
# ${Subject} 交付复盘报告

> **日期**: YYYY-MM-DD
> **审查人**: xxx

## 1 复盘范围与成功证据

- 本次交付覆盖的功能或修复范围
- 已通过的测试、BDD、部署验证或其他交付证据

## 2 会话中的主要阻点/痛点

- 痛点描述
  - **证据**：日志 / 报告 / 会话事实
  - **影响**：返工、误判、阻塞、额外环境成本等

## 3 根因归类

- 根因描述
  - **类别**：skill / README / AGENTS.md / spec-plan / 无需仓库改动

## 4 对流程资产的改进建议

- 建议描述
  - **落点**：目标 skill / README / AGENTS.md / spec-plan
  - **优先级**：high / medium / low

## 5 建议优先级与后续动作

- 下一轮最值得实施的改进项
- 可以延后处理的优化项
```

## 3 完成验证模板

```markdown
# ${Subject} 完成验证

> **日期**: YYYY-MM-DD
> **验证人**: xxx

**关联计划**: [计划文档](../plan/xxx/implementation.md)

## 1 验证范围

描述本次验证覆盖的功能范围。

## 2 验证结果

### 2.1 功能验证

| 功能点 | 结果 | 备注 |
|--------|------|------|
| 功能 1 | PASS/FAIL | 备注 |

### 2.2 测试覆盖

| 模块 | 覆盖率 | 要求 |
|------|--------|------|
| 模块 1 | 85% | 80% |

## 3 遗留问题

- 问题描述

## 4 结论

验证通过/不通过，说明原因。
```
