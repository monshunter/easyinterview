# UI Design 文档模板

## 1 目标架构模板

````markdown
# ${Subject} 目标架构

> **版本**: 1.0
> **状态**: active
> **更新日期**: YYYY-MM-DD

## 1 文档目的

说明本文档维护的目标 UI 架构、模块边界或交互流程，并注明是否已与 `ui-design/index.html` 当前静态原型对齐。

## 2 已确认决策

列出来自讨论并已反映到静态 UI 的产品和交互决策。

## 3 目标结构

```text
[App]
├─ Module A
├─ Module B
└─ Module C
```

## 4 用户流程

按用户目标描述主要路径、返回路径和异常路径。

## 5 模块边界

说明每个模块负责什么、不负责什么，以及是否属于一级导航、会话级页面或用户菜单入口。

## 6 后续实现输入

列出后续原型、路由、状态和页面重构需要遵守的约束。
````

## 2 模块表模板

```markdown
| 模块 | 用户任务 | 保留页面/能力 | 不再保留 | 说明 |
|------|----------|----------------|----------|------|
| 模拟面试 | 确认 JD、简历和轮次后开始一场完整面试 | `workspace`、`practice`、`voice`、`generating` | 当前岗位一级导航、练习模式卡片 | `workspace` 的语义是当前面试规划 |
| 简历 | 管理简历输入和版本 | `resume_versions`、原始简历预览、结构化主版本、岗位定制版本 | 独立经历库 | 简历和 JD 都是面试输入 |
| 报告 | 查看某次模拟面试的证据化结果 | `report`、题目回顾、证据详情、复练当前轮、进入下一轮 | 面试报告一级导航、时间线、刊物页 | 报告必须隶属于 session |
| 复盘 | 复盘真实面试 | `debrief`、复盘记录、复盘分析、复盘面试 | 模拟报告错题本 / 单题复练 | 记录真实过程并发起完整复盘面试 |
```
