# UI Design 文档模板

## 1 目标架构模板

````markdown
# ${Subject} 目标架构

> **版本**: 1.0
> **状态**: active
> **更新日期**: YYYY-MM-DD

## 1 文档目的

说明本文档维护的目标 UI 架构、模块边界或交互流程，并注明是否已与 `ui-design/index.html` 当前静态原型对齐。

## 2 已确认决策

列出来自讨论并已反映到静态 UI 的产品和交互决策。

## 3 目标结构

```text
[App]
├─ Module A
├─ Module B
└─ Module C
```

## 4 用户流程

按用户目标描述主要路径、返回路径和异常路径。

## 5 模块边界

说明每个模块负责什么、不负责什么，以及是否属于一级导航、会话级页面或用户菜单入口。

## 6 后续实现输入

列出后续原型、路由、状态和页面重构需要遵守的约束。
````

## 2 模块表模板

```markdown
| 模块 | 用户任务 | 当前页面/能力 | 范围边界 | 说明 |
|------|----------|----------------|----------|------|
| 面试 | 浏览面试规划列表，确认 JD、简历和轮次后开始一场完整面试 | `workspace` 规划列表 / 面试规划详情、连续文本 `practice`、`generating` | 当前岗位一级导航、题目地图、专用提示模式、可点击电话入口、独立语音页骨架 | `workspace` 无上下文时是面试规划列表，带上下文时复用统一面试规划详情母版；Practice 只保留自然推进的连续文本聊天 |
| 简历 | 管理简历资产 | `resume_versions`、平铺简历列表、上传 / 粘贴创建、只读详情正文 | 独立经历库、版本树 / 主版本 / 岗位定制版本、详情二次编辑 | 简历和 JD 都是面试输入 |
| 报告 | 查看某次模拟面试的证据化结果 | `report`、准备度、维度、证据详情、下一步、复练当前轮、进入下一轮 | 面试报告一级导航、逐题回顾、时间线、刊物页 | 报告必须隶属于 session，且以整场 conversation 为分析单位 |
```
