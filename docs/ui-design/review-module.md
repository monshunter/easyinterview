# 复盘目标模块

> **版本**: 2.4
> **状态**: active
> **更新日期**: 2026-05-02

## 1 文档目的

本文档定义一级 `复盘` 模块的目标边界。复盘模块不承载模拟面试报告里的错题、题目回顾或单题建议；它负责引导用户复盘一场真实发生过的面试，并基于目标岗位、关联模拟面试和绑定简历记录完整真实面试过程，再引导用户进行一场 `复盘面试`。

## 2 核心决策

1. 复盘是一级模块，对应顶部导航 `复盘`。
2. 复盘的对象是真实面试，不是模拟面试报告里的错题本。
3. 复盘开始时必须引导用户选择目标岗位 / JD；点击上下文卡片时在当前页打开 JD 选择弹窗。
4. 复盘需要选择关联模拟面试，作为分析真实面试时的对比基线；点击上下文卡片时在当前页打开模拟面试选择弹窗。
5. 复盘需要选择绑定简历，用于回答证据和简历素材对比；点击上下文卡片时在当前页打开简历选择弹窗。
6. 复盘过程可以读取用户简历、JD 和模拟报告，生成可能被问到的问题作为引导。
7. AI 推荐的问题不一定就是用户真实遇到的问题；用户必须可以跳过，也必须可以输入真实问题。
8. 复盘流程分为 `复盘记录`、`复盘分析`、`复盘面试` 三步，层层递进。
9. 复盘终点是发起一场完整 `复盘面试`，不是“带到下一轮”。

## 3 模块结构

```text
Debrief
├─ Context Strip
│  ├─ 目标岗位 / JD
│  │  └─ 更换 -> JD Picker Modal
│  ├─ 关联模拟面试
│  │  └─ 选择 -> Mock Interview Picker Modal
│  └─ 绑定简历
│     └─ 更换 -> Resume Picker Modal
├─ Stepper
│  ├─ 01 复盘记录
│  ├─ 02 复盘分析
│  └─ 03 复盘面试
├─ 复盘记录
│  ├─ AI 逐题推荐可能问题
│  ├─ 遇到过，记录
│  ├─ 没问到，跳过
│  ├─ 改成真实问题
│  ├─ 已形成的问题卡片
│  └─ 每题内部记录回答、追问、面试官反应、遗漏、整体感受
├─ 复盘分析
│  ├─ 题目重合度
│  ├─ 真实表现对比
│  ├─ 对照目标 JD
│  └─ 对照绑定简历
└─ 复盘面试
   ├─ 优先复现真实问题
   ├─ AI 追问薄弱处
   └─ 开始复盘面试
```

## 4 复盘流程

```text
Debrief Home
  -> 点击目标岗位 / JD 卡片 -> JD Picker Modal -> 选择目标岗位 / JD
  -> 点击关联模拟面试卡片 -> Mock Interview Picker Modal -> 选择关联模拟面试
  -> 点击绑定简历卡片 -> Resume Picker Modal -> 确认绑定简历
  -> AI 根据 JD + 简历 + 模拟报告生成可能问题
  -> 用户逐步回忆真实面试
     ├─ 遇到过 -> 记录问题和回答
     ├─ 没遇到 -> 跳过
     ├─ 推荐题不准确 -> 改成真实问题
     └─ AI 未覆盖 -> 用户手动添加真实问题
  -> 形成问题卡片
  -> AI 生成复盘分析
  -> 生成复盘面试内容
  -> 开始 Debrief Interview
```

## 5 复盘记录行为

### 5.1 上下文选择弹窗

```text
Context Strip
├─ 目标岗位 / JD
│  └─ 打开 JD Picker Modal
├─ 关联模拟面试
│  └─ 打开 Mock Interview Picker Modal
└─ 绑定简历
   └─ 打开 Resume Picker Modal
```

三个上下文动作都不得跳出复盘页。用户此时的任务是为当前复盘选择输入，而不是离开流程去管理模拟面试、报告或简历资产。

### 5.2 AI 推荐问题

AI 推荐问题用于降低用户回忆成本，不能被默认写成真实面试记录。

```text
SuggestedQuestion
├─ stage
├─ questionText
├─ whyLikelyAsked
├─ source: JD / resume / mock report
└─ userAction: occurred / skipped / edited
```

### 5.3 用户真实问题

用户可以随时输入真实遇到的问题。

```text
RealInterviewQuestion
├─ stage
├─ questionText
├─ userAnswer
├─ interviewerFollowUps
├─ interviewerReaction
├─ userReflection
├─ perceivedOutcome
└─ evidenceSource: user_entered / ai_confirmed
```

### 5.4 问题卡片

```text
QuestionCard
├─ Q 序号
├─ 阶段
├─ 真实遇到 / 已确认
├─ 问题文本
├─ 当时回答
├─ 追问 / 面试官反应
├─ 面试官反应标签
├─ 遗漏点 / 需要保留的信息
└─ 整体感受
```

当前静态 UI 的面试官反应标签包括 `明显投入`、`礼貌回应`、`追问 / 反推`、`有保留`；整体感受使用轻量情绪刻度，帮助用户还原真实场景，但不替代证据化分析。

## 6 复盘分析

复盘分析必须以已选择的上下文做对比。

```text
Debrief Analysis
├─ 与关联模拟面试对比
│  ├─ 题目重合度
│  ├─ 模拟中预测但真实未出现
│  └─ 真实出现但模拟未覆盖
├─ 与目标 JD 对比
│  ├─ 真实面试考察重点
│  └─ 暴露出的岗位风险
├─ 与绑定简历对比
│  ├─ 简历中有但真实回答没讲清的证据
│  └─ 简历中缺失但真实面试被追问的点
└─ 复盘面试内容建议
```

“分析真实面试”模块不得出现“将带入复盘面试”这类把内容和下一步混在一起的表达。真实问题、薄弱追问和复盘面试内容应归入 `复盘面试` 模块。

## 7 复盘面试

复盘面试是一场完整模拟面试，但生成上下文来自复盘记录。

```text
Debrief Analysis
  -> Create Debrief Interview
     ├─ 使用 TargetJob / JD
     ├─ 使用 ResumeVersion
     ├─ 优先复现真实问题
     ├─ 尽量按真实顺序重问
     ├─ 补充真实面试暴露出的薄弱点
     └─ 对照真实回答与复盘回答
```

复盘面试不是单题复练，也不是错题本 retry。它仍然是一场完整面试。

## 8 与模拟面试报告的边界

```text
模拟面试报告
└─ 题目回顾 / 回答分析 / 证据 / 本轮复练

复盘
└─ 真实过程记录 / 真实表现分析 / 复盘面试
```

模拟面试报告里的错题或题目问题不进入复盘模块。复盘模块只处理真实面试过程。关联模拟面试只作为对比基线，不是错题输入队列。

## 9 页面框架

```text
[Debrief]
├─ Header
│  ├─ 复盘 · 公司 · 轮次 · 日期
│  └─ 像真人复盘一样，还原真实面试，再进入复盘面试
├─ Context Strip
│  ├─ 目标岗位 / JD -> JD Picker Modal
│  ├─ 关联模拟面试 -> Mock Interview Picker Modal
│  └─ 绑定简历 -> Resume Picker Modal
├─ Stepper
│  ├─ 复盘记录
│  ├─ 复盘分析
│  └─ 复盘面试
├─ Record
│  ├─ AI 逐题复盘记录
│  ├─ 问题卡片
│  └─ 整体感受
├─ Analysis
│  ├─ 指标对比
│  ├─ 对照目标 JD
│  └─ 对照绑定简历
└─ Debrief Interview
   ├─ 复盘面试内容
   ├─ 复盘面试上下文
   └─ 开始复盘面试
```

## 10 不做的事

```text
不做:
├─ 接收模拟报告错题作为复盘输入
├─ 独立错题本
├─ 错题 retry
├─ 单题复练
├─ Drill builder
├─ 把 AI 推荐问题默认写成真实问题
└─ 把复盘终点定义为下一轮面试
```
