# 多 JD 与多简历目标管理结构

> **版本**: 2.3
> **状态**: active
> **更新日期**: 2026-07-06

## 1 文档目的

本文档定义当前 UI 中多份 JD、多份简历以及二者绑定关系的管理方式。JD 和简历都是一等公民；模拟面试规划负责把两者和面试轮次组合成一次模拟面试输入。

## 2 设计原则

1. 多 JD 以 `TargetJob` 管理，入口包括首页 JD 导入和最近模拟面试；岗位推荐模块已删除（D-17）。
2. 多简历以平铺 `Resume` 资产管理，入口在一级简历模块；不存在版本树、主版本或岗位定制版本（D-20）。
3. 每个模拟面试规划绑定一个 `TargetJob/JD`、一份 `Resume` 和一个 `InterviewRound`。
4. 系统必须同时保留每份简历的原始来源、解析文本和结构化内容。
5. 简历缺失时不阻断用户看 JD，但在需要保存、绑定或进入个性化模拟面试前触发补全和登录。
6. 模拟面试规划不是简历资产管理中心，只负责选择当前面试使用哪份简历。

## 4 多 JD 管理

### 4.1 首页

首页不是岗位资产管理页，而是开始或继续面试的入口，也是 JD 获取的唯一入口。

```text
Home
├─ Start New Interview
│  ├─ 输入源（二选一）
│  │  ├─ 粘贴 JD 输入框
│  │  └─ 上传 JD 文件
│  ├─ URL 导入
│  ├─ 选择已有 ready 简历（适度宽度下拉框）
│  │  └─ 还没有简历？1 分钟创建（下拉框右侧同行）
│  └─ 立即面试（简历选择下方）
└─ Recent Mock Interviews
   ├─ 最多 3 张最近卡片
   ├─ 岗位名称 / 公司 / 状态 / 面试轮次节点
   ├─ Open Mock Interview Plan
   └─ 更多 -> Mock Interview
```

### 4.2 模拟面试规划切换

```text
Mock Interview Plan Header
├─ 当前面试规划
├─ 切换规划
└─ 新建规划
```

切换规划时，页面上下文必须完整切换，包括 JD、简历绑定、当前轮次、历史会话和报告。

## 5 多简历管理

### 5.1 简历入口

```text
Resume / resume_versions
├─ Resume Workshop List（平铺）
│  ├─ 简历名称 / 语言
│  ├─ 来源（上传 / 粘贴）
│  ├─ 最近编辑
│  └─ 打开
├─ 新建简历
│  ├─ 上传文件
│  ├─ 粘贴文本
│  ├─ Agent 解析
│  └─ 预览确认保存 -> 加入当前列表
└─ Resume Detail
   ├─ 预览: 原件预览 / 导出 PDF / 复制纯文本
   ├─ 改写建议: 每条仅采纳；采纳后预览并选择覆盖或另存
   └─ 手动编辑: 保存改动

Mock Interview Plan / Parse
└─ 绑定简历
   ├─ 当前绑定简历
   └─ 更换 -> Resume Picker Modal
```

### 5.2 改写采纳收口

```text
Resume Detail · Rewrites
  -> 逐条查看原句 / AI 改写 / 为什么这么改
  -> 采纳需要的条目（默认不动作）
  -> 预览并保存
     ├─ 确认前预览整份改写结果
     ├─ 覆盖原简历 -> 更新当前简历结构化内容
     └─ 保存为新简历 -> 列表新增一份简历
```

两种结果都不修改原始来源快照；取消确认不写入任何简历。

## 6 JD 与简历绑定

```text
MockInterviewPlan
├─ planId
├─ targetJobId
├─ jdId
├─ resumeId
├─ roundId
└─ source: home / report_replay / report_next_round
```

同一份 JD 可以和不同简历形成不同面试规划。同一份简历也可以被多个 JD 使用。

## 7 UI 行为

| 场景 | 目标行为 |
|------|----------|
| 用户新增 JD | 首页先选择已有 ready 简历，再点击「立即面试」进入 `parse` 核对 JD；`parse` 继承首页显式选择的简历并确认轮次后立即面试，或仅保存规划进入模拟面试规划回访 |
| 用户有多份 JD | 首页最多显示最近 3 条模拟面试；更多内容通过“更多”进入一级 `模拟面试` 列表页 |
| 用户不想继续当前规划 | 在模拟面试页点击切换规划或新建规划 |
| 用户首次无简历 | 首页、解析确认页或模拟面试规划提示创建简历；首页不提供上传简历入口，只跳转到 `resume_versions(flow=create)` |
| 用户上传新简历 | 创建新的 `Resume`，解析后经预览确认保存 |
| 用户粘贴简历 | 创建新的 `Resume`，保留粘贴文本，并经预览确认保存 |
| 用户查看简历资产 | 平铺列表查看全部简历 |
| 用户查看原件 | 在简历详情页打开只读弹层，展示来源关系、原始文件视图和解析文本快照 |
| 用户采纳改写建议 | 采纳条目进入待保存集合；通过确认前预览选择覆盖原简历或保存为新简历 |
| 用户导出、复制或保存简历 | 给出可见反馈；保存为新简历时应在当前列表态体现结果 |
| 用户更换面试绑定简历 | 在弹窗选择已有 `Resume` 并绑定 |
| 用户在首页新建面试规划 | 在首页先从粘贴 JD 输入框或上传文件中二选一；通过适度宽度下拉框选择已有 ready `Resume`，创建简历入口在下拉框右侧同排；未选择简历或未提供 JD 前「立即面试」禁用，按钮位于简历选择下方，成功导入后 route 必须携带真实 `resumeId` |

## 8 不做的事

```text
不做:
├─ 岗位推荐 / 搜岗聚合（D-17）
├─ 简历版本树 / 主版本 / 岗位定制版本 / 分叉（D-20）
├─ 轻量问答建档（D-20）
├─ 独立经历库
├─ 要求用户先维护完整画像才能开始
├─ 多岗位信息混合在一个模拟面试规划里
├─ 让 Settings 承担简历管理职责
├─ 让更换简历直接跳出当前面试规划
└─ 覆盖原始来源快照
```

原始来源是用户提供的证据，任何结构化分析或改写采纳都不能覆盖它；`覆盖原简历` 覆盖的是该简历的结构化内容，不是原始来源。
