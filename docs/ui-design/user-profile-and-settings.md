# 用户画像与设置目标结构

> **版本**: 1.7
> **状态**: active
> **更新日期**: 2026-06-12

## 1 文档目的

本文档定义当前静态 UI 中 `用户画像` 和 `设置与隐私` 的边界。两者都从用户菜单进入，但服务不同任务：用户画像解释系统如何理解用户，设置与隐私管理账号基础信息和数据控制。

## 2 核心决策

1. 用户画像不是注册 onboarding，也不是个人基础资料表。
2. 用户画像由系统根据简历、JD、模拟面试、复盘和用户修正持续沉淀。
3. 用户画像必须可查看来源、可修正、可说明被哪些模块使用。
4. 个人设置只保存账号身份、登录安全、界面语言、时区、字体预设和隐私数据操作。
5. 岗位、年限、目标方向、薪资偏好、技能强弱等一对多或推断型信息不放在个人设置中。
6. 顶栏主题色、暗色模式和语言下拉是全局显示控制；设置页补充维护字体预设，不负责解释用户能力画像。

```text
[User Profile]
├─ Header
│  ├─ 用户画像 · AI 汇总
│  ├─ 系统理解你的那份画像
│  └─ 来源说明
├─ Identity Summary
│  ├─ 姓名 / 昵称
│  ├─ 职业摘要
│  ├─ 置信度
│  └─ 来源统计
│     ├─ 简历
│     ├─ JD
│     ├─ 模拟
│     └─ 复盘
├─ Sections
│  ├─ 职业定位
│  ├─ 技能与深度
│  ├─ 经历证据
│  └─ 面试表现
├─ Insight Cards
│  ├─ AI 当前推断
│  ├─ 置信度
│  ├─ 来源
│  └─ 修正入口
├─ 被哪些模块使用
│  ├─ 模拟面试规划
│  ├─ 报告分析维度
│  └─ 公开分享(关闭)
├─ 最近证据来源
└─ 纠偏原则
   └─ 用户修正优先级高于 AI 推断
```

## 4 用户画像修正流程

```text
用户点击修正
  -> Profile Correction Modal
     ├─ AI 当前推断
     ├─ 用户修正或补充
     ├─ 保存修正
     └─ 保留原始证据
```

保存后：

```text
Correction
├─ 不删除原始证据
├─ 作为高优先级覆盖层
├─ 用于模拟面试规划
├─ 用于报告分析维度
└─ 用于复盘分析
```

## 5 设置与隐私页面结构

```text
[Settings & Privacy]
├─ 个人资料
│  ├─ 账号基础信息
│  │  ├─ 显示姓名
│  │  ├─ 登录邮箱
│  │  ├─ 手机号
│  │  ├─ 界面语言
│  │  └─ 时区
│  ├─ 登录与安全
│  │  └─ 登录方式（邮箱验证码）
│  ├─ 界面偏好
│     └─ 字体预设
│        ├─ 编辑级: Noto Serif SC + Inter
│        ├─ 现代: Source Serif Pro + Geist
│        └─ 杂志: Cormorant Garamond + IBM Plex Sans
│  └─ 产品信息
│     ├─ 产品名称: EasyInterview
│     └─ 版本: v1.0
├─ 隐私与数据
│  ├─ 数据留存开关
│  │  ├─ 保留语音录音（仅当次会话）
│  │  ├─ 长期保留文字转写
│  │  ├─ 保留上传的简历
│  │  └─ 匿名贡献样本用于改进评分
│  ├─ 数据概览
│  ├─ 导出数据
│  ├─ 删除单次会话
│  ├─ 删除所有练习数据
│  └─ 注销账号
```

设置页只有 `个人资料` 与 `隐私与数据` 两个 tab；通知与订阅 P1 占位 tab 已删除（D-21）。设置页不展示目标角色、年限、求职偏好、技能标签或岗位相关信息。

当前静态 UI 的 `设置与隐私` 页面已将导出、删除、通知和订阅说明同步为 `题目回顾`、`准备度信号`、`练习数据`、`画像证据` 等当前模块术语。旧 `错题`、`成长`、`经历卡片` 文案不应恢复为独立错题本、成长中心或经历库模块入口。

### 5.1 顶栏与设置页的偏好边界

```text
TopBar
├─ 主题色
├─ 暗色模式
└─ 语言下拉

Settings -> 个人资料
└─ 界面偏好
   └─ 字体预设
```

主题色、暗色和语言下拉是跨页面即时显示控制；字体预设在设置页中成组维护，切换时同时改变标题 serif 和正文 sans。等宽字体用于数字、标签和状态，不随预设变化。

## 6 与其他模块的关系

```text
Resume
└─ 提供简历、项目、技能和证据来源

Mock Interview Plan
└─ 读取用户画像补充风险和可调用经历

Interview Session
└─ 读取用户画像作为问题和追问背景，但不直接要求用户维护画像

Report Dashboard
└─ 把报告信号写回用户画像证据层

Debrief
└─ 把复盘信号写回用户画像证据层
```

## 7 后续实现输入

1. 用户菜单中的 `用户画像` 和 `设置与隐私` 必须分开。
2. 设置页个人资料只保留账号基础信息；设置页只有个人资料与隐私数据两个 tab（D-21）。
3. 用户画像必须展示来源统计和最近证据；画像页是画像的唯一呈现处（岗位推荐画像摘要已随 D-17 删除）。
4. 用户修正必须作为覆盖层，不覆盖或删除原始证据。
5. 模拟面试规划、报告和复盘读取画像时，应优先读取用户修正。
