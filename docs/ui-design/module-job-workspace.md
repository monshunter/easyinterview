# Mock Interview 当前面试规划目标模块

> **版本**: 1.14
> **状态**: active
> **更新日期**: 2026-07-06

## 1 文档目的

本文档定义当前静态 UI 中 `模拟面试` 一级模块的目标结构。该页面不再定义为“当前岗位”，也不是首次 JD 导入的必经确认页；它是既有面试规划的回访枢纽：用户从首页最近模拟面试、报告、会话历史或一级导航回到这里，检查上下文、切换或新建规划，并再次发起面试。首页最近模拟面试只展示 3 条快捷卡片，`更多` 进入本页列表。首次导入新 JD 时，首页作为新建模拟面试规划快捷入口，必须先在粘贴 JD 输入框与上传文件 source 之间二选一，再通过适度宽度的下拉框显式选择已有 ready 简历；`还没有简历？1 分钟创建 →` 与下拉框同一行水平对齐，`立即面试` 位于简历选择下方。`parse` 解析确认页继承该选择并继续核对 JD / 轮次，缺少或无效简历时才兜底阻断和引导创建，系统不得默认替用户选中最近简历。

## 2 模块职责

当前面试规划负责：

- 明确这场模拟面试绑定的目标岗位 / JD。
- 明确这场模拟面试使用的简历。
- 明确当前处于哪一轮真实面试节点。
- 允许用户切换已有面试规划或新建规划。
- 允许用户在弹窗中更换绑定简历。
- 展示 JD 拆解、公司情报嵌入卡片、我的准备和当前面试规划下的模拟面试历史。
- 提供唯一主入口：`立即面试`。

当前面试规划不负责：

- 作为“当前岗位”一级模块存在。
- 作为首次 JD 导入链路中 `parse` 与 session 之间的第二个全页确认。
- 管理多岗位资产的全部生命周期。
- 管理简历资产。
- 展示练习模式卡片。
- 提供热身、反问、单题深钻或针对性复练。
- 承载无上下文的面试报告入口。

## 4 关键交互

### 4.1 进入当前面试规划

```text
入口:
├─ Home 最近模拟面试卡片（最多 3 条）或“更多”
├─ Home 新建规划快捷入口（JD source 二选一 + 选择已有简历 + JD import）
├─ Parse 解析确认页的仅保存规划
├─ 一级导航 模拟面试
├─ Report 的复练当前轮
└─ Report 的进入下一轮

进入:
  -> Mock Interview Plan(planId, targetJobId, jdId, resumeId, roundId)
```

无论从哪个入口进入，页面都必须明确显示 `JD + 简历 + 面试轮次`。首次导入主路径为 `Home 选择 JD source -> 选择已有简历 -> 立即面试 -> Parse 核对 -> practice`，不经过本页做第二次全页确认；只有用户选择“仅保存规划”或回访既有规划时才进入本页。

### 4.2 切换或新建规划

```text
用户不想继续当前规划
  -> 点击切换规划
     ├─ 选择已有 TargetJob / 简历 / Round 组合
     └─ 更新当前页面上下文
  -> 或点击新建规划
     └─ 回到首页导入新的 JD
```

这解决用户从一级 `模拟面试` 进入后，不想继续最近规划时无路可走的问题。

### 4.3 更换简历

```text
绑定简历卡片
  -> 更换
  -> Resume Picker Modal
     ├─ 展示已有简历名称
     ├─ 展示来源 / 匹配信息
     ├─ 选择一份简历
     └─ 确认绑定到当前规划
```

更换简历不应直接跳转到简历模块。用户要做的是为当前模拟面试选择输入，而不是离开流程管理简历资产。

### 4.4 立即面试

```text
Mock Interview Plan
  -> 立即面试
  -> requestAuth(create_session)
  -> Interview Session(sessionId)
     ├─ 文本面试
     └─ 语音面试
```

`立即面试` 必须携带 `planId + targetJobId + jdId + resumeId + roundId`，并在登录打断后通过 pending action 恢复。面试形式可在面试页选择或切换。规划页不展示模式卡片，也不让用户选择专项练习。

### 4.5 公司情报嵌入卡片

```text
公司情报嵌入卡片（规划页内）
├─ 一句话画像
├─ 近期公开信号（精选）
├─ 反问建议（精选）
└─ 合规来源数量与刷新时间说明
```

公司情报只使用合规公开来源，不展示雇主评分聚合、不抓登录后内容、不使用私域数据。它用于面试前理解公司和定制反问，不成为一级导航；`company_intel` 独立详情页与 route 已删除（D-18），嵌入卡片是唯一呈现。

### 4.6 查看当前规划的模拟面试历史

```text
右侧当前规划的模拟面试历史
  -> 点击某一次会话
  -> Report Dashboard(sessionId)
```

这里展示的是当前 `MockInterviewPlan` / `TargetJob` / `JD` 范围内的完整模拟面试会话历史，不混入其他公司、其他岗位或其他 JD 的会话。标题不能再使用旧版“岗位定制核心面、单题深钻、英文热身、反问专练”等专项流程。

## 7 与被移除模块的边界

| 被移除模块或流程 | 当前面试规划内是否保留相关能力 | 边界 |
|------------------|-------------------------------|------|
| 成长 | 否 | 不展示长期成长中心 |
| 多轮计划 | 否 | 只展示当前目标岗位的面试轮次节点，不维护独立计划系统 |
| 经历库 | 否 | 面试可读取简历和画像，但不要求用户维护经历库 |
| 追问树 | 否 | 追问在面试会话内发生，不作为入口 |
| 针对性复练 | 否 | 报告可发起复练当前轮，但不是单题 Drill |
| 练习模式选择 | 否 | 只有 `立即面试` 主动作 |

## 8 后续实现输入

1. 顶部导航文案为 `模拟面试`。
2. 页面主语使用 `当前面试规划`，不得再用 `当前岗位` 表示一级模块。
3. 主 CTA 文案为 `立即面试`。
4. 顶部线条节点展示真实面试轮次，不展示模拟面试内部题目。
5. 绑定简历的 `更换` 动作必须打开选择弹窗。
6. 右侧历史是当前面试规划 / 当前目标岗位下的完整模拟面试会话历史，不是全局历史，也不是专项练习历史。
7. 报告从历史会话或面试结束后进入，报告页必须带 session 上下文。
8. 公司情报只以规划页嵌入卡片呈现；`company_intel` 独立详情页与 route 已删除（D-18），旧 hash 归一回 `workspace`。
9. 公司情报嵌入卡片必须显示来源与刷新时间说明，并明确公开来源边界。
10. 页面缺少 JD 上下文时必须显示空状态并返回首页导入 JD；缺少简历绑定时必须提示创建或绑定简历，不展示假数据。
11. 本页是回访枢纽：不得把首次 JD 导入用户从 `parse` 强制带回本页做第二次全页确认；`parse` 的“仅保存规划”是进入本页的合法入口之一。
