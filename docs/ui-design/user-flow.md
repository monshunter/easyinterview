# EasyInterview UI 目标用户流程

> **版本**: 2.1
> **状态**: active
> **更新日期**: 2026-05-02

## 1 文档目的

本文档定义当前静态 UI 原型对应的目标用户流程。当前流程分为五条主线：首页启动、岗位推荐、模拟面试、简历管理、复盘。五条主线通过 `TargetJob/JD + ResumeVersion + InterviewRound` 组合在模拟面试规划中汇合。

## 2 主流程总图

```text
[首页启动]
Home
  ├─ 粘贴 JD -> 解析并确认面试 -> Mock Interview Plan
  ├─ 上传 JD / URL -> 解析并确认面试 -> Mock Interview Plan
  ├─ 最近模拟面试 -> Mock Interview Plan
  ├─ 还没有简历？1 分钟创建 -> Resume Intake
  ├─ 打开岗位推荐 -> Job Picks
  └─ 打开复盘 -> Debrief

[岗位推荐]
Job Picks
  -> 查看画像摘要和推荐 JD
  -> 在 为你推荐 / 联网搜索 / 关注列表 间切换
  -> 选择一份推荐 JD
  -> 确认面试
  -> JD 解析确认
  -> Mock Interview Plan

[模拟面试]
Mock Interview Plan
  -> 确认 JD / 目标岗位
  -> 选择绑定简历
  -> 确认当前面试轮次
  -> 立即面试
  -> Interview Session
  -> Report Dashboard

[简历管理]
Resume
  -> 简历工坊列表
  -> 按原始简历树管理 / 按版本平铺挑选
  -> 新建原始简历或从已有树分叉岗位定制版本
  -> 版本详情: 预览 / 改写建议 / 手动编辑

[复盘]
Debrief
  -> 选择目标岗位 / JD
  -> 选择关联模拟面试
  -> 选择绑定简历
  -> 复盘记录
  -> 复盘分析
  -> 复盘面试
```

## 3 首页流程

首页的首要任务是让用户开始或继续一场具体岗位相关的模拟面试。

```text
[Home]
├─ Hero
│  ├─ JD 粘贴输入框
│  ├─ 上传 .pdf / .docx / .md 弹窗
│  ├─ URL 导入弹窗
│  └─ 解析并确认面试
├─ 还没有简历？1 分钟创建
│  └─ Resume Intake
├─ 最近模拟面试
│  ├─ 目标岗位
│  ├─ 当前状态
│  └─ 面试轮次节点
└─ 辅助入口
   ├─ 岗位推荐
   └─ 复盘
```

首页不再展示“还没有目标岗位”“空白面试确认”等分散链路。最近列表不是岗位列表，而是最近模拟面试列表；卡片只保留上半部分核心信息，并展示这份目标岗位当前的面试轮次节点。

## 4 岗位推荐流程

```text
Job Picks
  -> 读取用户画像 / 简历 / 岗位偏好
  -> 展示 为你推荐 / 联网搜索 / 关注列表
     ├─ 匹配分
     ├─ 首要原因
     ├─ 必需项 / 加分项命中
     └─ 不匹配或风险提示
  -> 用户打开一张 JD 详情
  -> 确认面试
  -> JD 解析确认(parse)
  -> Mock Interview Plan
```

岗位推荐是一级导航。页面里的用户画像摘要用于解释推荐依据；完整画像详情在用户菜单的 `用户画像` 页面。当前静态 UI 中，推荐详情的主按钮不是直接打开 `workspace`，而是先进入 `parse`，让用户核对 JD 基础信息、必需项、加分项、隐性关注点和轮次假设。

## 5 模拟面试规划流程

模拟面试规划不是“当前岗位”一级模块。它是用户开始一场模拟面试前的资料确认页。

```text
Mock Interview Plan
├─ 当前面试规划
│  ├─ TargetJob / JD
│  ├─ ResumeVersion
│  └─ InterviewRound
├─ 切换规划
│  └─ 从已有 JD + 简历 + 轮次组合中选择
├─ 新建规划
│  └─ 回到首页输入或导入新的 JD
├─ 面试轮次
│  ├─ HR 初筛
│  ├─ 技术一面
│  ├─ 技术二面
│  └─ 经理面
├─ 绑定简历
│  └─ 更换 -> 简历选择弹窗
├─ JD 拆解 / 公司情报 / 我的准备
├─ 右侧当前规划的模拟面试历史
│  └─ 只展示当前 TargetJob / JD 范围内的会话
└─ 立即面试
```

如果用户从一级 `模拟面试` 进入但不想继续当前规划，必须能通过 `切换规划` 或 `新建规划` 切到新的面试上下文。

## 6 面试会话流程

```text
Mock Interview Plan
  -> 立即面试
  -> Interview Session
     ├─ 面试形式
     │  ├─ 文本面试: 打字回答
     │  └─ 语音面试: 实时语音对话
     ├─ 当前问题
     ├─ 对话记录
     ├─ 回答输入
     │  ├─ 文本输入
     │  ├─ 语音转文字填入输入框
     │  ├─ 提示
     │  └─ 跳过 / 提交
     ├─ JD 关联和可调用经历
     └─ 结束并生成报告
```

文本和语音是整场面试形式，不是练习类别。文本面试中的麦克风按钮是语音转文字，识别结果填入当前文本回答框，本质仍是文本面试。

## 7 报告流程

报告隶属于一次已完成的模拟面试，不能作为无上下文的一级页面打开。

```text
Interview Session
  -> Generating Report
  -> Report Dashboard(sessionId)
     ├─ 所属会话
     ├─ 目标岗位
     ├─ 面试轮次
     ├─ 绑定简历
     ├─ 完成时间
     ├─ 沟通形式
     ├─ 准备度详情
     ├─ 维度详情
     ├─ 题目回顾页
     ├─ 证据详情
     └─ 复练计划
        ├─ 路径 A: 复练当前轮
        └─ 路径 B: 进入下一轮
```

`复练当前轮` 和 `进入下一轮` 必须在报告中明确区分：

| 路径 | 含义 | 使用场景 |
|------|------|----------|
| 复练当前轮 | 重复当前面试轮次，把报告里的错题、证据缺口和追问风险带入 | 准备度提示建议再练，或用户想把本轮答扎实 |
| 进入下一轮 | 沿用同一 JD 和简历，但切换到下一轮面试官视角和题目结构 | 用户决定推进到下一轮，例如从技术一面进入技术二面 |

题目回顾承载“错题本”的用户价值，但不生成独立错题本模块，也不进入复盘。

## 8 简历流程

```text
Resume
├─ Resume Workshop List
│  ├─ Stats: 原始简历 / 全部版本 / 最高匹配 / 最近编辑
│  ├─ 按原始分组
│  │  ├─ 原始简历树
│  │  ├─ 主版本
│  │  ├─ 岗位定制版本
│  │  ├─ 归档原始简历
│  │  └─ 选为底稿 -> 基于这棵树新建版本
│  └─ 按版本平铺
│     ├─ 来源原始
│     ├─ 目标岗位
│     ├─ 匹配分
│     └─ 最近编辑
├─ Resume Create Flow
│  ├─ 上传文件
│  ├─ 粘贴文本
│  ├─ 轻量问答
│  ├─ Agent 解析
│  └─ 预览确认后保存 v1
├─ Resume Branch Flow
│  ├─ 底稿来源
│  ├─ 版本名称
│  ├─ 目标岗位 / 公司
│  ├─ 侧重方向
│  └─ Bullet 初始化方式
└─ Version Detail
   ├─ Branch Map: 原始简历 / 主版本 / 当前版本
   ├─ Preview: 只读预览 / 导出 PDF / 复制纯文本 / 查看原件
   ├─ Rewrites: 原句 / AI 改写 / 为什么这么改 / 拒绝 / 编辑 / 采纳
   └─ Edit: 一句话标题 / 简介 / 工作经历 / 技能
```

上传或制作出的简历必须有可识别名称。原始简历只读保存；结构化字段、改写建议和岗位定制版本都不能覆盖原件。主版本的改写建议入口应禁用或说明“主版本保持干净”；岗位定制版本里的 `拒绝 / 编辑 / 采纳` 只作用于当前版本，不影响主版本或其它兄弟版本。

## 9 首次无简历流程

```text
Home
  -> 还没有简历？1 分钟创建
     ├─ 上传 / 粘贴简历
     │  ├─ 保留原始文件或文本
     │  ├─ 解析结构化简历
     │  ├─ 预览结构化草稿
     │  └─ 确认后生成 v1
     └─ 轻量问答
        ├─ 目标岗位
        ├─ 最近工作 / 职位
        ├─ 主要方向
        ├─ 代表项目
        ├─ 量化结果
        └─ 生成 v1
```

简历引导不挡在首页之前。用户可以先看岗位或粘贴 JD，但保存简历、绑定简历、进入高度个性化面试时需要登录或授权保存。

## 10 复盘流程

```text
Debrief
  -> 选择上下文
     ├─ 点击目标岗位 / JD 卡片 -> 打开 JD 选择弹窗
     ├─ 点击关联模拟面试卡片 -> 打开模拟面试选择弹窗
     └─ 点击绑定简历卡片 -> 打开简历选择弹窗
  -> 复盘记录
     ├─ AI 逐题提出可能被问到的问题
     ├─ 用户选择遇到过并记录
     ├─ 用户选择没问到并跳过
     ├─ 用户把推荐题改成真实问题
     └─ 用户手动补充 AI 没覆盖的真实问题
  -> 形成多个问题卡片
  -> 复盘分析
     ├─ 对照目标 JD
     ├─ 对照绑定简历
     ├─ 对照关联模拟面试
     └─ 找出真实面试暴露的问题
  -> 复盘面试
     ├─ 优先复现真实问题
     ├─ AI 补充薄弱追问
     └─ 开始复盘面试
```

复盘终点是 `复盘面试`，不是“带到下一轮”。复盘面试是一场完整面试，不是单题复练。

## 11 认证流程

```text
默认:
Home 可直接访问

未登录用户:
├─ 顶部用户区显示 登录 / 注册
├─ 可粘贴 JD 和浏览公开状态
└─ 保存、上传、跨设备继续、查看账户数据时触发认证

认证页面:
├─ auth_login
│  ├─ 密码登录
│  ├─ 邮箱验证码
│  ├─ 第三方登录
│  └─ 忘记密码
├─ auth_register
│  └─ 创建账号并验证邮箱
├─ auth_verify
├─ auth_reset
└─ auth_logout
```

当前静态 UI 已落独立认证页面，但尚未把 `AuthGate(pendingAction)` 接入各业务按钮：登录或邮箱验证成功后回到 `Home`，退出登录后可重新登录或返回首页。后续实现动作级登录拦截时，必须补齐 pending action 恢复，不能在文档中声称当前静态稿已经具备该恢复行为。

## 12 全局显示控制流程

```text
TopBar
├─ 主题色
│  ├─ 暖陶 / 苔林 / 深海 / 梅子
│  └─ 自定义
│     ├─ 拖动色相
│     ├─ 拖动饱和度
│     └─ 恢复主题默认色
├─ 暗色模式
│  └─ 在当前主题下切换 light / dark
└─ 语言
   └─ 中 / EN

Settings -> 个人资料 -> 界面偏好
└─ 字体预设
   ├─ 编辑级
   ├─ 现代
   └─ 杂志
```

全局显示控制可在未登录状态下使用。它不改变业务进度，不触发认证，也不替代用户画像或账号设置。

## 13 用户画像与设置流程

```text
User Menu
├─ 用户画像
│  ├─ 系统推断
│  ├─ 来源统计
│  ├─ 最近证据
│  ├─ 用户修正
│  └─ 被哪些模块使用
└─ 设置与隐私
   ├─ 显示姓名
   ├─ 登录邮箱
   ├─ 手机号
   ├─ 界面语言
   ├─ 时区
   ├─ 密码 / 登录方式 / 两步验证
   ├─ 界面偏好
   │  └─ 字体预设
   ├─ 隐私、导出、删除
   ├─ 通知(P1 占位)
   └─ 订阅(P1 占位)
```

个人资料页面只保存账号基础信息。目标岗位、年限、求职偏好等一对多或由系统推断的信息应进入用户画像，而不是个人设置。

## 14 失败和返回路径

| 场景 | 目标行为 |
|------|----------|
| JD 解析失败 | 留在解析确认步骤，保留原始 JD 并允许重试 |
| 用户取消登录 | 当前静态稿未接业务级 AuthGate；后续接入时回到触发登录前页面，不丢失已输入内容 |
| 简历解析失败 | 保留原始文件或文本，提示用户重试、编辑或改用问答 |
| 用户无简历 | 可继续看 JD，但个性化承诺降低；开始面试前提示绑定或创建简历 |
| 用户创建简历但取消预览确认 | 返回简历输入页，保留当前输入，不生成正式 v1 |
| 用户创建岗位定制版本但缺少版本名称或目标岗位 | 停留在分叉表单，禁用创建动作并提示补全必要字段 |
| 用户不想继续当前面试规划 | 在模拟面试页通过切换规划或新建规划选择新上下文 |
| 模拟面试历史出现其他公司或岗位 | 过滤回当前面试规划 / 当前 TargetJob / JD 范围 |
| 面试中断 | 回到模拟面试规划，提供恢复 session 或重新开始 |
| 报告生成失败 | 显示重试和返回模拟面试规划，不创建空报告 |
| 用户选择复练当前轮 | 进入同一轮次的模拟面试规划 |
| 用户选择进入下一轮 | 创建下一轮面试规划，保留同一 JD 和简历 |
| AI 推荐复盘问题不匹配 | 用户可以跳过、编辑成真实问题或手动补充 |
| 用户要更换复盘上下文 | 在复盘页打开对应选择弹窗，不跳转到模拟面试、报告或简历页面 |
| 访问旧 hash route | 通过 `routeAliases` 折回当前目标模块，不进入已清理的旧页面 |
