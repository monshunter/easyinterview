# BUG-0095: output schema validation accepted trailing model prose

> 模块: eval | 严重度: medium | 状态: resolved
> 发现日期: 2026-05-24 | 关联 Commit: `fix(prompt-rubric): reject trailing AI output schema content (BUG-0095)`
> 关联日志: [2026-05-24](../work-journal/2026-05-24.md)

## 1 现象

- `plan-code-review prompt-rubric-registry/002-output-schema-contract --fix` 反查 A3 `validateOutputSchema` 时发现：validator 只解码第一个 JSON value 后立刻做 schema 校验。
- 触发样例：模型返回 `{"answer":"valid"} trailing prose` 时，第一个 JSON object 满足 schema，原实现不会把后续非空 prose 计为 `AI_OUTPUT_INVALID`。
- 影响范围：F3 output schema 的 strict JSON 契约可被 trailing prose 绕过；A3 validation status、`ai_output_validation_failures_total` 与 log 事件可能误记为通过，后续业务 parser 才暴露错误。

## 2 诊断过程

### 假设 1：prompt / schema lint 已覆盖 strict JSON trailing 内容

- 验证方法：读取 `scripts/lint/prompt_lint.py` 与 `prompt_lint_test.py`，确认 lint 主要校验 schema 子集、rendered contract block、example JSON 与 schema/struct required path。
- 结果：排除。lint 能证明 prompt body 的 example JSON 是 schema-valid，但不覆盖运行时 provider response 的 trailing token。

### 假设 2：A3 validator 会拒绝 trailing token

- 验证方法：读取 `backend/internal/ai/aiclient/observability/decorator.go` 的 `validateOutputSchema`，并新增 focused negative test `TestDecorator_OutputSchemaRejectsTrailingTokens`。
- 结果：确认缺口。Red 阶段测试失败，原实现未返回错误。

### 假设 3：缺口只影响测试断言，不影响运行时信号

- 验证方法：检查 decorator 在 schema validation error 时设置 `ValidationStatusInvalid`、返回 `AI_OUTPUT_INVALID`、递增 validation failure counter 的路径。
- 结果：排除。由于原实现没有产生 validation error，运行时观测信号也会误判。

## 3 根因分析

- 根本原因：`json.Decoder.Decode` 只调用一次，成功读取第一个 JSON document 后没有再次解码确认 EOF。
- 涉及路径：`backend/internal/ai/aiclient/observability/decorator.go`。
- 为什么之前没有发现：Phase 6 只覆盖 invalid JSON、缺 required、enum mismatch 与 array enum valid，没有覆盖 schema-valid JSON 后追加非空内容的 strict JSON negative path。

## 4 修复方案

- 在第一次 decode 成功后执行第二次 decode，只接受 `io.EOF`。
- 如果第二次 decode 得到第二个 JSON value 或非空 trailing token，返回 validation error，由 decorator 统一包装为 `AI_OUTPUT_INVALID`。
- 新增 `TestDecorator_OutputSchemaRejectsTrailingTokens`，作为 Phase 6 L2 remediation gate。

## 5 验证

- Red：`go test ./backend/internal/ai/aiclient/observability -run 'TestDecorator_OutputSchemaRejectsTrailingTokens' -count=1` 失败，错误为未返回 `AI_OUTPUT_INVALID`。
- Green：`go test ./backend/internal/ai/aiclient/observability -run 'TestDecorator_OutputSchemaRejectsTrailingTokens' -count=1` 通过。
- Adjacent：`go test ./backend/internal/ai/aiclient/observability -run 'TestDecorator_OutputSchema' -count=1` 通过。
- Regression：`go test ./backend/internal/ai/registry/... ./backend/internal/ai/aiclient/... ./backend/internal/targetjob/... ./backend/internal/resume/jobs/... ./backend/internal/review/... ./backend/internal/practice/... ./backend/internal/debrief/... ./backend/internal/jdmatch/... ./backend/cmd/api/... -race` 通过。

## 6 经验教训

- schema validator 不能只证明“第一个 JSON value 符合 schema”；strict JSON 契约必须证明整个响应是单个 JSON document。
- 对 provider / model 输出校验补 negative case 时，应同时覆盖 invalid JSON、missing required、enum mismatch、array top-level valid 和 trailing prose。
