# BUG-0150: archiveTargetJob handler was generated but not mounted in API mux

> 模块: workspace | 严重度: high | 状态: resolved
> 发现日期: 2026-07-09 | 关联 Commit: `fix(targetjob): persist workspace archive delete (BUG-0150)`
> 关联日志: `-`

## 1 现象

- 真实本地浏览器验收中，workspace 卡片右上角删除按钮已调用 generated `archiveTargetJob` client，但请求 `POST /api/v1/targets/{targetJobId}/archive` 返回 `HTTP 404 Not Found: 404 page not found`。
- 同一接口在 OpenAPI、generated server interface、TargetJob handler/service/store focused tests 中均已存在，导致组件测试和契约生成无法暴露真实 API runtime 404。
- 影响范围是 workspace 删除/归档：用户点击删除会失败并保留卡片，无法完成持久归档。

## 2 诊断过程

### 假设 1：前端请求路径或 base URL 错误

- 验证方法：在真实浏览器联调中点击 workspace 删除按钮，观察错误文案和 Network 目标；随后用 `curl -X POST /api/v1/targets/{targetJobId}/archive` 探测 route。
- 结果：排除。请求路径符合 OpenAPI；修复前返回裸 `404 page not found`，不是 generated B1 error envelope。

### 假设 2：handler/service/store 归档实现缺失

- 验证方法：反查 `backend/internal/targetjob/handler.go`、`service.go`、`store.go`，并运行 focused backend tests。
- 结果：排除。`Handler.ArchiveTargetJob`、`Service.ArchiveTargetJob` 和 `Store.ArchiveTargetJob` 均已实现，focused tests 覆盖 success、缺 `Idempotency-Key`、重复归档 conflict、跨用户 404 和 read-side soft-delete。

### 假设 3：生产 API mux 漏挂新增 route

- 验证方法：反查 `backend/cmd/api/main.go` TargetJob route mounting 和 `TestBuildAPIHandlerMountsTargetJobRoutesBehindSessionMiddleware`。
- 结果：确认。`/api/v1/targets/import`、`GET/PATCH /api/v1/targets/{targetJobId}` 已挂载，但新增的 `POST /api/v1/targets/{targetJobId}/archive` 没有进入 production mux。

## 3 根因分析

- 根本原因：B2/OpenAPI additive route 和 generated handler signature 已更新，但 production runtime route table 没有同步接入新增 operation。
- 涉及路径：
  - `backend/cmd/api/main.go`
  - `backend/cmd/api/main_test.go`
  - `backend/cmd/api/targetjob_http_scenario_test.go`
  - `backend/internal/targetjob/handler.go`
  - `frontend/src/app/screens/workspace/WorkspaceScreen.tsx`
- 为什么之前没有发现：focused TargetJob tests 验证的是 handler/service/store contract，OpenAPI/codegen gate 验证的是接口形状，缺少对 production mux route table 的 archive operation 断言；真实浏览器才经过 `cmd/api` mux。

## 4 修复方案

- 在 `backend/cmd/api/main.go` 挂载 `POST /api/v1/targets/{targetJobId}/archive`，使用 auth session middleware 并调用 `targetJobHandler.ArchiveTargetJob`。
- 扩展 `backend/cmd/api/main_test.go` 的 route mounting regression，证明 archive route 和其它 TargetJob route 一样受 session middleware 保护。
- 扩展 `backend/cmd/api/targetjob_http_scenario_test.go` fake store，保证 cmd/api focused tests 能编译并覆盖新的 TargetJob store interface。
- 真实后端 redeploy 后重新点击浏览器删除按钮，确认卡片删除、刷新后不回灌，并通过 DB readback 确认 `status='archived'` 与 `deleted_at`。

## 6 经验教训

- 对新增 OpenAPI operation，除了 generated interface、handler/service/store tests，还必须在 `cmd/api` production mux 层加 route mounting regression。
- 该问题符合 `docs/bugs/PATTERNS.md` 的“Domain service 已实现但 runtime caller 未接入”模式；当前模式库已有对应检查清单，本次不新增重复模式。
