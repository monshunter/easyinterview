# BUG-0172: UI Demo pruning gate missed renamed browser contracts and owner drift

> 模块: test | 严重度: medium | 状态: resolved
> 发现日期: 2026-07-15 | 关联 Commit: `fix(ui-design): close pruning review findings (BUG-0172)`
> 关联日志: [2026-07-15](../work-journal/2026-07-15.md)

## 1 现象

- 审查 `opt/ui-design-source-pruning-0715` 相较于 `main` 的变更时，`make lint-ui-demo-pruning` 报告 `active_residuals (0)`，但多个 active spec/plan 仍要求已经删除的 `responsive browser verification`。
- `frontend-shell/004-url-addressable-routing` 继续把 `#route=...` adapter、`bootstrapRoute.ts` 和 browser harness 写成当前合同，实际实现已经忽略 fragment 并删除对应文件。
- `frontend-shell/003-ui-design-responsive-browser-gate` 的验收标准把应删除目录误写成 `frontend/`。

## 2 诊断过程

### 假设 1：新增 pruning lint 没有覆盖改名后的 browser gate

- 验证方法：加入包含 `responsive browser verification`、`test:responsive-browser` 和 `serve-responsive-browser` 的 focused test。
- 结果：确认。旧实现返回空 finding；repo-wide 搜索同时发现 18 个 active residual。

### 假设 2：hash adapter 仍有正式 runtime 或 scenario consumer

- 验证方法：反查 `routeStore.ts`、route focused tests、003/004 context、正式 UI 文档和全仓 deleted-symbol search。
- 结果：排除。正式 route contract 只接受 canonical path/query；4 个 route suites 共 42 tests 证明 fragment 被忽略并在 mount canonicalization 时移除。

### 假设 3：既有全量门禁能发现 owner 语义冲突

- 验证方法：运行原 pruning lint、`make docs-check` 和根 `make test`。
- 结果：确认门禁缺口。这些命令均能通过，因为原 lint 只识别旧 parity 名称，docs gate 只检查 Header/INDEX/link 结构，代码测试不验证 completed plan 的当前措辞。

## 3 根因分析

- `scripts/lint/ui_demo_pruning.py` 的 forbidden pattern 覆盖 `pixel parity` 和旧 package script，却没有覆盖本次文档迁移实际采用的 browser-gate 同义词。
- 清理 003 时只更新了 004 指向新目录名的引用，没有反向审计 004 的当前合同、checklist、context discovery 和验收标准。
- 003 coverage matrix 与验收标准中的目录名缺少同一语义断言，因此 `frontend/` 误写未被 executable gate 捕获。

## 4 修复方案

- 先用 focused Red/Green test 固化三个 browser-gate 同义词，再扩展 pruning forbidden pattern。
- 将 lint 暴露的 18 个 active residual 改为现存的 formal component/responsive/accessibility gates 或明确的 real API/UI 场景 owner。
- 原地修订 004 plan/checklist/context，删除 hash-adapter/browser-harness 当前合同，保留 canonical path/query、fragment rejection 和 host fallback 真相。
- 修正 003 中两处 `frontend/` 目录误写，并通过 Phase 7 remediation checklist 记录当前证据。

## 5 验证

- `python3 -m pytest scripts/lint/ui_demo_pruning_test.py -q`：11 passed。
- `make lint-ui-demo-pruning`：`ui_demo_directory: absent`，`active_residuals (0)`。
- route focused Vitest：4 files / 42 tests passed。
- 根 `make test`：Python 544 passed / 4241 subtests、backend 全包、frontend 123 files / 969 tests passed。
- `make build`、`make docs-check`、`make codegen-check`、003/004 context validation 与 `git diff --check` 全部通过。

## 6 经验教训

- 删除 gate 的负向词表必须包含迁移过程中出现的同义词，并由每个同义词的 Red test 证明；不能只搜索最初命令名。
- pruning review 要反向检查引用新 owner 的 completed sibling plan，而不只是修链接路径。
- Header/INDEX/link gate 只能证明结构一致，不能代替 active contract 的语义残留扫描。
