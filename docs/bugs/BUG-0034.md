# BUG-0034: mockruntime named scenario test copied stale fixture expectation

> 模块: test | 严重度: medium | 状态: resolved
> 发现日期: 2026-05-10 | 关联 Commit: `fix(mock-contract): align mockruntime named scenarios`
> 关联日志: `-`

## 1 现象

- `cd backend && go test ./...` 在 `internal/api/mockruntime TestHandlerSelectsNamedSeedScenariosAndFailsUnknown/missing-session` 失败。
- 测试硬编码期望 `401 AUTH_UNAUTHORIZED`，但 `openapi/fixtures/PracticeSessions/getPracticeSession.json` 的 `missing-session` scenario 当前 truth source 是 `404 PRACTICE_SESSION_NOT_FOUND`。
- 单包 `cmd/api` 和 practice 相关包均已通过，剩余失败集中在 mock runtime 测试消费者。

## 2 诊断过程

### 假设 1：practice handler 未认证优先级错误

- 验证方法：读取 `backend-practice` spec 和 `getPracticeSession` fixture，确认未认证与资源不存在的错误边界。
- 结果：排除。未认证应返回 `401 AUTH_UNAUTHORIZED`，但 `missing-session` fixture 表达的是已认证上下文下资源不存在，当前 B2 fixture 为 `404 PRACTICE_SESSION_NOT_FOUND`。

### 假设 2：mockruntime handler 没有正确选择 named scenario

- 验证方法：读取 `backend/internal/api/mockruntime/mockruntime.go` 的 `Prefer: example=<scenario>` selector，并观察失败响应体。
- 结果：排除。handler 正确返回了 fixture 中的 `missing-session` scenario body；失败来自测试硬编码期望过期。

### 假设 3：mock-contract owner 测试复制了 fixture truth source

- 验证方法：读取 `backend/internal/api/mockruntime/mockruntime_test.go`。
- 结果：确认。测试用 hard-coded status / JSON field 断言 named scenarios，而不是读取对应 fixture scenario 的 `response.status` / `response.body`。

## 3 根因分析

- `TestHandlerSelectsNamedSeedScenariosAndFailsUnknown` 把 named scenario 期望复制成代码常量，违反 mock-contract-suite spec 中“B2 fixtures 是 mock 数据真理源”的约束。
- 当 `getPracticeSession` 的 `missing-session` fixture 收敛为 404 资源不存在语义后，测试消费者没有自动跟随 fixture 更新，导致 full-suite gate 失败。

## 4 修复方案

- 更新 `backend/internal/api/mockruntime/mockruntime_test.go`：named scenario 测试增加 `fixturePath`，通过 helper 读取 fixture scenario 的 `response.status` / `response.body`，再与 mock handler response 做 JSON 等价断言。
- 保留 unknown scenario 的 loud failure 断言，确保 selector 不会静默回落到 default。
- 原地更新 `mock-contract-suite/001-fixture-backed-mock-runtime` plan/checklist v1.3，新增并完成 4.5 remediation gate。

## 5 验证

- `cd backend && go test ./internal/api/mockruntime -run TestHandlerSelectsNamedSeedScenariosAndFailsUnknown -count=1`：Red 后 Green。
- `cd backend && go test ./internal/api/mockruntime -count=1`：PASS。
- `cd backend && go test ./...`：PASS。
- `python3 .agent-skills/sync-doc-index/scripts/sync-doc-index.py --check`：zero drift。

## 6 经验教训

- Fixture-backed mock tests 不应复制 fixture response 的 status / body / error code；应从 `openapi/fixtures` 读取 scenario truth source。
- 对 named scenario 只保留 selector 行为和 unknown scenario fail-loudly 的测试逻辑，具体 response 语义归 B2 fixture owner。
