# BUG-0022: speech adapters ignored profile timeouts

> 模块: platform | 严重度: medium | 状态: resolved
> 发现日期: 2026-05-08 | 关联 Commit: `fix(ai-provider): enforce speech adapter timeouts`
> 关联日志: [2026-05-08](../work-journal/2026-05-08.md)

## 1 现象

- 代码审查 `dev` 相较 `feat/cascaded-speech-provider` 时发现 `doubao_speech` 与 `minimax_speech` adapter 的 `postJSON` 接收 `timeoutMs` 参数，但没有给请求 context 设置 deadline。
- 当上游 speech provider 慢响应时，TTS / STT 调用会等待 HTTP client 自然返回，而不是按 profile `timeout_ms` 返回 `AI_PROVIDER_TIMEOUT`。
- 影响范围为 `practice.voice.stt.default` / `practice.voice.tts.default` 后续启用真实 provider 后的语音 turn 编排与 TTS fallback。

## 2 诊断过程

### 假设 1：client fallback 层没有识别 timeout
- 验证方法：检查 `backend/internal/ai/aiclient/client.go` 的 `executeWithFallback` 与既有 timeout condition。
- 结果：排除。client 层会根据 `AI_PROVIDER_TIMEOUT` 匹配 `timeout` fallback 条件。

### 假设 2：speech provider adapter 没有把 profile timeout 应用到 HTTP 请求
- 验证方法：对比 `openai_compatible.postJSON` / `postMultipart` 与新增 `doubao_speech.postJSON`、`minimax_speech.postJSON`。
- 结果：确认。`openai_compatible` 使用 `context.WithTimeout`，两个 speech adapter 只透传原始 context。

### 假设 3：contract tests 已覆盖慢响应
- 验证方法：检查 `doubao_speech` / `minimax_speech` mockserver 与 contract tests。
- 结果：确认缺口。Doubao mockserver 已有 `SleepMs` 字段但未使用；MiniMax mockserver 没有延迟字段；contract tests 只覆盖 4xx / 5xx，没有覆盖慢响应 deadline。

## 3 根因分析

- 根本原因：新增 speech adapter 复制了 JSON request helper 结构，但漏掉既有 provider adapter 中的 timeout wrapping 约定。
- 涉及代码路径：
  - `backend/internal/ai/aiclient/providers/doubao_speech/adapter.go`
  - `backend/internal/ai/aiclient/providers/minimax_speech/adapter.go`
  - `backend/internal/ai/aiclient/providers/doubao_speech/mockserver/mockserver.go`
  - `backend/internal/ai/aiclient/providers/minimax_speech/mockserver/mockserver.go`
- 之前没有发现的原因：Phase 3 checklist 要求 timeout coverage，但 adapter contract tests 只验证了上游 5xx 映射，没有通过 delayed mockserver 证明 `profile.TimeoutMs` 真正中断请求。

## 4 修复方案

- 在两个 speech adapter 的 `postJSON` 中，当 `timeoutMs > 0` 时用 `context.WithTimeout` 包装请求 context。
- 当 HTTP call 因 context deadline 失败时，统一映射为 retryable `AI_PROVIDER_TIMEOUT`。
- 扩展 Doubao mockserver 使用已有 `SleepMs` 字段，并给 MiniMax mockserver 增加 `SleepMs` 字段。
- 增加 contract tests：
  - `TestSynthesize_RespectsProfileTimeout` for Doubao TTS
  - `TestTranscribe_RespectsProfileTimeout` for Doubao STT
  - `TestSynthesize_RespectsProfileTimeout` for MiniMax TTS

## 5 验证

- `go test ./backend/internal/ai/aiclient/providers/doubao_speech -run 'Test(Synthesize|Transcribe)_RespectsProfileTimeout' -count=1`
- `go test ./backend/internal/ai/aiclient/providers/minimax_speech -run TestSynthesize_RespectsProfileTimeout -count=1`
- `go test ./backend/internal/ai/aiclient/providers/doubao_speech ./backend/internal/ai/aiclient/providers/minimax_speech -count=1`
- `go test ./backend/internal/ai/aiclient/... -count=1`
- `git diff --check`

## 6 经验教训

- Provider adapter contract tests must include delayed upstream responses when a profile timeout parameter exists.
- Mockserver behavior structs should not expose unused delay fields; unused test-only controls create false confidence that timeout behavior is covered.
