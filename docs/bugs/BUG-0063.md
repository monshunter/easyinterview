# BUG-0063: frontend report dashboard L2 review exposed route and parity drift

> 模块: review | 严重度: high | 状态: resolved
> 发现日期: 2026-05-16 | 关联 Commit: `fix(frontend-report): remediate report dashboard L2 findings (BUG-0063)`
> 关联日志: `-`

## 1 现象

- `report?sessionId=S` 缺 `reportId` 时没有在 `ReportScreen` 边界提前拦截，可能落入 dashboard 读路径后表现为泛化加载/错误态。
- Playwright pixel parity 使用 `#route=generating/report` 静态 hash route，但 `main.tsx` 未解析 hash，导致测试可能落到 Home 而不是真实目标路由。
- `useReportGenerationPoll` 在 visibility / focus 恢复时可能立即重发请求，而不是沿用暂停前已调度的 retry。
- report/generating 实现仍使用 prototype short CSS token；移动端真实 report 路由存在横向 overflow。

## 2 诊断过程

### 假设 1：报告缺参边界只覆盖 sessionId，未覆盖 reportId

- 验证方法：读取 `ReportScreen` 分发逻辑和缺参测试。
- 结果：确认。缺 `reportId` 的 report route 没有专属 early return，也缺少 i18n/test 覆盖。

### 假设 2：pixel parity route 未真正进入目标页面

- 验证方法：检查 `frontend/tests/pixel-parity/*.spec.ts` 的 URL 形态和 `frontend/src/main.tsx` bootstrap 路径。
- 结果：确认。测试 URL 使用 hash route，但 bootstrap 只读取 `window.__EASYINTERVIEW_INITIAL_ROUTE__`，没有解析 `window.location.hash`。

### 假设 3：移动端 overflow 是真实 report 页面布局问题

- 验证方法：修复 hash bootstrap 后重跑 `test:pixel-parity`，观察 mobile 390px viewport 的 `scrollWidth` 断言。
- 结果：确认。问题来自固定 grid track、长 ID/text 未允许换行，以及部分容器缺少 `minWidth: 0`。

## 3 根因分析

- route bootstrap gate 只断言了 URL 形态，没有断言页面实际 route identity，因此静态 hash preview 没有覆盖真实 report/generating DOM。
- report route 缺参边界只围绕 `sessionId` 建模，遗漏了 `reportId` 是 dashboard read path 的必要参数。
- poller 暂停/恢复逻辑没有保存待执行 attempt 与 delay，恢复时重新进入 polling effect 导致重复请求风险。
- legacy-negative lint 只覆盖旧模块/旧 route 字面量，没有覆盖 prototype short CSS token provenance。
- pixel parity 在真实路由启动前未能暴露移动端横向 overflow。

涉及代码路径：

- `frontend/src/app/bootstrapRoute.ts`
- `frontend/src/main.tsx`
- `frontend/src/app/screens/report/ReportScreen.tsx`
- `frontend/src/app/screens/report/components/ReportDashboard.tsx`
- `frontend/src/app/screens/generating/hooks/useReportGenerationPoll.ts`
- `scripts/lint/frontend_report_dashboard_legacy.py`

## 4 修复方案

- 新增 `parseInitialRouteHash` 并在 `main.tsx` 中作为 `__EASYINTERVIEW_INITIAL_ROUTE__` 的 fallback，使静态 hash route 能真实启动目标页面。
- 在 `ReportScreen` 中新增缺 `reportId` 的 `ReportMissingSessionState kind="missingReport"` 分支，保证不发 `getFeedbackReport`。
- 在 `useReportGenerationPoll` 中引入 `resumePlanRef` 和 `scheduleAttempt`，恢复时复用已计划 attempt，不立即重复请求。
- 将 report/generating 的 prototype short token 替换为正式 `--ei-color-*` / `--ei-font-*` token，并扩展 legacy lint + pytest。
- 修复 report dashboard / tab / context strip 的 responsive grid、`minWidth: 0` 和 `overflowWrap`，消除 390px mobile overflow。

## 6 经验教训

- UI pixel parity 必须断言真实 route identity；仅靠 hash URL 或截图非空不足以证明目标页面被覆盖。
- 用户可见 dashboard 的 mobile gate 应固定包含 `documentElement.scrollWidth <= innerWidth`。
- UI token provenance 需要和 legacy-negative literal 一起进入 scoped lint。
