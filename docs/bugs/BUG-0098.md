# BUG-0098: prompt output schema review exposed jd_match posted drift and lint traceback

> 模块: eval | 严重度: medium | 状态: resolved
> 发现日期: 2026-05-24 | 关联 Commit: `fix(prompt-rubric): close output schema review findings (BUG-0098)`
> 关联日志: [2026-05-24](../work-journal/2026-05-24.md)

## 1 现象

- `plan-code-review` follow-up 审查 `prompt-rubric-registry/002-output-schema-contract` 时发现 `jd_match.recommendation` schema 将 `posted` 标为 required。
- 生产 `compactJDMatchJobsPool` 只向模型输入 `jobMatchId`、`title`、`company`、`level`、`location`、`score`、`highlights`，没有 freshness / posted 字段；generator struct 也把 `Posted` 定义为 optional。
- 另一个审查点发现 `prompt_lint.py` 对缺少 `description` 的 invalid schema 会先进入 renderer 并抛 `KeyError` traceback，而不是输出 lint diagnostic。

## 2 诊断过程

### 假设 1：`posted` 是后端真实依赖字段
- 验证方法：读取 `config/prompts/jd_match.recommendation/v0.1.0.schema.json`、`backend/internal/jdmatch/generators/recommendation.go`、`backend/cmd/api/jdmatch_runtime.go`。
- 结果：排除。schema required 包含 `posted`，但生产 jobs pool 不提供该值，`llmRecommendation.Posted` 与 persistence `PostedLabel` 都是 optional。

### 假设 2：lint traceback 只是单测直接调用 renderer 的假象
- 验证方法：补充 integration-style pytest fixture，body 使用有效 contract，但 schema 文件删除 root `description` 后运行 `prompt_lint.py`。
- 结果：确认。当前 lint 在记录 subset error 后仍调用 `render_output_contract(schema)`，触发 `KeyError: 'description'`。

## 3 根因分析

- `jd_match.recommendation` 的 schema required 集合没有反查生产 prompt input 与 generator optional 字段，导致 output schema 比真实 runtime contract 更严格。
- `prompt_lint.py` 没有在 schema subset / contract validation 已失败时短路 renderer；invalid schema 同时成为 renderer 输入，错误表现从可读 lint diagnostic 退化为 traceback。
- 之前 gate 主要证明 schema / prompt / renderer 自洽，没有覆盖“required 字段必须由生产输入或 consumer contract 支撑”和“invalid schema 也必须稳定报告”的负向路径。

## 4 修复方案

- 将 `jd_match.recommendation` 的 `posted` 从 schema required 与 `FEATURE_CONTRACTS` required paths 中移除，保留为 optional 字段。
- 重新渲染 multi/en prompt contract block，刷新对应 YAML `template_hash`，同步 `000010` seed migration 中的 body/hash。
- 在 `prompt_lint.py` 中对 schema subset / contract errors 做短路，保留诊断并跳过 renderer。
- 新增 pytest 回归：`posted` 不再是 required；缺 `description` schema 输出 `missing non-empty description` 且不含 `Traceback`。

## 6 经验教训

- AI output schema 的 required 字段不能只从 prompt/struct 表面推导；还必须反查生产 caller 输入、consumer optional 字段和 persistence optionality。
- Lint 工具的 invalid-input path 也需要 integration-style regression，保证错误以稳定 diagnostic 形式返回，而不是 Python traceback。
