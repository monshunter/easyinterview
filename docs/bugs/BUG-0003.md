# BUG-0003: local quality gates skipped real backend and frontend execution

> 模块: test | 严重度: medium | 状态: resolved
> 发现日期: 2026-04-30 | 关联 Commit: `fix(ci-pipeline): remediate local quality gates`
> 关联日志: [2026-04-30](../work-journal/2026-04-30.md)

## 1 现象

- `/plan-code-review ci-pipeline-baseline/plans/001-local-quality-gates --fix` 发现 `make test` 和 `make build` 只输出 backend/frontend child subspec TODO，占位后 exit 0。
- `make -n lint` 显示 backend / frontend lint 仍依赖不存在的 `backend/Makefile` / `frontend/Makefile` fallback，而不是执行 plan 要求的 backend `golangci-lint run ./...` 和 frontend `pnpm lint`。
- 直接执行 `cd backend && golangci-lint run ./...` 失败，说明根 `make lint` 的通过证据没有覆盖真实 Go lint。

## 2 诊断过程

### 假设 1：根 Makefile 聚合层没有接入真实 backend/frontend 命令

- 验证方法：运行 `make test`、`make build`、`make -n lint`。
- 结果：确认。`make test` / `make build` 只打印 TODO；`make -n lint` 也显示会进入 `recurse_target` fallback。

### 假设 2：Go lint 失败来自 golangci-lint 配置未按 v2 schema 生效

- 验证方法：运行 `cd backend && golangci-lint run ./...`，再对比 v2 `linters.settings.revive.rules` 写法。
- 结果：确认。旧 `linters-settings` 未约束 revive 只跑 `var-naming`，导致默认 revive 规则扩大；迁移到 v2 配置后剩余 var-naming 命中点可通过命名修复。

### 假设 3：plan checklist 的证据命令本身存在可执行漂移

- 验证方法：运行 owner 标签 grep 与 secret 前缀 grep。
- 结果：确认。`make help` 缺少 checklist 要求的部分 owner 标签；`Makefile` help 文案包含 `AI_GATEWAY_`，导致 secret 红线 grep 误命中。

## 3 根因分析

- 根因是 A5 聚合层沿用 A1 早期 `recurse_target` 占位模式，但 checklist 已升级为真实 backend/frontend gate；计划完成时没有用 `make -n` 或实际 `make test/build` 核对命令是否进入真实执行路径。
- Go lint 失败来自 `backend/.golangci.yml` 使用了旧配置键，golangci-lint v2 没有按预期只启用 revive `var-naming` 规则。
- 文档验收命令没有在收口时逐条复跑，导致 `make help` owner 标签与 secret grep 证据漂移。

## 4 修复方案

- 将根 `Makefile` 的 `lint` / `test` / `build` 改为直接执行 backend/frontend 真实命令；仅保留 F1 observability 与 D1 frontend package script 中明确归属的 placeholder。
- 将 `backend/.golangci.yml` 迁移到 golangci-lint v2 的 `linters.settings.revive.rules` 写法，并修复剩余 var-naming 命中点。
- 补齐 `make help` 中 `lint-conventions (B1)` / `lint-config (A4)` / `lint-observability (F1)` / `codegen-conventions (B1)` / `codegen-openapi (B2)` 标签。
- 移除 `Makefile` help 文案里的 secret-shaped 字面量，避免验收 grep 误判。
- 原地修订 `ci-pipeline-baseline/001-local-quality-gates` plan/checklist 到 v1.3，并重新记录验证证据。

## 5 验证

- `cd backend && go test ./cmd/codegen/conventions ./cmd/codegen/openapi ./internal/platform/config ./internal/ai/aiclient/providers/openai_compatible -count=1`
- `cd backend && golangci-lint run ./...`
- `make help` owner 标签 grep 全部命中。
- `grep -nE '(AI_GATEWAY_|DB_|REDIS_|POSTHOG_)' Makefile scripts/lint/*.py` 不命中。
- `make lint`
- `make test`
- `make build`
- `make docs-check`
- `make codegen-check`
- `git diff --check`

## 6 经验教训

- 聚合型 gate 的完成证据必须同时包含 `make -n` 路径核对和实际命令执行，避免 fallback placeholder 与真实 gate 混淆。
- lint 工具主版本升级后，应以实际 `golangci-lint run` 作为配置生效证据，而不是只检查配置文件存在。
