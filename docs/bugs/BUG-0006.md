# BUG-0006: openai-compatible adapter assumed provider-specific model naming

> 模块: eval | 严重度: medium | 状态: resolved
> 发现日期: 2026-05-05 | 关联 Commit: `fix(historical-spec): deep reconcile existing plans`
> 关联日志: [2026-05-05](../work-journal/2026-05-05.md)

## 1 现象

- L2 review 发现 `backend/internal/ai/aiclient/providers/openai_compatible` 的 mock contract tests 使用具体 vendor-style model id（如 GPT / Claude 命名）作为 fixture。
- `modelFamily()` 使用“截断最后一个 hyphen 后缀”的规则推导 `AICallMeta.ModelFamily`，会把普通 hyphen model id（例如 embedding small 类名称）误截断。
- 影响范围是 AI call metadata、observability label、fallback log 和 future cost/quality aggregation；不会直接泄漏 secret 或调用真实 provider。

## 2 诊断过程

### 假设 1：只是测试 fixture 文案旧

- 验证方法：把 OpenAI-compatible adapter 测试改成 provider-neutral fixture，并增加 `ModelFamily` 断言。
- 结果：排除。Red phase 失败，说明生产代码确实依赖了旧命名结构。

### 假设 2：`modelFamily()` 存在旧模型命名假设

- 验证方法：focused test 覆盖 `chat-primary-2026-05-05` 和 `embed-small` 两种模型 id。
- 结果：确认。旧逻辑输出 `chat-primary-2026-05` 和 `embed`，不符合 provider-neutral model id 语义。

## 3 根因分析

- 根本原因：`modelFamily()` 将所有 hyphen 后缀都视为可剥离版本段，实际只应处理明确的日期版本后缀。
- 涉及代码路径：`backend/internal/ai/aiclient/providers/openai_compatible/adapter.go`。
- 之前没有发现的原因：adapter contract tests 只断言 `ModelID`，没有断言 `ModelFamily`，且 fixture 使用具体 vendor-style model id，掩盖了通用 OpenAI-compatible endpoint 的命名差异。

## 4 修复方案

- 将 adapter contract tests 改为 provider-neutral model id，并新增 `ModelFamily` 断言。
- 将 fallback / observability tests 的 model fixture 改为 neutral values，避免继续暗示固定 provider 或模型。
- 修改 `modelFamily()`：只有末尾形如 `YYYY-MM-DD` 的日期版本后缀才会被剥离；普通 hyphen model id 保持原值。

修改文件：

- `backend/internal/ai/aiclient/providers/openai_compatible/adapter.go`
- `backend/internal/ai/aiclient/providers/openai_compatible/contract_test.go`
- `backend/internal/ai/aiclient/observability/decorator_test.go`

- Global：本轮 historical-spec final reconcile 已通过 `make docs-check`、`make codegen-check`、DB-backed `make migrate-check`、focused tests、`make lint`、`make test` 与 `make build`；完整证据见 [收口账本](../reports/2026-05-05-historical-spec-deep-reimplementation-ledger.md)。

## 6 经验教训

- Provider-neutral adapter 的测试 fixture 不应使用具体 vendor/model branding，除非测试目标就是兼容该 wire shape。
- 对 derived metadata 字段必须有直接断言；只断言原始字段会漏掉 observability / cost 聚合语义错误。
