# BUG-0136: repo pruning cleanup review exposed lint and frontend guard drift

> 模块: test | 严重度: medium | 状态: resolved
> 发现日期: 2026-07-07 | 关联 Commit: `fix(review): close repo pruning cleanup findings (BUG-0136)`
> 关联日志: [2026-07-07](../work-journal/2026-07-07.md)

## 1 现象

- 对 `fix/repo-pruning-cleanup-0706` 做 review remediation 时，发现 `core_loop_pruning_surface` 会把生产路径中的 non-current 残留误归入 negative bucket，只要行文本包含 `delete` 等负向上下文词。
- `ResumePickerModal` 对 `boundResumeId="resume-unbound"` 这类 synthetic / stale id 仍允许确认，可能把不存在于已加载 active resume 列表的 id 传给上层。
- 前端 focused Vitest 被 pnpm 11 拒绝执行，因为 `pnpm-workspace.yaml` 的 `allowBuilds` 仍是占位值，依赖 build approval gate 无法通过。

## 2 诊断过程

### 假设 1：Pruning lint 只在测试或脚本路径放行 negative wording

- 验证方法：新增 `test_negative_context_words_do_not_allow_production_runtime_residuals`，在 `backend/internal/api/debriefs/handler.go` 中写入带 `delete` 的 debrief 残留。
- 结果：确认失败。旧实现把该生产路径残留归入 `negative_tests`，导致 `real_residuals` 为空。

### 假设 2：Resume picker 只检查 selected id 是否 truthy

- 验证方法：新增 focused Vitest，传入 `boundResumeId="resume-unbound"`，等待 active resume option 渲染后检查 confirm 状态。
- 结果：确认问题。旧实现用 `disabled={!selectedResumeId}` 和 `if (selectedResumeId)` 作为唯一 guard，无法区分 active resume id 和 synthetic / stale id。

### 假设 3：pnpm test 失败来自依赖安装策略而非测试代码

- 验证方法：运行 `CI=true pnpm --filter @easyinterview/frontend test src/app/screens/workspace/modals/ResumePickerModal.test.tsx`。
- 结果：确认 pnpm 11 报 `[ERR_PNPM_IGNORED_BUILDS]`，需要把 workspace 级 `allowBuilds` 占位值替换为布尔配置。

## 3 根因分析

- `scripts/lint/core_loop_pruning_surface.py` 的 `classify` 在路径 allowlist 之外仍按行文本命中 `NEGATIVE_CONTEXT_RE` 放行，使生产代码可以通过写入负向语义逃过 real residual gate。
- `frontend/src/app/screens/workspace/modals/ResumePickerModal.tsx` 没有把选中 id 约束到当前加载出的 active resume 集合，truthy id 就能进入 confirm path。
- `pnpm-workspace.yaml` 存在由 pnpm approval flow 写入的占位 `allowBuilds` 值，pnpm 11 不接受占位文本，导致前端测试执行前失败。
- 之前没有发现，是因为既有 pruning test 覆盖了 allowed negative contexts，但没有覆盖“生产路径 + negative wording”的反例；Resume picker 测试也缺少 stale bound id 场景；pnpm config 只在安装/执行前端 gate 时暴露。

## 4 修复方案

- 删除 `classify` 中基于 `NEGATIVE_CONTEXT_RE` 的全局内容放行，只保留 migration、normalization、test、lint/scenario 等显式路径放行，并补充生产路径反例测试。
- `ResumePickerModal` 增加 `activeResumeIds`，在加载结束后清理 invalid selected id，并让 selected state、confirm disabled、click guard、cursor 与 opacity 都依赖 `selectedResumeIsActive`。
- 将 `pnpm-workspace.yaml` 的 `allowBuilds` 项改为 pnpm 11 可接受的布尔值，恢复 frontend test gate。
- 对 branch 中 gofmt 漂移文件执行 `gofmt`，并清理 stricter pruning gate 曝光的 active README / comment 残留。

## 6 经验教训

- Drift lints 应优先使用路径/文件类型等稳定上下文放行；面向全仓扫描的 production path 不应被单行负向语义豁免。
- UI selection guard 不应只判断 id truthiness；凡 id 来自 URL、缓存、mock 或 bound props，都要回查当前 loaded entity set。
- pnpm approval 配置需要在 install/test gate 中保留可执行形态，不能留下交互式工具生成的占位值。
