# BUG-0077: OpenAPI diff baseline lag hid the current additive contract change

> 模块: schema | 严重度: medium | 状态: resolved
> 发现日期: 2026-05-18 | 关联 Commit: `fix(resume-workshop): close real backend gaps (BUG-0076, BUG-0077)`
> 关联日志: [2026-05-18](../work-journal/2026-05-18.md)

## 1 现象

- `make openapi-diff` failed after `RequestResumeTailorRequest.resumeVersionId` was added.
- The wrapper correctly classified `components.schemas.RequestResumeTailorRequest.resumeVersionId` as `additive`.
- The non-zero exit came from older accepted Debrief/schema drift against `openapi/baseline/openapi-v1.0.0.yaml`, plus `openapi/diff-config.yaml` still declaring the pre-expansion 34-operation inventory.

## 2 诊断过程

### 假设 1: The new resume field was breaking

- 验证方法：Ran `make openapi-diff` and inspected the JSON findings.
- 结果：排除。The new field was reported as `field-added` with severity `additive`.

### 假设 2: The accepted baseline was stale relative to the current v1.0.0 freeze

- 验证方法：Compared `openapi/baseline/openapi-v1.0.0.yaml`, `main:openapi/openapi.yaml`, and current `openapi/openapi.yaml`; checked `openapi/diff-config.yaml` inventory.
- 结果：确认。`main:openapi/openapi.yaml` already had 59 operations and the accepted Debrief schema shape; the baseline still represented the older accepted contract and config still expected 34 operations.

## 3 根因分析

- Accepted OpenAPI additive expansions had updated live OpenAPI, generated artifacts, fixtures, and history, but the executable `openapi-v1.0.0` baseline and diff inventory were not re-frozen to the same accepted contract.
- Because the wrapper compares the working OpenAPI against the latest baseline, historical accepted drift was mixed with the current branch delta and caused a false failing gate.
- The baseline README still described the old 34-operation snapshot, so the governance text did not match the active spec's 59-operation v1.0.0 freeze.

## 4 修复方案

- Re-froze `openapi/baseline/openapi-v1.0.0.yaml` from `main:openapi/openapi.yaml`, preserving the current branch's `resumeVersionId` as the only remaining additive diff.
- Updated `openapi/diff-config.yaml` to expect 59 operations.
- Updated `openapi/baseline/README.md`, `openapi-v1-contract` spec/history, and `docs/spec/INDEX.md` to document the pre-launch accepted-baseline refresh and the optional targeted-version field.
- Kept the baseline out of the new `resumeVersionId` field so `make openapi-diff` still proves the current change is additive.

## 5 验证

- `make openapi-diff` now exits 0 with `breaking=0`, `additive=1`, and the only finding `components.schemas.RequestResumeTailorRequest.resumeVersionId`.
- `make lint-openapi`
- `make validate-fixtures`
- `python3 -m pytest scripts/lint/openapi_diff_test.py`
- `make docs-check`
- `git diff --check`

## 6 经验教训

- OpenAPI additive work is not closed until live OpenAPI, generated artifacts, fixtures, history, baseline, and diff inventory describe the same accepted contract boundary.
- For follow-up schema fixes, refresh the accepted baseline to the parent branch contract rather than the working tree when the current branch still needs to prove an additive delta.
