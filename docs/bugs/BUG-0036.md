# BUG-0036: Vite dev preview hit frontend port instead of fixture-backed API

> 模块: frontend | 严重度: high | 状态: resolved
> 发现日期: 2026-05-10 | 关联 Commit: `fix(frontend-dev): default vite preview to fixture-backed API`
> 关联日志: [2026-05-10](../work-journal/2026-05-10.md)

## 1 现象

- Running the frontend dev server without a backend produced many real API errors.
- Browser requests for generated-client calls went to `http://localhost:5173/api/v1/...`, which is the Vite frontend port, not the backend port.
- As a result, implemented pages often could not be inspected locally because bootstrap data failed before the user reached the page state.

## 2 诊断过程

### 假设 1：项目没有 MSW，所以所有 dev 页面必然打真实 backend

- 验证方法：读取 frontend API mock runtime and test utilities.
- 结果：排除一半。The repo already had a fixture-backed transport, but it was only used by tests or injected clients, not by the actual Vite app bootstrap.

### 假设 2：Vite has a proxy from 5173 to backend

- 验证方法：读取 `frontend/vite.config.ts`.
- 结果：排除。There was no `/api` proxy, so a relative API base URL stayed on the Vite origin.

### 假设 3：Generated client default base URL caused 5173 API traffic

- 验证方法：read `EasyInterviewClient` default base URL and `frontend/src/main.tsx` bootstrap.
- 结果：确认。The generated client defaulted to relative `/api/v1`, and `main.tsx` used `new EasyInterviewClient()` directly. In Vite dev, the browser resolves relative URLs against `http://localhost:5173`.

## 3 根因分析

- The mock-contract plan delivered fixture-backed transport but stopped at test/injected-client usage.
- The real application bootstrap path was not covered by a gate, so `main.tsx` still constructed the generated client directly.
- Vite dev had no proxy and no explicit real-backend default, so `/api/v1` implicitly targeted the frontend port.
- Previous Playwright/component tests often injected clients or route mocks, which missed the actual app bootstrap failure.

## 4 修复方案

- Add `createAppClient` to resolve API mode from Vite env.
- Default Vite dev to fixture-backed generated client data.
- Keep explicit real-backend opt-out with `VITE_EI_API_MODE=real`; in dev real mode, default to `http://localhost:8080/api/v1` unless `VITE_EI_API_BASE_URL` is provided.
- Add `createDevMockClient` that imports all OpenAPI fixture JSON files and asserts coverage against generated `ALL_OPERATION_IDS`.
- Change `main.tsx` to use `createAppClient()` instead of `new EasyInterviewClient()`.
- Document the dev modes in `frontend/README.md` and `frontend/src/api/README.md`.

## 6 经验教训

- Frontend dev preview must test the actual bootstrap path (`main.tsx`), not only component tests with injected clients.
- Relative generated-client base URLs are unsafe in Vite dev without an explicit proxy or mock default because they resolve to the frontend origin.
- Mock-contract gates should include a dev-preview smoke that asserts no unintended `/api/v1` requests are emitted when the backend is intentionally absent.
