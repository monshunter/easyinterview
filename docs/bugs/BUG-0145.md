# BUG-0145: target import parse prompt omitted JD identity fields

> 模块: eval | 严重度: medium | 状态: resolved
> 发现日期: 2026-07-08 | 关联 Commit: `fix(workspace): close plan list UX and context regressions (BUG-0143, BUG-0144, BUG-0145)`
> 关联日志: [2026-07-09](../work-journal/2026-07-09.md)

## 1 现象

- JD 简历解析有时丢失公司、岗位等信息。
- 解析结果虽然能生成 requirements / summary，但 TargetJob 的 `title` / `companyName` 可能仍停留在导入 hint 或空值。
- 影响规划详情首屏识别和后续面试上下文质量。

## 2 诊断过程

### 假设 1：LLM prompt/schema 没要求输出岗位名和公司名

- 验证方法：读取 `config/prompts/target.import.parse/v0.1.0.md` 与 `.schema.json`。
- 结果：确认。输出 contract 只要求 `coreThemes`、`interviewHypotheses`、`strengths`、`gaps`、`riskSignals`、`requirements`。

### 假设 2：后端 parser 即使收到字段也不会持久化

- 验证方法：读取 `parseAIResponse`、`CompleteParseSuccessInput` 和 `applyParseResultTx`。
- 结果：确认。parser struct 没有 `title/companyName`，parse success 事务只更新 `analysis_status`、`summary`、`fit_summary` 和 requirements。

## 3 根因分析

- F3 `target.import.parse` baseline prompt/schema 与 TargetJob identity persistence 脱节。
- 后端 parse executor 未校验 JD identity 字段，store success path 未把解析出的 identity 写回 `target_jobs`。
- 之前的 prompt/schema gate 只覆盖 6 个业务分析 key，没有把 title/company 作为必填 parse contract。

## 4 修复方案

- `target.import.parse` schema/prompt required 增加 `title` / `companyName`，并在 prompt 正文强调从 JD text、URL、heading 或 metadata-like lines 中提取。
- `parseAIResponse` 增加 `Title` / `CompanyName`，parse success 前校验非空。
- `CompleteParseSuccess` / `applyParseResultTx` 在同一事务中写回 `target_jobs.title` / `company_name`。
- 同步 prompt hash、seed migration prompt body、resolved eval prompts 与 prompt linter required paths。

## 5 验证

- `cd backend && go test ./cmd/api -run 'TestBuildTargetJobRuntimeWiresDrainerAndAIClient|TestE2EP0010HTTPTextImportParseReady' -count=1`
- `cd backend && go test ./internal/targetjob -count=1`

## 6 经验教训

- Prompt output schema must include fields that downstream persistence treats as canonical identity, not only analysis summaries.
- Schema-rendered contract tests should fail when consumer structs gain or lose required fields.
