# BUG-0001: OpenAPI breaking-change gate missed composition diffs

> 模块: schema | 严重度: medium | 状态: resolved
> 发现日期: 2026-04-29 | 关联 Commit: `fix(openapi): tighten breaking-change gate composition diff`
> 关联日志: `-`

## 1 现象

- `/plan-code-review openapi-v1-contract/plans/003-breaking-change-gat --fix` 发现 `scripts/lint/openapi_diff.py` 对 OpenAPI composition schema 的 breaking change 检测不完整。
- 当前 contract 已使用 `oneOf` 表示 nullable 字段、使用 `allOf` 表示 paginated envelope；把 `MistakeEntry.sourceSessionId` 从 `string|null` 改成 `integer|null`，或把 `PaginatedTargetJob.items` 从 `TargetJob` 改成 `PracticePlan`，旧 wrapper 返回 zero findings。
- privacy export `501→202` 白名单 gate 默认按 `HEAD` 比较 `history.md` 行数；当变更与 history 行已经提交到 feature branch 后，旧 wrapper 会误报 `history-not-incremented`。

## 2 诊断过程

### 假设 1：schema diff 没有递归 composition 分支

- 验证方法：构造 `oneOf` branch type 变更与 `allOf` branch `$ref` 变更，直接调用 `diff_documents()`。
- 结果：确认。旧实现只比较 direct `$ref`、direct `type`、direct object `properties`、array `items` 和 direct `enum`，没有进入 `oneOf` / `allOf` / `anyOf`。

### 假设 2：history gate 的比较基准不符合 PR 语义

- 验证方法：在临时 git repo 中从 `main` 切 feature branch，提交 privacy export `501→202` 与新增 history 行后运行 wrapper。
- 结果：确认。旧默认 `--history-ref HEAD` 使 `rowsAtRef == rowsAtWorkingTree`，因此已提交的合法 history 增量仍被误判。

## 3 根因分析

- `scripts/lint/openapi_diff.py` 的 schema classifier 覆盖了常见 direct schema 形状，但遗漏了 OpenAPI 3.1 composition keywords。
- history gate 把“工作树 vs 当前 HEAD”当成默认比较模型，只适合未提交 self-check，不适合 plan §2.2 要求的“同 PR 增量”。
- 旧测试只覆盖 direct field/type/required/enum/query/status 场景，没有覆盖当前 contract 中实际存在的 `oneOf` / `allOf` schema，也没有覆盖“变更已提交到 feature branch”的白名单正向路径。

## 4 修复方案

- `scripts/lint/openapi_diff.py` 增加 `oneOf` / `allOf` / `anyOf` 分支递归比较；composition branch 内的 type / `$ref` / required / enum 变更继续复用现有 classifier，branch 新增或删除按 breaking 保守处理。
- history gate 默认解析 `openapi/diff-config.yaml` 的 `tooling.historyDiffBase`（当前 `dev`），使用该 ref 与 `HEAD` 的 merge-base 作为比较基准；找不到时回退 `main` / `master` 候选，最后才回退 `HEAD`。
- `Makefile` 的 `openapi-diff` target 增加 `HISTORY_REF=<git-ref>` 显式覆盖入口。
- `scripts/lint/openapi_diff_test.py` 增加 composition diff 与 committed feature-branch history increment 的回归用例。

## 5 验证

- `python3 -m unittest scripts.lint.openapi_diff_test -v`：24 个用例通过。
- `make codegen-check && make validate-fixtures && make openapi-diff`：通过；`openapi-diff` summary 为 `breaking=0, additive=0, informational=0`。
- `python3 .agent-skills/sync-doc-index/scripts/sync-doc-index.py --check`：Zero drift detected。
- `git diff --check`：通过。

## 6 经验教训

- 自研 contract diff gate 必须用真实 contract 里已使用的 schema 形态做回归样本，不能只覆盖 direct object schema。
- “同 PR 增量”类检查不应默认比较 `HEAD`，应默认比较 base branch merge-base，并允许显式覆盖以支持本地复现。
