# BUG-0102: Promptfoo eval output was written under config

> 模块: eval | 严重度: medium | 状态: resolved
> 发现日期: 2026-05-24 | 关联 Commit: `fix(prompt-rubric): keep promptfoo eval output in test output (BUG-0102)`
> 关联日志: [2026-05-24](../work-journal/2026-05-24.md)

## 1 现象

- 用户指出 `config/evals/.generated/promptfoo` 是 Promptfoo 测试运行时产物目录，应统一放在 `.test-output/`，不应放在 `config/`。
- 当前分支在合并 `main` 前已有 untracked `config/evals/.generated/`，包含 Promptfoo SQLite DB、日志、`promptfoo.yaml` 和生成的 `promptfoo_tests.yaml`。
- `main` 中 `make eval-offline` 会继续把 `prompts-tests` 输出、`PROMPTFOO_CONFIG_DIR` 和 `PROMPTFOO_LOG_DIR` 指向 `config/evals/.generated/`，且 `.gitignore` 隐藏该目录，容易让后续运行继续污染配置真理源目录。

## 2 诊断过程

### 假设 1：只是本地临时文件残留

- 验证方法：检查 `git status`、`find config/evals`、`git ls-tree main config/evals` 和 `Makefile` 的 eval target。
- 结果：排除。`main` 已新增真实 `config/evals/*` 源资产，同时 `Makefile` 明确把 Promptfoo 运行态写入 `config/evals/.generated/`。

### 假设 2：Promptfoo 配置必须相对 `config/evals/` 读取测试文件

- 验证方法：将 generated tests 与渲染后的 Promptfoo config 输出改为同一个 `$(EVAL_OUTPUT_DIR)`，让源 `config/evals/promptfooconfig.yaml` 仅作为模板，并运行 `make eval-offline`。
- 结果：排除。Promptfoo 可以正常从 `.test-output/evals/promptfoo_tests.yaml` 读取测试文件，52 个 case 全部通过。

## 3 根因分析

- 根本原因：`make eval-offline` 的运行态隔离修复只避免写入用户 home，但错误地把 repo 内隔离目录选在 `config/evals/.generated/`，把配置/评估源资产目录和运行时证据目录混在一起。
- 涉及路径：
  - `Makefile` 的 `eval-offline` target。
  - `config/evals/promptfooconfig.yaml` 的 generated tests 路径。
  - `config/evals/README.md` 和 `docs/spec/prompt-rubric-registry/plans/004-real-model-profile-and-evals/*` 的运行产物约束。
- 之前没有发现的原因：004 plan 的 Phase 4 gate 证明了 Promptfoo 不写用户 home，但没有要求运行产物必须归入仓库统一 `.test-output/` 目录，也没有拒绝 `config/evals/.generated/` 被 gitignore。

## 4 修复方案

- 将 `make eval-offline` 的 generated tests、渲染后 Promptfoo config、Promptfoo config dir、SQLite DB 和日志统一改到 `$(EVAL_OUTPUT_DIR)`（默认 `.test-output/evals/`）。
- 将 `config/evals/promptfooconfig.yaml` 改为模板，运行时把 provider/assert/tests 三个 `file://` 路径渲染为同一次 `EVAL_OUTPUT_DIR` 下的真实文件。
- 从 `.gitignore` 移除 `config/evals/.generated/`，让该目录一旦回归就暴露为 untracked 污染。
- 更新 `config/evals/README.md` 与 004 plan/checklist v1.3，明确 `config/evals/` 只保存 eval 源资产，Promptfoo 运行产物必须在 `.test-output/evals/`。

## 5 验证

- `make eval-offline`：`evalkit drift-check: OK`、`evalkit run: OK`、Promptfoo `52 passed / 0 failed / 0 errors`。
- `make -n eval-offline EVAL_OUTPUT_DIR=/private/tmp/easyinterview-evals-review`：确认 generated tests、渲染后 config、Promptfoo state/logs 全部指向 override 目录。
- `find config/evals -maxdepth 3 -name '.generated' -o -path 'config/evals/.generated/*' -print`：无输出。
- `find .test-output/evals -maxdepth 3 -print`：确认 `promptfoo_tests.yaml`、渲染后的 `promptfooconfig.yaml`、Promptfoo DB 和日志位于 `.test-output/evals/`。
- `rg -n 'config/evals/\\.generated|\\.generated/promptfoo' Makefile config/evals/promptfooconfig.yaml .gitignore`：无输出。
- `git diff --check` 与 `git diff --cached --check` 均通过。

## 6 经验教训

- 运行态隔离不应只从“不要写用户 home”判断，还必须落到仓库统一证据目录 `.test-output/`。
- 对 source-of-truth 目录新增工具 runner 时，checklist 应同时锁定 source asset 路径和 runtime artifact 路径。
