# BUG-0080: tool-name branch prefixes bypassed branch naming governance

> 模块: platform | 严重度: medium | 状态: resolved
> 发现日期: 2026-05-21 | 关联 Commit: `fix(governance): reject tool-name branch prefixes (BUG-0080)`
> 关联日志: [2026-05-21](../work-journal/2026-05-21.md)

## 1 现象

- 用户发现当前分支为 `codex/backend-profile-and-jobs-recommendations-bootstrap`，但预期应为 `spec-design/backend-profile-and-jobs-recommendations-bootstrap`。
- 历史本地分支中还存在多个 `codex/*` 前缀分支，说明这不是一次性的命名笔误。
- 分支名把 Agent / CLI 工具身份写入了协作域，而不是表达 `fix` / `docs` / `design` / `spec-design` 等工作类型或业务域。

## 2 诊断过程

### 假设 1：`/implement` 自动分支脚本生成了 `codex/` 前缀

- 验证方法：读取 `.agent-skills/implement/SKILL.md` Step 4.5 与 `.agent-skills/implement/shared/scripts/detect_session_branch.py`。
- 结果：排除。`/implement` 的命名约定为 `{type}/{subspec}-{plan}-{MMDD}`，脚本只匹配 `feat|fix|opt|docs` 前缀，不会生成或接受 `codex/`。

### 假设 2：设计 / 入口类 skill 的分支门禁只强调先建 feature branch，没有禁止工具名前缀

- 验证方法：读取 `AGENTS.md` §7、`.agent-skills/design/SKILL.md`、`.agent-skills/change-intake/SKILL.md`、`.agent-skills/plan-review/SKILL.md`、`.agent-skills/plan-code-review/SKILL.md`。
- 结果：确认。根级策略原本只写 `feature branch`，`/design` 虽已有 `design/` / `spec-design/` 示例，但未显式禁止 `codex/`、`claude/` 等工具名前缀；部分 review/fix 入口也缺少命名前缀约束。

## 3 根因分析

- 根本原因：分支治理只规定了“不能在默认父分支上写文件”和部分示例前缀，没有把“分支第一段必须是语义域，不得是工具身份”作为强制门禁。
- 涉及路径：
  - `AGENTS.md` §7 Git 分支策略
  - `.agent-skills/design/SKILL.md` Step 2.5
  - `.agent-skills/change-intake/SKILL.md` Branch Guard
  - `.agent-skills/plan-review/SKILL.md` Fix mode
  - `.agent-skills/plan-code-review/SKILL.md` Fix mode
- 为什么之前没有发现：既有 contract test 覆盖了“分支门禁必须在写文件前执行”，但没有覆盖“禁止工具名前缀”和“错误分支未推送时先重命名”的语义。

## 4 修复方案

- 将当前本地分支从 `codex/backend-profile-and-jobs-recommendations-bootstrap` 重命名为 `spec-design/backend-profile-and-jobs-recommendations-bootstrap`，保留已有提交。
- 在 `AGENTS.md` §7 明确：允许 `feat/`、`fix/`、`docs/`、`opt/`、`design/`、`spec-design/` 等语义前缀，禁止 `codex/`、`claude/`、`gemini/`、`agent/` 等工具名前缀。
- 在 `/design`、`/change-intake`、`/plan-review --fix`、`/plan-code-review --fix` 的分支门禁中固化同一规则，并要求未推送的新工具名前缀分支先重命名。
- 增加 contract test 覆盖 `AGENTS.md`、`/design`、`/change-intake` 中的禁止工具名前缀规则。

## 5 验证

- `git status --short --branch` 显示当前分支已为 `spec-design/backend-profile-and-jobs-recommendations-bootstrap`。
- `python3 -m pytest .agent-skills/design/scripts/test_design_skill_contract.py .agent-skills/change-intake/scripts/test_change_intake_skill_contract.py .agent-skills/implement/scripts/test_execution_automation_docs_contract.py .agent-skills/implement/scripts/test_session_branch_helper.py`：30 passed。

## 6 经验教训

- 分支门禁不能只检查“是否在 feature branch”，还必须检查 feature branch 的语义前缀。
- 入口 skill 的示例前缀不够稳定；禁止项必须写成明确负向规则，并配 contract test。
