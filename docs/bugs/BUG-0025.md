# BUG-0025: analysis failed redline rejected documented provider secret code

> 模块: workspace | 严重度: medium | 状态: resolved
> 发现日期: 2026-05-08 | 关联 Commit: `feat(backend-targetjob): complete import parse bootstrap handoff`
> 关联日志: [2026-05-08](../work-journal/2026-05-08.md)

## 1 现象

- `E2E.P0.012` 覆盖 retryable provider-secret failure 时，`target.analysis.failed` payload 没有写入。
- 合法 B1 error code `AI_PROVIDER_SECRET_MISSING` 被 privacy redline 的 forbidden substring `provider_secret` 误判为敏感泄露。
- 影响范围是 TargetJob 解析失败链路：用户可重试的 B1 错误无法稳定进入 outbox evidence，削弱 failure-path 可观测性。

## 2 诊断过程

### 假设 1：parse executor 没有调用 analysis failed payload builder
- 验证方法：检查 `backend/internal/targetjob/parse_executor.go` 的 retryable / non-retryable 错误路径。
- 结果：排除。失败路径会构造 `TargetAnalysisFailedInput` 并调用 payload builder。

### 假设 2：error code 不在 B1 registry 中
- 验证方法：检查 generated shared error registry 与 `AI_PROVIDER_SECRET_MISSING` 定义。
- 结果：排除。该 error code 是合法 B1 documented code。

### 假设 3：payload redline 把合法 error code 当成 secret 文本
- 验证方法：新增 `TestBuildTargetAnalysisFailedPayload_AllowsDocumentedProviderSecretCode` 并运行 focused test。
- 结果：确认。旧实现先对完整 payload 做 forbidden substring 扫描，导致合法 error code 因包含 `provider_secret` 被拒绝。

## 3 根因分析

- 根本原因：privacy redline 没有区分“受控枚举值”和“自由文本 payload”，把 B1 documented error code 当成可能泄漏 provider secret 的自由文本。
- 涉及代码路径：
  - `backend/internal/targetjob/payload.go`
  - `backend/internal/targetjob/payload_test.go`
  - `backend/internal/targetjob/e2e_scenario_test.go`
- 之前没有发现的原因：既有 redline tests 覆盖 forbidden free text，但没有覆盖“合法 error registry value 命中 forbidden substring”的边界。

## 4 修复方案

- `BuildTargetAnalysisFailedPayload` 先要求 `errorCode` 存在于 `sharederrors.CodeRegistry`。
- Redline 扫描时把已验证的 error code 替换为占位符 `B1_DOCUMENTED_ERROR_CODE`，继续扫描 message、retryability、target id 等 payload 字段。
- 新增 tests：
  - `TestBuildTargetAnalysisFailedPayload_AllowsDocumentedProviderSecretCode`
  - `TestBuildTargetAnalysisFailedPayload_RejectsUndocumentedError`

## 6 经验教训

- Privacy redline 对 structured payload 应先校验 registry / enum，再扫描自由文本字段；否则合法枚举值可能因命名包含敏感词而被误杀。
- Error-code tests 应覆盖 documented code 与 redline token 的交集，不只覆盖非法自由文本。
