# BUG-0033: practice session L2 review exposed first-question and replay drift

> 模块: practice | 严重度: high | 状态: resolved
> 发现日期: 2026-05-10 | 关联 Commit: `fix(backend-practice): remediate session orchestration L2 findings`
> 关联日志: `-`

## 1 现象

- L2 code review 发现 `startPracticeSession` 首题解析仍只接受 `questionText` / `questionIntent`，而 F3 `practice.session.first_question` prompt 要求模型返回 `question` / `intent` / `focus_dimension` / `expected_signals` / `time_budget_seconds`。
- 同一路径把非 JSON AI 输出当作成功首题返回，削弱了 `AI_OUTPUT_INVALID` gate。
- `startPracticeSession` success replay 读取当前 `practice_sessions` / `practice_turns` 状态，而不是读取首次成功时写入 `idempotency_records.response_body` 的 response snapshot。
- `000003_practice_idempotency_baseline.down.sql` 仍会 drop baseline-owned `idempotency_records` 表和 index。

## 2 诊断过程

### 假设 1：首题解析只是 OpenAPI response 字段名与 domain 字段名不同

- 验证方法：读取 `backend/internal/practice/session_starter.go`、F3 prompt markdown 与现有 `session_starter` tests。
- 结果：排除。当前 prompt 明确要求 strict JSON keys `question` / `intent` 等；实现只解析 legacy/generated response keys，并对非 JSON 文本 fallback 为成功。

### 假设 2：success replay 已经由 `response_body` 覆盖

- 验证方法：读取 `backend/internal/store/practice/create_plan.go` 的 reserve/replay/commit 路径和 scenario in-memory store。
- 结果：排除。commit 会写 `response_body`，但 succeeded replay 分支调用 `selectSessionForUser`，scenario harness 也返回 mutable session record。

### 假设 3：000003 down migration 仍拥有 idempotency table

- 验证方法：对比 `migrations/000001_create_baseline.up.sql` 与 `migrations/000003_practice_idempotency_baseline.down.sql`。
- 结果：排除。pre-launch baseline rebase 后 `000001` 已创建 `idempotency_records`；`000003` down 不应删除 baseline-owned 表或 index。

## 3 根因分析

- F3 prompt schema 与 `session_starter` parser gate 没有同测，导致真实模型输出 shape 与 runtime parser 分叉。
- Idempotency replay 的测试只覆盖“无重复副作用”，没有断言 replay response 必须来自持久化 snapshot。
- Migration ownership rebase 后缺少 down-path zero-drop contract test，导致旧 down 文件保留了 pre-rebase table ownership。

## 4 修复方案

- `backend/internal/practice/session_starter.go`：`parseFirstQuestion` 改为必须解析 strict JSON object，兼容 `question` / `intent` 与 `questionText` / `questionIntent`，并拒绝空内容、非 JSON、缺少 question 的输出。
- `backend/internal/store/practice/create_plan.go`：succeeded replay 查询并解码 `idempotency_records.response_body`，返回首次成功 response snapshot，不再读取 mutable session state。
- `backend/cmd/api/practice_http_scenario_test.go`：scenario in-memory harness 改为从 stored response snapshot replay，并新增 replay drift 断言。
- `migrations/000003_practice_idempotency_baseline.down.sql`：移除对 baseline-owned `idempotency_records` table/index 的 drop。
- `docs/spec/backend-practice/plans/001-plan-and-session-orchestration/`：原地追加 1.1 remediation checklist，并在验证后恢复 `completed`。

## 6 经验教训

- Prompt truth source 的 JSON keys 必须有 runtime parser 测试直接覆盖；不能只测 generated API response shape。
- Idempotency replay gate 必须断言 replay response 来自 stored snapshot，并主动制造 mutable state drift。
- Pre-launch baseline rebase 后，新增 migration down-path ownership test，防止后续 rollback 删除 baseline-owned schema。
