# BUG-0067: frontend debrief L2 review exposed route hydration and pixel gate drift

> 模块: debrief | 严重度: high | 状态: resolved
> 发现日期: 2026-05-17 | 关联 Commit: `fix(frontend-debrief): close debrief L2 gaps (BUG-0067)`
> 关联日志: [2026-05-17](../work-journal/2026-05-17.md)

## 1 现象

- L2 review 发现 `DebriefScreen` 只读取当前 in-memory selected context，没有把 route params / InterviewContext 中的 `targetJobId`、`sessionId`、`resumeVersionId` 拉取并水合到页面草稿状态。
- English UI 下提交 `createDebrief` 仍发送 `language: "zh-CN"`，因为实现错误地从 `ctx.modality` 推导语言。
- `createDebrief` 202 成功后没有立即进入 Step 1，401 submit 没有走 auth pending action，Step 2 replay handoff 也缺少 `language`。
- P0.069 checklist 把 Playwright pixel parity 标成 completed/deferred，但场景脚本没有真实执行 debrief Playwright gate；`test-checklist.md` / `bdd-checklist.md` 状态为 completed 却仍保留未勾选项。

## 2 诊断过程

### 假设 1：Route / InterviewContext ID 是否真的会水合页面上下文
- 验证方法：阅读 `DebriefScreen.tsx` 与 `DebriefScreen.test.tsx`，新增红测断言 route/context ID 会触发 `getTargetJob` / `getPracticeSession` / `getResumeVersion` 并展示标题。
- 结果：确认。红测失败，说明页面不会从 route/context 自动恢复 target / session / resume。

### 假设 2：提交语言与 auth gate 是否来自正确来源
- 验证方法：新增红测在 English UI 下提交，并模拟 401；检查 `createDebrief` body 与 `requestAuth` payload。
- 结果：确认。提交语言来自 `ctx.modality`，401 分支没有 pending action payload。

### 假设 3：Pixel parity / BDD 证据是否真实闭环
- 验证方法：读取 P0.069 `trigger.sh` / `verify.sh`、plan checklist、test checklist 与 BDD checklist。
- 结果：确认。P0.069 只跑 i18n/privacy/legacy gates，没有调用 Playwright；completed checklist 与 BDD checklist 也没有实际勾选状态。

## 3 根因分析

- `DebriefScreen` 初始草稿状态和全局 InterviewContext / route params 之间缺少恢复层，导致 URL 或 auth pending action 回来后页面仍处于缺上下文状态。
- 提交流程复用了面试 modality 字段来判断语言，把 UI language 与 interview modality 混在一起。
- Submit success、submit auth gate 和 replay handoff 没有覆盖完整端到端断言，导致 runtime drift 通过了历史测试。
- P0.069 场景 wrapper 把 Playwright parity 当作外部/后续项，verify 也没有要求真实 Playwright marker，造成 false-green。

## 4 修复方案

- `DebriefScreen` 增加 route/context hydration：根据 route params 或 InterviewContext ID 拉取 target job、practice session、resume version，并回填 `selectedContext`。
- `createDebrief` language 改为跟随 UI locale；202 成功后立即进入 Step 1；401 submit 进入 `submit_debrief` pending action，payload 只携带 stable IDs、`practiceGoal: "debrief"` 与 language。
- Step 2 replay handoff 补齐 `language`，继续保持 debrief owner 不直接调用 `createPracticePlan` / `startPracticeSession`。
- 新增 debrief Playwright parity spec；P0.069 trigger/verify 改为真实执行 frontend build + Playwright gate，并修正 plan/test/BDD checklist 证据。

## 6 经验教训

- Frontend route entry、auth pending action replay 和 in-memory context 必须作为同一个恢复链验证；只测试直接 render 会漏掉真实用户入口。
- UI language、practice modality 和 interview mode 是不同字段，提交/恢复 payload 需要分别断言。
- Completed checklist 不能保留 unchecked items 或 deferred gate；scenario verify 必须检查真实 runner marker 与 pass marker，而不是检查 spec 文件存在或历史说明。
