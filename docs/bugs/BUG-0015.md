# BUG-0015: runtime topology lint missed false-negative worker handoff forms

> 模块: platform | 严重度: medium | 状态: resolved
> 发现日期: 2026-05-06 | 关联 Commit: `fix(runtime): harden worker topology false-negative gate`
> 关联日志: [2026-05-06](../work-journal/2026-05-06.md)

## 1 现象

- `/plan-code-review backend-runtime-topology/001-worker-consolidation repo --fix` 复扫时，`make lint-runtime-topology` 已通过，但 active plan/ADR handoff 中仍存在旧口径。
- 主要漏报形式包括 `` `worker` producer``、`app/worker listen addr`、`worker bindings` 与旧 future subject shorthand `backend-async-runtime`。
- 影响范围覆盖 runtime topology lint、B3 event/outbox handoff、A4 secrets/config completed plan 说明和 ADR-Q3 analytics handoff。

## 2 诊断过程

### 假设 1：event/generated contract 已漂移

- 验证方法：读取 `shared/events.yaml`、Go/TS generated envelope、JSON Schema/baseline，并运行 `make lint-events` 与 `make codegen-events-check`。
- 结果：排除。当前 event truth source 和 generated artifacts 使用 `backend_async`，contract 本身未漂移。

### 假设 2：runtime topology lint 的正则覆盖不完整

- 验证方法：为 `scripts/lint/runtime_topology_test.py` 增加 active surface fixture，覆盖 `` `worker` producer``、`app/worker listen addr`、`worker bindings` 和 `backend-async-runtime`。
- 结果：确认。Red 阶段 unittest 失败，说明现有 lint 只覆盖 producer-before-worker、精确 worker listen addr config、部分 worker wording，未覆盖反向语序、自然语言组合和旧 subject shorthand。

### 假设 3：问题只存在于允许保留的历史记录

- 验证方法：区分 `history.md`、tests、backend-runtime-topology owner negative assertions 与其他 active handoff docs。
- 结果：排除。漏报命中的是 B3/A4/ADR-Q3 active handoff 文案，不是单纯历史证据。

## 3 根因分析

- 根本原因是 `scripts/lint/runtime_topology.py` 的 pattern 清单过窄，只把上一轮 L2 发现的字面形式固化为 gate，没有为同义语序和旧 owner shorthand 添加 Red fixture。
- 涉及路径：
  - `scripts/lint/runtime_topology.py`: `worker producer enum` 只匹配 `producer ... worker`，漏掉 `worker ... producer`；`worker listen addr config` 只匹配精确 config key；未扫描 `backend-async-runtime` shorthand。
  - `docs/spec/event-and-outbox-contract/plans/001-bootstrap/plan.md`: completed plan handoff 仍写 `worker` producer 与 `backend-async-runtime`。
  - `docs/spec/secrets-and-config/plans/001-bootstrap/plan.md`: completed plan 修订说明仍写 `app/worker listen addr` / `worker bindings`。
  - `docs/spec/engineering-roadmap/decisions/ADR-Q3-analytics-platform.md`: handoff 仍写 `backend-async-runtime`。
- 之前没有发现，是因为上一轮 gate 只证明已知 literal 能被拦截，没有用 negative fixtures 覆盖同义旧口径。

## 4 修复方案

- 扩展 `scripts/lint/runtime_topology_test.py`，新增 false-negative fixture 并先验证 Red。
- 扩展 `scripts/lint/runtime_topology.py`：
  - 捕获 `worker ... producer` 与 `producer ... worker` 两种语序。
  - 捕获 `app/worker listen addr` 和 `worker bindings`。
  - 捕获旧 `backend-async-runtime` shorthand。
- 修订 B3/A4/ADR-Q3 active handoff 文案为 `backend_async`、`app listen addr`、`backend-async-runner` / backend async runner。
- 将 `backend-runtime-topology` spec/history/plan/checklist/context 升级到 v1.2，并新增 Phase 4.4 remediation evidence。

## 6 经验教训

- 负向 lint 不能只覆盖已经暴露的 exact literal；应为同义语序、自然语言 shorthand 和 owner 名称别名补 Red fixture。
- Completed plan/checklist 作为 active handoff source 时，仍必须被语义 lint 约束。
- Future subject rename 必须进入同一个 drift gate，否则下游 plan 可能继续引用旧 owner 名称。
