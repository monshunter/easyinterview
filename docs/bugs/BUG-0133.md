# BUG-0133: home quick-start source and CTA layout drifted from planning flow

> 模块: frontend | 严重度: medium | 状态: resolved
> 发现日期: 2026-07-06 | 关联 Commit: `fix(frontend-home): separate source and submit layout (BUG-0133)`
> 关联日志: [2026-07-06](../work-journal/2026-07-06.md)

## 1 现象

- 用户截图反馈 Home 的 JD 输入框、上传文件入口、简历选择和 `立即面试` 按钮仍混在一组松散布局里，无法表达“粘贴 JD”和“上传文件”是二选一 source。
- `选择已有简历` 下拉框撑满页面可用宽度，`还没有简历？1 分钟创建 →` 被推到远端，未贴在下拉框右侧同一行。
- `立即面试` 按钮仍位于 JD textarea 卡片右下角，和简历选择步骤脱节；这与首页作为“模拟面试新建规划快捷方式”的语义不一致。

## 2 诊断过程

### 假设 1：当前 textarea footer 可同时承载上传/URL 和提交按钮

- 验证方法：对照用户截图、owner spec Phase 10 需求、`ui-design/src/screen-home.jsx` 和正式 `HomeScreen.tsx`，新增 `HomeLayout.test.tsx` 断言 source 区域层级。
- 结果：排除。红灯显示旧实现缺少 `home-source-layout`、`home-jd-paste-panel`、`home-upload-source-panel` 和 `home-jd-input-card`，上传按钮仍在 textarea 卡片内。

### 假设 2：简历下拉满宽只是视觉细节，不影响功能

- 验证方法：在 `HomeLayout.test.tsx` 与 Home pixel parity 中断言 `home-resume-select` 定宽、`home-resume-create` 位于同一行，并确认移动端仍不溢出。
- 结果：排除。满宽下拉会把创建简历 CTA 推离选择动作，破坏首页快速新建规划的表单顺序。

### 假设 3：只移动正式前端即可闭环

- 验证方法：检查 `ui-design/` 真理源、`docs/ui-design/`、owner spec / plan / BDD、P0.014 / P0.015 场景脚本与 pixel parity。
- 结果：排除。截图反馈属于 UI 契约漂移，必须同步静态原型、正式实现、focused tests、pixel gate 和 BDD 场景文档。

## 3 根因分析

- Phase 8/9 已把首页定义为模拟面试新建规划快捷入口，但导入区域的视觉结构仍沿用“textarea footer 辅助上传 + 右下角提交”的旧 JD 解析口径。
- `home-resume-select` 只被测试为 native `select`，没有进一步锁定适度宽度、同排创建简历 CTA、提交按钮在简历选择下方的布局关系。
- P0.014 / P0.015 场景之前覆盖了简历绑定和 recent cap，但缺少 source 分区、submit-row 顺序和 textarea card 内部负向断言。

## 4 修复方案

- 更新 `ui-design/src/screen-home.jsx`：新增 `home-source-layout`，把 `home-jd-paste-panel` 和 `home-upload-source-panel` 分区；`home-resume-row` 使用 360px 下拉框并将创建简历 CTA 放到同一行；`home-submit-row` 放在简历选择状态下方。
- 更新正式 `HomeScreen.tsx`：源级复刻新原型 DOM 锚点，保留 paste / upload / URL 导入行为、ready resume gate、真实 `resumeId` 透传、import error 和 pending auth 逻辑。
- 新增 `HomeLayout.test.tsx`，扩展 Home pixel parity、`ui-design` contract、P0.014 / P0.015 trigger / verify 和相关 README / expected outcome。
- 更新 owner spec / plan / checklist / BDD、`docs/ui-design/*`，并在 Phase 10 / BDD checklist 写入本轮验证证据。

## 6 经验教训

- 首页导入入口一旦被定义为“新建模拟面试规划”，测试不只要覆盖数据绑定，还要覆盖表单步骤顺序和关键 CTA 所属区域。
- 对截图反馈中的布局关系，应补充 DOM 层级和 bounding box gate，避免后续只满足文案或控件类型而再次漂移。
- UI truth source、正式前端和 BDD 场景需要在同一个修复中同步，否则 pixel parity 和场景脚本会继续接受旧布局。
