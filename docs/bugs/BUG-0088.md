# BUG-0088: repo lint gates rejected current practice voice and JD Match contracts

> 模块: test | 严重度: medium | 状态: resolved
> 发现日期: 2026-05-22 | 关联 Commit: `fix(repo-lint): align current contract gates (BUG-0088)`
> 关联日志: `-`

## 1 现象

- `/plan-code-review local-dev-stack/001-bootstrap repo --fix` 修复 Postgres 18 本地栈后，仓库级 `make lint` 继续失败在多个与 local-dev-stack 无关的当前契约上。
- `lint-backend-practice-legacy` 把 practice voice MVP 的 `practice.voice.*`、`/voice-turns` 和 `voice-turn://...` 误判为已退役独立 Voice route。
- `lint-rubrics` 拒绝 JD Match F3 rubric 的 `relevance_to_profile`、`risk_clarity`、`actionability`、`query_alignment`、`diversity`、`privacy_compliance` 六个当前维度。
- `lint-mock-contract` 把 `createPracticeVoiceTurn` fixture/generated artifact 中合法的 Voice turn 文本和 `/voice-turns` 路径误判为退役独立 Voice tag/route。
- 后续 `make lint` 还暴露 active docs 中遗留旧 runtime topology wording，以及 Go revive 对内部 `ID` / `UI` 命名的阻断。

## 2 诊断过程

### 假设 1：practice voice MVP 已经被当前产品废弃

- 验证方法：通过 `/change-intake` 定位 owner plan，并验证 `docs/spec/practice-voice-mvp/plans/001-cascaded-stt-llm-tts/context.yaml`。
- 结果：排除。practice voice MVP 是当前 owner，`createPracticeVoiceTurn`、`/voice-turns` 和 `PracticeVoiceTurn*` 是现行契约，失败来自 retired-route lint 的匹配范围过宽。

### 假设 2：JD Match rubric 文件使用了非标准维度

- 验证方法：通过 `/change-intake` 定位 `backend-jobs-recommendations/001-jd-match-real-backend-baseline`，检查当前 F3 rubric 设计与 `config/rubrics/README.md` allowlist。
- 结果：确认 allowlist 漂移。JD Match rubric 维度已由 owner plan 引入，但共享 lint allowlist 和 README 没有同步扩展。

### 假设 3：mock runtime boundary 应继续禁止所有 Voice token

- 验证方法：验证 `mock-contract-suite/001-fixture-backed-mock-runtime` owner context，并反查 practice voice MVP fixture/generated artifacts。
- 结果：部分确认。独立 Voice tag/route 仍应禁止，但 practice voice MVP 所属的 Voice turn schema、operationId 和 `/voice-turns` path 必须允许。

### 假设 4：后续 lint 失败来自同一业务缺陷

- 验证方法：逐一运行对应 lint/test target，并读取失败路径。
- 结果：排除。runtime topology wording 与 Go revive 命名是串行暴露的独立质量门禁问题，需要分别按 owner truth source 修复。

## 3 根因分析

- 根本原因：多个 lint gate 用历史退役口径做 broad substring matching 或静态 allowlist，未随当前 owner contract 增量同步；`make lint` 又是 fail-fast，前一个 blocker 掩盖了后续 gate 漂移。
- 涉及路径：
  - `scripts/lint/backend_practice_legacy.py`
  - `scripts/lint/backend_practice_legacy_test.py`
  - `scripts/lint/rubric_lint.py`
  - `scripts/lint/rubric_lint_test.py`
  - `scripts/lint/mock_runtime_boundary.py`
  - `scripts/lint/mock_runtime_boundary_test.py`
  - `config/rubrics/README.md`
  - `backend/internal/practice/append_session_event_service.go`
  - `backend/internal/profile/`
  - `backend/cmd/api/jdmatch_runtime.go`
  - owner docs under `docs/spec/practice-voice-mvp/`, `docs/spec/backend-jobs-recommendations/`, `docs/spec/mock-contract-suite/`, and `docs/spec/backend-async-runner/`
- 为什么之前没有发现：completed plan 的局部 verification 没有强制跑到当前 repo-wide lint 全绿；退役口径 lint 没有为“当前 feature 名称包含旧 token 片段”的场景保留正向 fixture；rubric allowlist 没有被 F3 owner checklist 作为同步 gate 固化。

## 4 修复方案

- 将 `backend_practice_legacy.py` 的 Phase 3 Voice route 检测收窄为独立 `/voice` route / alias，不再匹配 `practice.voice`、`/voice-turns` 或 `voice-turn://`。
- 为 practice voice MVP operation profiles 和 fixture refs 增加 lint regression test。
- 将 JD Match F3 六个 rubric dimension 加入 `rubric_lint.py` allowlist，并同步 `config/rubrics/README.md`。
- 将 `mock_runtime_boundary.py` 从 broad string token scan 改为结构化 regex pattern：独立 `/voice` path 继续失败，`/voice-turns`、`PracticeVoiceTurn*` 和合法 Voice turn 文本允许。
- 修订 active docs 中旧 runtime topology wording，明确当前为 backend in-process runtime / 后台执行进程。
- 修复 Go revive 命名：内部 helper 使用 `clientEventIDMismatchToConflict`，profile 内部结构使用 `UILanguage`，同时保留 generated API DTO 的 `UiLanguage` 字段。
- 同步相关 owner spec / plan / checklist / history / INDEX，避免修复只停留在脚本层。

## 6 经验教训

- 退役模块 lint 必须区分“独立旧入口”与“当前 owner contract 内合法复用的词根”，尤其是 Voice / voice-turn 这类短 token。
- 新 rubric、schema、operation profile 或 generated artifact 引入共享 allowlist 时，owner checklist 应同时要求 lint allowlist、README 和 regression test 同步。
- Repo-wide lint 最好有一个审计模式可以收集全部失败项；fail-fast 适合 CI，但不适合 L2 remediation 的一次性归因。
