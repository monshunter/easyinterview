# BUG-0122: debrief suggestions ignored sessionId in real backend context

> 模块: debrief | 严重度: medium | 状态: resolved
> 发现日期: 2026-06-14 | 关联 Commit: `fix(debrief): wire session context for suggestions (BUG-0122)`
> 关联日志: [2026-06-14](../work-journal/2026-06-14.md)

## 1 现象

- `/plan-code-review backend-debrief/001-debrief-record-and-analysis backend --fix` 后续 sweep 反查 `suggestDebriefQuestions` 的 `sessionId` contract。
- OpenAPI fixture 与 BDD.P0.063 request body 已包含 `sessionId`，plan Phase 3 也声明 "可选 session 摘要"。
- 真实 backend 只在 handler/service 结构体间转发 `SessionID`，store 未按 `sessionId` 读取 practice session context，真实 F3 prompt marker `{{mock_report_summary}}` 也未被 service 替换。

## 2 诊断过程

### 假设 1：handler 没有映射 generated `sessionId`
- 验证方法：读取 `backend/internal/api/debriefs/handler.go`。
- 结果：排除。handler 已将 `body.SessionId` 映射到 `domain.SuggestQuestionsRequest.SessionID`，但没有对应测试锁定。

### 假设 2：service 没有把 `SessionID` 传给 context store
- 验证方法：读取 `backend/internal/debrief/service.go` 与新增 Red test。
- 结果：排除一半。service 已传递 `SessionID`，但真实 prompt marker `{{mock_report_summary}}` 未被替换，session summary 即使存在也不会进入真实 prompt。

### 假设 3：store 没有读取 practice session context
- 验证方法：新增 `TestStoreGetSuggestionContext_LoadsPracticeSessionSummary` 与 `TestStoreGetSuggestionContext_CrossUserSessionNotFound`。
- 结果：确认。Red 阶段 `SessionSummary` 为空，missing/cross-user session 也返回 nil error。

## 3 根因分析

- 根本原因：Phase 3 的 "可选 session 摘要" 停在 domain request 字段和文档声明层，`Repository.GetSuggestionContext` 只读取 target job 与 resume，不读取 `practice_sessions` / `practice_turns` / `feedback_reports`。
- 涉及路径：
  - `backend/internal/store/debrief/store.go`
  - `backend/internal/debrief/service.go`
  - `backend/internal/api/debriefs/handler.go`
  - `test/scenarios/e2e/p0-063-debrief-suggest-questions/scripts/trigger.sh`
  - `test/scenarios/e2e/p0-063-debrief-suggest-questions/scripts/verify.sh`
- 之前没有发现的原因：P0.063 wrapper 只在 BUG-0121 后强化了 `resumeId` focused gates，未把同一个 fixture 中的 `sessionId` 作为真实 backend gate；service tests 使用内部 marker 名称，没有覆盖真实 prompt config 的 `{{mock_report_summary}}`。

## 4 修复方案

- `Repository.GetSuggestionContext` 在 request 带 `SessionID` 时，按 `(user_id, target_job_id, session_id)` 读取 `status='completed'` 的 practice session derived summary，并包含 `practice_turns.answer_summary` 与 ready `feedback_reports` 摘要；missing/cross-user/wrong-target/uncompleted session fail-closed 为 `ErrDebriefPrerequisite`。
- `renderSuggestTemplate` 兼容真实 F3 prompt marker：`{{role_title}}`、`{{job_summary}}`、`{{mock_report_summary}}`、`{{resume_highlights}}`，同时保留既有内部 marker。
- 增加 store/service/API tests，并把 P0.063 trigger/verify 扩展到 sessionId contract marker、test start/PASS marker、package `ok` 与 no-op test 负向断言。
- 原地更新 backend-debrief owner plan/checklist/test/BDD/history，确保 operation matrix 记录真实 sessionId handler/service/store/prompt/scenario gate。

## 6 经验教训

- Cross-layer fixture 中的每个 optional context id 都需要独立的 handler/service/store/prompt/scenario gate；不能因为同 operation 的另一个字段已闭环就视为整条 request contract 已闭环。
- AI prompt 测试要覆盖真实 F3 prompt marker，而不只是 service-local marker。
