# BUG-0177: OPENAPI-001 refreeze left its root regression test bound to the new baseline

> 模块: test | 严重度: medium | 状态: resolved
> 发现日期: 2026-07-15 | 关联 Commit: `feat(report): align context grid and acceptance (BUG-0177)`
> 关联日志: [2026-07-15](../work-journal/2026-07-15.md)

## 1 现象

- 报告 Context Strip 与响应式布局实现完成后，根 `make test` 出现 3 个 `OpenAPI001V17ConversationTests` 失败。
- 失败声称 merge-base 必须保留 `listPracticeSessions`，同时无法观测该 operation 的 removal finding。
- 本次 UI diff 没有修改 `openapi/`、baseline、decision oracle 或 OpenAPI linter 实现，但该漂移会阻塞任何后续功能的根回归收口。

## 2 诊断过程

### 假设 1：本次报告 UI 变更意外修改了 OpenAPI 契约

- 验证方法：检查 `git diff -- openapi scripts/lint/openapi_diff.py scripts/lint/openapi_diff_test.py`，并对比当前 baseline/current operation inventory。
- 结果：排除；失败发生时上述路径无本次改动，当前 baseline 与 current 都已包含 report-owned conversation 契约。

### 假设 2：OPENAPI-001 v1.7 测试仍从已 refreeze 的 `main` 读取“旧 baseline”

- 验证方法：读取 `OpenAPI001V17ConversationTests._baseline()`、OPENAPI-001 preserved audit 和 `003-breaking-change-gate` Phase 11 的 post-refreeze 证据。
- 结果：确认；helper 使用 `git show main:openapi/baseline/openapi-v1.0.0.yaml`，而 refreeze 后该文件已经是新契约。preserved audit 仍正确记录 pre-refreeze Git source 与 15 条 findings。

## 3 根因分析

- `scripts/lint/openapi_diff_test.py` 的 OPENAPI-001 v1.7 helper 把分支名 `main` 当作不可变旧 baseline。
- Phase 11 完成 guarded refreeze 后，`main` baseline 合法更新，但 helper 未切换到 `openapi/baseline/audits/OPENAPI-001-report-conversation.json` 中的 preserved `baselineSource`。
- 之前的 post-refreeze 验证覆盖 preserved audit replay，却没有让该早期 helper 在 refreeze 后重新解析同一 preserved source，导致根测试随后自相矛盾。

## 4 修复方案

- 让 OPENAPI-001 v1.7 test helper 从 preserved audit 读取并校验 `git:<ref>:<path>`，再由 `_git_show` 加载 pre-refreeze baseline。
- 保留 current OpenAPI、baseline、oracle、linter 和正式 API 契约不变。
- 未选择修改 baseline 或放宽 contract validator，因为失败来自测试事实源漂移，而不是产品契约错误。

## 5 验证

- `python3 -m pytest scripts/lint/openapi_diff_test.py -k OpenAPI001V17ConversationTests`：4 passed。
- 根 `make test`：559 Python tests / 4481 subtests、Go 全包、frontend 126 files / 1003 tests 全部通过。
- `make lint`、`make build`、frontend typecheck 与 `make docs-check` 通过。

## 6 经验教训

- 需要跨 refreeze 保留的 decision test 必须以 preserved audit 的 Git source 为事实源，不能把可前进的分支名当成历史快照。
- 本次模式具有复用价值，但先由当前 Bug 记录承接；是否抽成共享 helper 可在后续 OpenAPI gate 维护中评估。
