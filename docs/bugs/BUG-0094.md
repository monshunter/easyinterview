# BUG-0094: gitleaks lint scanned ignored local env secrets

> 模块: test | 严重度: medium | 状态: resolved
> 发现日期: 2026-05-23 | 关联 Commit: `feat(prompt-rubric): close output schema contract (BUG-0094)`
> 关联日志: [2026-05-23](../work-journal/2026-05-23.md)

## 1 现象

- 实施 `prompt-rubric-registry/002-output-schema-contract` 后执行 `make lint`，`lint-secrets-pattern` 在本机失败。
- 失败来自 `scripts/lint/gitleaks.sh` 对整个工作树运行 gitleaks；本机 `deploy/dev-stack/.env` 已被 `.gitignore` 排除，但仍被扫描到。
- 影响范围：带本地运行时 `.env` 的开发机无法稳定运行 repo-wide lint；secret scanner 也会读取不应进入仓库质量门禁输入集的本机私有配置文件。

## 2 诊断过程

### 假设 1：新增 prompt schema 或 migration 写入了疑似 secret

- 验证方法：读取 `make lint` 失败输出，反查 gitleaks 报告中的文件路径。
- 结果：排除。失败路径指向本机 `deploy/dev-stack/.env`，不是本次新增的 schema、prompt body、Go 代码或 migration。

### 假设 2：`.env` 文件应被纳入仓库 lint 输入

- 验证方法：检查 `.gitignore` 与 `git ls-files --others --exclude-standard` 候选集，确认该文件属于本机 ignored runtime config。
- 结果：排除。该文件不应作为 repo lint 的候选输入；lint 应覆盖 tracked files 与未忽略的新增文件，而不是扫描所有本机运行时文件。

### 假设 3：gitleaks 脚本的扫描入口与 Makefile 语义不一致

- 验证方法：读取 `scripts/lint/gitleaks.sh`，对比 `lint-secrets-pattern` 期望的仓库候选文件语义。
- 结果：确认。脚本使用整棵工作树作为 source，导致 `.gitignore` 无法保护本机 runtime `.env`。

## 3 根因分析

- 根本原因：`scripts/lint/gitleaks.sh` 把 gitleaks source 指向整个 repo 工作树，而不是 git tracked + unignored candidate set。
- 涉及路径：`scripts/lint/gitleaks.sh`。
- 为什么之前没有发现：此前验证环境没有带真实本地 `deploy/dev-stack/.env`，或者没有安装会触发二层扫描的 gitleaks；已有 gate 没有覆盖 ignored runtime config 被 scanner 误纳入的负向场景。

## 4 修复方案

- 将 `scripts/lint/gitleaks.sh` 改为创建临时 scan mirror，只复制 `git ls-files --cached --others --exclude-standard` 返回的 tracked 与未忽略候选文件。
- 对临时 mirror 运行 `gitleaks detect --no-git --redact`，保留真实 secret pattern 检测能力，同时不读取 `.gitignore` 下本机 runtime `.env`。
- 保留兜底行为：未安装 gitleaks 时仍执行仓库内轻量 secret pattern 扫描。

## 5 验证

- `bash -n scripts/lint/gitleaks.sh`：通过。
- `bash scripts/lint/gitleaks.sh .`：通过，未扫描 ignored local `.env`。
- `make lint`：通过，包含 `lint-secrets-pattern`、`lint-prompts`、`lint-rubrics`、OpenAPI check 与 frontend lint placeholder。
- `git diff --check`：通过。

## 6 经验教训

- secret / lint gate 的输入集必须显式定义，优先使用 git tracked + unignored candidates；不得默认扫描整棵开发机工作树。
- 对本地 runtime `.env`、临时证据、缓存目录等 ignored 文件，quality gate 应证明不会读取它们；真实 secret 只能通过未忽略文件或 tracked 文件进入失败路径。
