# BUG-0060: backend-practice hint AI drifted from lightweight observe prompt contract

> 模块: practice | 严重度: high | 状态: resolved
> 发现日期: 2026-05-15 | 关联 Commit: `fix(backend-practice): align hint prompt contract`
> 关联日志: `-`

## 1 现象

- L2 review 发现 `practice.turn.lightweight_observe` 的真实 F3 prompt 要求模型返回 `cue` / `severity` / `dimension_hint`，但 backend-practice `parseHint` 只接受测试 fixture 中自造的 `hint` 字段。
- 同一路径的 prompt renderer 复用了 first-question renderer，导致 `{{question}}`、`{{partial_answer}}`、`{{elapsed_seconds}}` 等 hint 模板占位符原样传给 A3 provider。
- 影响范围是 assisted `hint_requested` 主路径：真实 F3 输出会被误判为空并 graceful degrade 为 `session_wait`，且 A3 prompt 缺少当前问题、局部答案与耗时上下文。

## 2 诊断过程

### 假设 1：F3 lightweight observe 当前输出 schema 与 service parser 不一致

- 验证方法：新增 `TestParseHintAcceptsLightweightObserveCueSchema`，输入 `{"cue":"...","severity":"nudge","dimension_hint":"evidence"}`。
- 结果：确认。修复前 `parseHint` 返回 `hint is empty`，导致 `applyHintAI` 把真实 cue response 走 invalid-output degrade。

### 假设 2：成功路径测试 fixture 掩盖了真实 prompt contract

- 验证方法：把 `TestApplyHintAISuccess` fake AI response 改成当前 F3 prompt 要求的 `cue` JSON Content。
- 结果：确认。修复前测试返回 `session_wait`，没有产生 `show_hint`。

### 假设 3：hint prompt 已经正确展开模板变量

- 验证方法：新增 `TestApplyHintAIBuildsPromptFromF3Template`，使用含 `{{question}}`、`{{partial_answer}}`、`{{elapsed_seconds}}`、`{{language}}` 的 F3-like 模板并捕获 provider prompt。
- 结果：排除。修复前 prompt 仍包含未渲染占位符，只额外追加了 question/answer 长度摘要。

## 3 根因分析

- 根本原因：003 hint AI 接入时用本地测试 fixture 定义了 `{"hint":...}` 成功 schema，没有把 F3 `practice.turn.lightweight_observe` 当前 prompt 的 response contract 纳入 service 单元测试。
- prompt 侧复用了 first-question 模板 renderer；该 renderer 不认识 hint 模板 token，因此不会展开当前问题、局部答案和 elapsed seconds。
- 涉及的代码路径：
  - `backend/internal/practice/hint_ai.go`
  - `backend/internal/practice/append_session_event_service_test.go`
- 之前没有发现的原因：既有成功路径断言只证明自造 `hint` fixture 可解析，未覆盖真实 F3 prompt schema；prompt 测试只检查明文泄漏风险，没有断言模板变量已经展开且无残留。

## 4 修复方案

- `parseHint` 同时接受历史 fixture 的 `hint` 与当前 F3 prompt 的 `cue`，保持向后兼容并恢复真实 `lightweight_observe` 主路径。
- `hintPayload` 改用 dedicated hint renderer，展开 `{{language}}`、`{{question}}`、`{{partial_answer}}`、`{{elapsed_seconds}}`、`{{practice_goal}}`、`{{practice_mode}}`。
- 更新 service 单元测试，使成功路径使用 `cue` JSON Content，并新增 schema/parser 与 prompt template 断言。
- 同步 003 plan checklist、test-plan、test-checklist、BDD checklist 与 Bug 索引，避免文档证据继续指向旧测试名。

## 6 经验教训

- AI-backed feature 的成功 fixture 必须来自当前 prompt contract，而不是由 service 测试临时发明字段名。
- Prompt renderer 测试需要同时覆盖隐私红线和模板变量完整展开；只检查“不泄漏”不足以证明 prompt 可用。
- completed plan 的 L2 review 仍要反查 F3 prompt、A3 payload、service parser、测试 fixture 与文档证据，避免单层绿色掩盖跨 owner drift。
