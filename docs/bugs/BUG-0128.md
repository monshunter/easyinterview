# BUG-0128: core-loop pruning gates missed design canvas and privacy hook drift

> 模块: test | 严重度: medium | 状态: resolved
> 发现日期: 2026-06-29 | 关联 Commit: `fix(core-loop): close pruning review drift (BUG-0128)`
> 关联日志: [2026-06-29](../work-journal/2026-06-29.md)

## 1 现象

- `product-scope/001-core-loop-module-pruning` 已完成并通过历史 final gates 后，L2 review 仍发现 `ui-design/canvas.html` 保留正向 Profile / Debrief 画板。
- `backend/internal/privacy/runner` 仍暴露 `PrivacyDeleteHandlerOptions.ProfileData`，测试把 `DeleteCandidateProfileForUser` 当作 privacy delete 主路径扩展点。
- `backend/README.md` 仍把 `profile` 列为后端领域模块，和 D-22 删除候选人画像领域的完成口径不一致。

## 2 诊断过程

### 假设 1：UI truth source gate 只覆盖 `ui-design/src`
- 验证方法：读取 `ui-design/ui-design-contract.test.mjs` 与 `ui-design/canvas.html`。
- 结果：确认。原测试只检查 deleted screen files 与 `src/app.jsx` route alias，`canvas.html` 只在主题/语音测试中被读取，没有 non-current artboard negative assertion。

### 假设 2：privacy delete 残留只是测试字符串
- 验证方法：搜索并读取 `backend/internal/privacy/runner/delete_handler.go`、`delete_handler_test.go` 和 `backend/cmd/api/main.go`。
- 结果：部分排除。`cmd/api` 当前没有传入 hook，但 production handler options 和 handler execution path 仍允许旧 profile cleanup hook 被重新接入。

### 假设 3：文档漂移只影响历史记录
- 验证方法：读取 `backend/README.md` 当前模块边界描述。
- 结果：确认不是历史记录。该 README 是 active backend owner contract，仍会误导后续后端 workstream。

## 3 根因分析

- 零残留 gate 把 `ui-design/src/app.jsx` 当作唯一静态原型 truth source，漏掉了同样 Git 跟踪且可见的 `ui-design/canvas.html` 画板总览。
- privacy boundary gate 关注表、route、OpenAPI、prompt/config 删除，但没有断言后端 handler options 不再暴露候选人画像领域扩展点。
- plan/checklist 历史 PASS 证据未覆盖这些当前态 artifact，导致 completed 状态掩盖了 residual active truth-source drift。

## 4 修复方案

- 在 `ui-design/ui-design-contract.test.mjs` 的 D-22 测试中读取 `canvas.html`，断言旧 Profile / Debrief section、artboard 和 route 不再出现。
- 删除 `ui-design/canvas.html` 中的 Profile、Debrief 和语音复盘正向画板，仅保留当前核心入口和简历工坊画板。
- 删除 `PrivacyDeleteHandlerOptions.ProfileData`、handler 内部 `profileData` 调用以及旧 fake profile deleter 测试，新增反射回归断言防止该 hook 回流。
- 更新 `backend/README.md` 后端领域列表，移除 `profile` 并保留当前 `resume` domain。
- 原地修订 `product-scope/001-core-loop-module-pruning` plan/checklist，把 canvas non-current-artboard gate 和 privacy hook legacy grep 固化为 v1.1 完成证据。

## 5 验证

- `node --test ui-design/ui-design-contract.test.mjs` PASS (28 tests)。
- `go test ./backend/cmd/api ./backend/internal/privacy/runner -count=1` PASS。
- `make docs-check` PASS。
- `git diff --check` PASS。
- `rg 'route="profile"|route="debrief"|<DCSection id="profile"|<DCSection id="p1-depth"|<DCSection id="p1-voice-debrief"|<DCArtboard id="profile-|<DCArtboard id="debrief' ui-design/canvas.html` 无命中。
- `rg "ProfileData|DeleteCandidateProfileForUser|fakeProfileDataDeleter|profile delete failed" backend/internal/privacy backend/README.md` 仅命中测试中的 `FieldByName("ProfileData")` 防回归断言。

## 6 经验教训

- Module pruning 的 zero-reference gate 不能只覆盖 runtime code 或主 source tree；可见 prototype overview、owner README、handler option surface 也必须纳入 active artifact map。
- 对删除类计划，应新增“正向旧入口零残留”断言，而不是只证明旧 screen file 已删除。
