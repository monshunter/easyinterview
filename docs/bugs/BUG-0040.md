# BUG-0040: workspace pixel gate depended on stale hydration and ignored baselines

> 模块: frontend | 严重度: medium | 状态: resolved
> 发现日期: 2026-05-10 | 关联 Commit: `fix(frontend-shell): harden workspace pixel parity gate`
> 关联日志: `-`

## 1 现象

- 触发条件：在当前 frontend dist 上运行完整 `pnpm --filter @easyinterview/frontend test:pixel-parity`。
- 可观察错误：`screenshot.spec.ts` 与 `workspace.spec.ts` 首跑需要 `*-snapshots/*-darwin.png`，这些 PNG 被 `.gitignore` 排除；workspace full-state 用例通过 Home recent card 进入，拿到 `resume-unbound` 后停在 missing-resume 状态，`workspace-header-title` 不出现。
- 影响范围：完整 Playwright pixel parity gate 无法在 clean checkout 上稳定通过；E2E.P0.006 的 verify 脚本仍按旧 68 tests / 7 spec 口径验收，不能证明 workspace spec 已进入 full-state。

## 2 诊断过程

### 假设 1：截图 baseline 缺失只是本地环境问题

- 验证方法：聚焦运行 `pnpm --filter @easyinterview/frontend exec playwright test tests/pixel-parity/screenshot.spec.ts tests/pixel-parity/workspace.spec.ts`。
- 结果：确认失败来自 Playwright `toHaveScreenshot` 缺少 ignored `*-darwin.png` baseline；这不是可要求 clean checkout 预置的文件。

### 假设 2：workspace full-state 是业务数据 fixture 缺失

- 验证方法：检查 `workspace.spec.ts`、`interviewContextFromTargetJob` 与 workspace hooks 的 `normalizeServerBoundId` 使用。
- 结果：确认 Home recent card route params 使用 `resume-unbound` 是产品语义；workspace hook 正确过滤 synthetic id，并因此展示 missing-resume，而 pixel full-state 测试错误复用了该入口。

## 3 根因分析

- 根本原因：pixel gate 在扩展业务屏覆盖后继续沿用早期 D3 的本地 screenshot baseline 前提，并把 workspace full-state 测试绑定到 Home recent card 入口。
- 涉及代码路径：
  - `frontend/tests/pixel-parity/screenshot.spec.ts`
  - `frontend/tests/pixel-parity/workspace.spec.ts`
  - `frontend/src/main.tsx`
  - `test/scenarios/e2e/p0-006-ui-design-pixel-parity-gate/scripts/verify.sh`
- 为什么之前没有发现：P0.006 初始 plan 只覆盖 4 spec / 48 tests，后来新增 home / parse / jd_match / workspace 后，scenario README 与 verify 脚本没有同步升级；README 虽已记录 clean checkout 不应依赖 ignored baseline，但旧 spec 仍保留 hard gate。

## 4 修复方案

- 选择方案：常规 screenshot gate 改为非空 screenshot buffer smoke；workspace full-state 通过测试注入的 initial route 使用 server-bound `targetJobId` / `resumeVersionId` / `planId` 进入，不修改 Home recent card 的 `resume-unbound` 业务语义。
- 放弃方案：不把 ignored PNG 签入仓库，也不把 `interviewContextFromTargetJob` 改为绑定默认简历，因为这会绕过 workspace synthetic id 防线并改变产品入口语义。
- 修改文件列表：`frontend/src/main.tsx`、`frontend/tests/pixel-parity/{screenshot,workspace}.spec.ts`、`frontend/README.md`、`test/scenarios/e2e/p0-006-ui-design-pixel-parity-gate/*`、`docs/spec/frontend-shell/plans/003-ui-design-pixel-parity-gate/*`。

## 5 验证

- Red：聚焦 Playwright 首跑 `screenshot.spec.ts + workspace.spec.ts` 复现 12 failed / 24 passed，失败原因覆盖 missing baseline 与 hydrated workspace timeout。
- Green：同一聚焦命令修复后 36 passed。
- 回归：`pnpm --filter @easyinterview/frontend test:pixel-parity` 110 passed；E2E.P0.006 setup→trigger→verify→cleanup PASS；`pnpm --filter @easyinterview/frontend test` 108 files / 681 tests PASS；`pnpm --filter @easyinterview/frontend typecheck`、`pnpm --filter @easyinterview/frontend build`、`make docs-check`、context validator、`git diff --check` 均通过。

## 6 经验教训

- 被 `.gitignore` 排除的 screenshot baseline 不能作为默认 clean checkout gate；常规 gate 应优先使用 DOM / computed / bounding / responsive geometry / screenshot smoke。
- Full-state pixel tests 如果要验证业务态，必须明确注入 server-bound route params 或场景 fixture，不能复用会刻意进入 empty / missing-state 的用户入口。
