# BUG-0156: root test gate omitted Python tooling and skill contracts

> 模块: test | 严重度: medium | 状态: resolved
> 发现日期: 2026-07-10 | 关联 Commit: `fix(test): aggregate Python contract tests (BUG-0156)`
> 关联日志: -

## 1 现象

- 根 `make test` 只执行 backend Go 与 frontend Vitest，未执行 `scripts/` 和 `.agent-skills/` 下的 Python tests。
- 直接运行全量 Python suite 时，463 个测试中 1 个失败：work-journal auto-mode subject derivation 仍断言旧文案。
- 在修复前，根 test gate 可以保持绿色，同时遗漏数百个 tooling/skill contract tests。

## 2 诊断过程

### 假设 1：Python tests 只是未使用的零引用文件

- 验证方法：分别执行 `pytest --collect-only` 扫描 `scripts/` 与 `.agent-skills/`。
- 结果：排除。两处共收集 463 个测试，覆盖 lint/codegen/scenario/governance tools 与 skill contracts。

### 假设 2：work-journal SKILL 规则回退

- 验证方法：对照 `SKILL.md` 当前英文/ASCII commit policy 与失败断言。
- 结果：排除。SKILL 当前要求移除 Phase 前缀后翻译或概括为简洁英文，并在自然情况下小写；失败测试仍锁定旧的直接 lowercase remainder 文本。

### 假设 3：根 test 聚合与 Python dependency 声明均未随资产增长更新

- 验证方法：读取 A5 spec/plan、根 Makefile、README 与 Python import inventory。
- 结果：确认。A5 只登记 Go/TS；仓库没有 requirements 文件，而 Python tooling/tests 的外部依赖实际只有 `pytest` 与 `PyYAML`。

## 3 根因分析

- A5 test 聚合在早期只覆盖 Go/TS，后续新增的 Python tooling 与 skill contracts 没有回填根入口。
- test 文件存在并可被 pytest 收集，但缺少统一执行面，因此单文件 contract drift 不会阻断常规 `make test`。
- work-journal 规则已经增强为中文 heading 的英文翻译/概括，测试仍保留旧字面量断言。
- 涉及路径：`Makefile`、`requirements-dev.txt`、`.agent-skills/work-journal/scripts/test_work_journal_skill_contract.py`、`scripts/lint/makefile_dry_run_test.py`。

## 4 修复方案

- 修正 work-journal 合同测试，使其断言当前英文翻译/概括与自然小写规则，不倒改 SKILL。
- 新增根 `requirements-dev.txt`，精确声明当前已验证的 `pytest` 与 `PyYAML` 版本。
- 在既有 `make test` 首段执行 `python3 -m pytest scripts .agent-skills -q`，随后保留 Go 与 frontend tests；不新增平行 target。
- 用 Makefile contract test 固定 Python → Go → frontend 顺序与依赖声明，并同步 A5 owner 和 onboarding 文档。

## 5 验证

- RED：focused gate 为 2 failures，分别证明 Python suite/requirements 未接入，以及 work-journal 断言陈旧。
- GREEN：focused tests 为 2 / 2；全量 Python suite 为 464 passed + 4269 subtests。
- 根 `make test` 继续通过全部 backend packages 与 frontend 136 files / 836 tests。
- `PYTEST_ADDOPTS` 零匹配探针使 Python 首段返回 5，`make test` 随即非零退出且未进入 Go/Vitest。
- `make lint`、A5/product contexts、requirements temporary-target dry-run、docs/index/diff/pruning gates 通过。

## 6 经验教训

- 技术栈测试文件的存在不等于聚合 gate 已执行；应定期对 test inventory 与根 target 做双向核对。
- contract test 应锁定当前语义，不应为了匹配旧字面量迫使更严格的新规则回退。
- Python tooling 一旦进入根 gate，其外部依赖必须与 Go/Node 依赖一样显式声明。
