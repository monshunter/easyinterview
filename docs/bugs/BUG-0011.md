# BUG-0011: mock contract gate ignored empty retired fixture tag directories

> 模块: test | 严重度: medium | 状态: resolved
> 发现日期: 2026-05-06 | 关联 Commit: `fix(mock-contract): reject retired fixture tag dirs`
> 关联日志: [2026-05-06](../work-journal/2026-05-06.md)

## 1 现象

- `$plan-code-review mock-contract-suite` 复扫发现 `openapi/fixtures/Growth` 与 `openapi/fixtures/Mistakes` 仍作为空目录存在。
- `git status --short --untracked-files=all` 无输出，因为 Git 不跟踪空目录。
- `make lint-mock-contract` 通过，说明现有 gate 只覆盖 fixture JSON 和文件内容，未覆盖目录级 artifact。

## 2 诊断过程

### 假设 1：旧目录中仍有可跟踪 fixture 文件

- 验证方法：运行 `git ls-files openapi/fixtures/Growth openapi/fixtures/Mistakes` 与 `find openapi/fixtures/Growth openapi/fixtures/Mistakes -maxdepth 2 -print`。
- 结果：排除。两个目录为空且未被 Git 跟踪。

### 假设 2：retired token gate 只扫描文件内容

- 验证方法：阅读 `scripts/lint/mock_runtime_boundary.py`，并运行目录集合负向检查。
- 结果：确认。lint 扫描了 fixture JSON、前端 API 和 mockruntime 文件内容，但没有检查 `openapi/fixtures` 的 tag 目录集合。

### 假设 3：现有聚合 gate 会通过其他方式拒绝旧 tag 目录

- 验证方法：运行 `make lint-mock-contract`。
- 结果：排除。聚合 gate 在空旧目录存在时仍通过。

## 3 根因分析

- 根本原因：mock runtime boundary lint 没有把 fixture tag 目录名作为 contract artifact 校验对象。
- 涉及路径：
  - `scripts/lint/mock_runtime_boundary.py`
  - `scripts/lint/mock_runtime_boundary_test.py`
  - `openapi/fixtures/`
- 为什么之前没有发现：旧目录为空且未被 Git 跟踪，常规 `git status` 和文件内容负向搜索都不会暴露。

## 4 修复方案

- 在 `mock_runtime_boundary.py` 中从 `openapi/openapi.yaml` 的 `tags` 派生允许 tag 目录集合。
- 对 `openapi/fixtures/` 下额外或缺失的 tag 目录直接报错，并输出 `docs/spec/mock-contract-suite/spec.md` owner hint。
- 增加 focused test：即使旧 tag 目录为空，也必须失败。
- 删除实际工作区残留的空 `Growth` / `Mistakes` 目录。
- 原地修订 `mock-contract-suite` spec / plan / checklist，固化目录级 gate。

## 5 验证

- Red：`python3 -m unittest scripts.lint.mock_runtime_boundary_test.MockRuntimeBoundaryTest.test_retired_fixture_tag_directory_fails_even_when_empty -v` 先失败，返回码仍为 0。
- Green：同一 focused test 通过。
- 回归：
  - `python3 scripts/lint/mock_runtime_boundary.py --repo-root .`
  - `python3 -m unittest scripts.lint.mock_runtime_boundary_test -v`
  - `make lint-mock-contract`
  - `make codegen-check`
  - `make docs-check`

## 6 经验教训

- 对“目录结构本身也是契约”的真理源，负向搜索不能只扫文件内容；需要显式校验目录集合。
- Git 不跟踪空目录，涉及 zero-reference / retired-directory 场景时必须用 filesystem-level 检查补足。
