# BUG-0014: backend runtime topology L2 review exposed worker drift gate gap

> 模块: platform | 严重度: medium | 状态: resolved
> 发现日期: 2026-05-06 | 关联 Commit: `fix(runtime): harden worker topology drift gate`
> 关联日志: [2026-05-06](../work-journal/2026-05-06.md)

## 1 现象

- `/plan-code-review backend-runtime-topology/001-worker-consolidation repo --fix` 复扫发现 worker consolidation 已完成后，active runtime 代码注释和多个 completed owner plan/checklist 仍保留 non-current standalone worker process 口径。
- 主要表现包括 `cmd/worker`、`cmd/{api,worker}`、worker producer enum、worker 类组件 probe、privacy worker、Asynq worker 等旧语义残留。
- 影响范围覆盖 backend runtime topology owner spec/plan、Makefile 聚合 lint、backend 注释，以及 event/outbox、secrets/config、local-dev-stack、db migrations、AI provider routing 等历史 owner plan 的交接语义。

## 2 诊断过程

### 假设 1：旧 4.2 negative search gate 没有扫描 active plan/checklist 面

- 验证方法：按 L2 review 要求反向搜索 `docs/spec/*/plans/**`，并对照 `backend-runtime-topology` checklist 4.2 的历史 evidence。
- 结果：确认。旧 gate 主要证明 runtime/code/config 中不再存在 standalone worker entrypoint，但未把 completed owner plan/checklist 的继续交付指令纳入同一零残留范围。

### 假设 2：代码注释中的 non-current worker scope 未被现有 lint 捕获

- 验证方法：新增 `make lint-runtime-topology` 后先在当前仓库运行。
- 结果：确认。lint 首轮拦截到 `backend/internal/migrations/privacy.go`、`backend/internal/platform/secrets/doc.go`、`backend/internal/platform/secrets/env_provider.go` 中的旧 worker 口径。

### 假设 3：问题不是历史记录本身，而是 active handoff surface 漂移

- 验证方法：区分 `docs/spec/backend-runtime-topology/history.md` 等历史说明与其他 active owner plan/checklist 的可执行任务语句。
- 结果：确认。历史记录可保留旧词作为事实描述；active handoff plan/checklist、runtime code comment、Make target 和 config/deploy 文档不应继续指导独立 worker process。

## 3 根因分析

- 根本原因是 worker consolidation 原完成 gate 依赖人工 `rg` 组合，扫描面和 pattern 清单都偏窄，未形成仓库级可重复执行的 runtime topology drift lint。
- 涉及路径：
  - `docs/spec/backend-runtime-topology/plans/001-worker-consolidation/checklist.md`: 旧 4.2 evidence 没有约束 `docs/spec/*/plans/**`。
  - `Makefile`: 修复前没有 `lint-runtime-topology` 聚合 gate。
  - `backend/internal/migrations/privacy.go`、`backend/internal/platform/secrets/doc.go`、`backend/internal/platform/secrets/env_provider.go`: active code comments 保留旧 worker scope。
  - `docs/spec/*/plans/**`: completed owner plan/checklist 保留 non-current worker handoff 语义。
- 之前没有发现，是因为历史 PASS 和 completed checklist 被当作完成线索；L2 复查前没有 artifact-level 反查 active owner plan/checklist，也没有用 executable lint 固化负向搜索。

## 4 修复方案

- 新增 `scripts/lint/runtime_topology.py` 和 `scripts/lint/runtime_topology_test.py`，将 non-current worker process 负向搜索固化为可测试 lint，并保留历史 owner 文档 allowlist。
- 在 `Makefile` 中新增 `lint-runtime-topology`，并接入 `make lint` 聚合 gate。
- 修订 backend active 注释和多个 owner plan/checklist，把独立 worker entrypoint / worker producer / worker probe 口径改为 backend internal runner、backend runtime entrypoint 或当前有效命名。
- 将 `backend-runtime-topology` spec/plan/checklist/context bump 到 v1.1，新增 C-5 acceptance 和 Phase 4.3 L2 remediation evidence，要求后续通过 `make lint-runtime-topology`。
- 同步 `docs/spec/INDEX.md` 与 `docs/spec/backend-runtime-topology/plans/INDEX.md`。

## 6 经验教训

- 删除 runtime entrypoint 或组件拓扑时，negative gate 必须扫描 active owner plan/checklist，而不只是 runtime code 和 config。
- Completed plan/checklist 仍可能承载后续 handoff 指令；只要语句是当前可执行口径，就应纳入漂移检查。
- Non-current topology 词汇适合用 executable lint 固化，并用 unit test 覆盖 allowlist 与 negative fixture，避免后续人工 `rg` 漏扫。
