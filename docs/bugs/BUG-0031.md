# BUG-0031: workspace L2 review exposed route hydration and start flow drift

> 模块: frontend | 严重度: medium | 状态: resolved
> 发现日期: 2026-05-09 | 关联 Commit: `fix(frontend-workspace): remediate workspace L2 findings`
> 关联日志: `-`

## 1 现象

- `$plan-code-review frontend-workspace-and-practice --fix` L2 复核发现 workspace plan 已标记完成，但正式前端仍存在多处运行时漂移。
- `App.tsx` 没有把 `workspace/practice/generating` route params 自动 hydrate 到 `InterviewContext`，只能依赖测试手动写入 context。
- `WorkspaceScreen` 未正式挂载 `PlanSwitcherModal` / `ResumePickerModal`，`getPracticePlan` refresh 与 archived/404 replacement path 不完整，`autoStartPractice=1` 登录恢复未自动执行 startPractice。
- BDD checklist 与 scenario scripts 仍停留在弱验证状态，pixel parity 只覆盖空态，无法证明 full workspace 与 modal geometry。

## 2 诊断过程

### 假设 1：只是 checklist 状态未同步
- 验证方法：直接读取 plan/checklist/BDD checklist、workspace runtime code、App route wiring、scenario scripts 与 pixel parity spec。
- 结果：排除。文档状态漂移只是表象；代码层确实缺 route hydrate、modal mount、practice plan refresh 和 auto-start runtime 行为。

### 假设 2：问题集中在测试断言不足
- 验证方法：反查 `WorkspaceScreen`、`useStartPractice`、`useWorkspacePracticePlan`、`WorkspaceAuthGate.test.tsx`、`workspace.spec.ts` 与 P0.018-P0.021 trigger/verify scripts。
- 结果：部分确认。旧测试以 hook/组件局部行为为主，未覆盖真实 App route → context、WorkspaceScreen button → modal、archived/404 replacement、pendingAction auto-start 与 full workspace screenshot。

## 3 根因分析

- 首次实现把 `InterviewContext` 当作局部测试前置条件处理，没有在 App route 层建立统一 hydrate/clear owner。
- modal 与 startPractice 的 plan 任务被局部组件测试覆盖，但缺正式 WorkspaceScreen integration gate，导致按钮与 modal、最新 planId 导航、auth 恢复流程没有被同一个 runtime path 锁住。
- BDD checklist 未随脚本真实断言更新，历史 PASS 与 checklist 勾选掩盖了场景脚本弱验证。

## 4 修复方案

- 在 `App.tsx` 增加 route sync，在 carry routes 内 dispatch `HYDRATE_FROM_ROUTE`，离开 carry route 时 `CLEAR`。
- 扩展 `InterviewContext` reducer，增加 `CLEAR_RESUME` / `CLEAR_PRACTICE_PLAN` / `CLEAR_AUTO_START`，并 hydrate `mode/modality/hint/session/autoStartPractice` 等 route params。
- 在 `WorkspaceScreen` 正式挂载 `PlanSwitcherModal` / `ResumePickerModal`，接入 `useWorkspacePracticePlan`，补齐 authenticated auto-start effect 与 replacement planId 导航。
- 加固 `useWorkspacePracticePlan`、`useWorkspaceResume`、`useStartPractice` 的 404 / archived / replacement 行为。
- 补充 `WorkspaceModalIntegration.test.tsx`、App route hydration、auth auto-start、archived plan replacement 与 full workspace/modal pixel parity。
- 加固 P0.018-P0.021 trigger/verify scripts，并把 plan/checklist/BDD checklist 状态更新到 `v1.1 completed`。

## 6 经验教训

- 完成态 plan 的 L2 复核必须以 App runtime path 为准，不能把 hook/组件局部测试等同于正式 route 行为。
- 用户可见 UI plan 的 BDD checklist 必须与可执行 trigger/verify 断言一起收口；历史 PASS 或弱脚本不能作为完成证据。
- route context、modal、auth pendingAction、pixel parity 这类跨层行为应至少各有一个真实 integration gate。
