# BUG-0007: AI provider contract retained gateway terminology

> 模块: platform | 严重度: medium | 状态: resolved
> 发现日期: 2026-05-05 | 关联 Commit: `fix(ai-provider): remove gateway terminology from provider contract`
> 关联日志: [2026-05-05](../work-journal/2026-05-05.md)

## 1 现象

- 用户在 A3 L2 review 中指出当前项目只关心 AI provider，但 active spec、plan、runtime config、Model Profile schema、A4 env 字典和 generated owner-boundary 注释仍传播 gateway 口径。
- 直接影响范围包括 `docs/spec/ai-provider-and-model-routing/`、`backend/internal/ai/aiclient/`、`backend/internal/platform/config/`、`config/ai-profiles/`、`deploy/dev-stack/`、`shared/conventions.yaml` 与 generated Go / TS AI vocabulary。
- 原有 `env_dict` gate 已通过，说明门禁只保护旧 env 字典一致性，不能发现旧术语已经成为错误真理源。

## 2 诊断过程

### 假设 1：只是少量代码命名遗漏

- 验证方法：对 backend/config/deploy/active docs 反向搜索 `gateway`、旧 env key、旧 Go 字段和旧 profile schema key。
- 结果：排除。旧口径同时存在于 active spec、ADR、plan、runtime config、profile schema、deploy docs、tests 与 generated artifacts，不是局部遗漏。

### 假设 2：spec 本身仍在驱动旧实现

- 验证方法：读取 A3 spec、001 plan/checklist、A4 env dictionary 与 AIClient 配置代码，对比用户确认的 provider-only 目标。
- 结果：确认。代码基本按旧 spec 落地；必须先原地修订 spec/plan/checklist，再改实现和 gate。

## 3 根因分析

- 根本原因：A3/A4 文档真理源把 AI provider 连接误建模为 gateway 连接，导致代码、配置、fixtures、generated comments 与 deploy 文档一致地实现了错误抽象。
- 涉及路径：
  - `docs/spec/ai-provider-and-model-routing/spec.md`
  - `docs/spec/ai-provider-and-model-routing/plans/001-aiclient-and-profile-bootstrap/plan.md`
  - `backend/internal/ai/aiclient/config.go`
  - `backend/internal/ai/aiclient/profile.go`
  - `backend/internal/platform/config/bindings.go`
  - `scripts/lint/ai_provider_terminology.py`
- 之前没有发现，是因为既有 gate 只验证 `.env.example` / A4 spec / code 之间的结构一致性，没有对 active provider terminology 做负向语义检查。

## 4 修复方案

- 选择方案：按用户确认的方案 A 做硬切换，不保留旧 env key、旧 schema key或旧 Go API 字段兼容层。
- 主要修改：
  - A3 subject 与 ADR-Q6 文件更名为 `ai-provider-and-model-routing` / `ADR-Q6-ai-provider-and-model-routing.md`。
  - runtime config 切到 `AI_PROVIDER_BASE_URL` / `AI_PROVIDER_API_KEY`、`ProviderBaseURL` / `ProviderAPIKey` / `ErrMissingProviderConfig`、`ai.provider*`。
  - Model Profile schema 切到 `route` / `Route`，同步 fixtures、loader tests、adapter contract tests 与 meta builder。
  - 新增 `scripts/lint/ai_provider_terminology.py` 与 contract tests，并接入 `make lint-config`。
  - 同步 A2/A3/A4/B1/roadmap active docs、INDEX 与 lifecycle 状态。

## 6 经验教训

- 术语类修订不能只做字符串替换；必须先确认 active spec/plan 是否本身已经是旧语义真理源。
- 对“旧口径不得复发”的要求，应补成 repo-tracked 负向 lint，而不是只把本轮 `rg` 输出写进 checklist。
- 历史 work journal / report / bug 记录应作为例外处理，active docs 和代码不能继续引用历史口径作为当前事实。
