# BUG-0016: runtime topology lint missed scripts and raw producer fields

> 模块: platform | 严重度: medium | 状态: resolved
> 发现日期: 2026-05-07 | 关联 Commit: `fix(runtime): harden worker topology scripts gate`
> 关联日志: -

## 1 现象

- Code review 发现 `make lint-runtime-topology` 虽已覆盖 active docs、runtime code、config、deploy 与 generated contract，但没有扫描 `scripts/` 工具脚本。
- 同一 review 还发现 producer 旧口径正则只覆盖 `` `worker` producer`` 等自然语言形式，没有覆盖 truth source / generated artifact 常见的 `producer: worker` 和 `"producer": "worker"` 字段形态。
- 影响范围包括 `scripts/lint/env_dict.py`、`scripts/lint/getenv_boundary.go` 这类 worker allowlist / env key tooling，以及 `shared/events.yaml`、event baseline/schema/fixture 中的 producer 字段回流。

## 2 诊断过程

### 假设 1：`scripts/` 没有进入 runtime topology 扫描面

- 验证方法：在 `scripts/lint/runtime_topology_test.py` 新增 fixture，让 `scripts/lint/env_dict.py` 写回 `backend/cmd/worker`，让 `scripts/lint/getenv_boundary.go` 写回 `WORKER_LISTEN_ADDR`。
- 结果：确认。Red 阶段测试失败，stderr 没有 `scripts/lint/env_dict.py` 或 `scripts/lint/getenv_boundary.go`。

### 假设 2：raw producer 字段没有被 `worker producer enum` 正则命中

- 验证方法：在同一 Red fixture 中写入 `shared/events.yaml` 的 `producer: worker` 和 `shared/events/baseline/events.v1.json` 的 `"producer":"worker"`。
- 结果：确认。旧正则只匹配反引号或斜杠包裹的 worker 口径，不匹配 YAML / JSON 字段。

### 假设 3：扩大扫描面会误伤 runtime topology lint 自身

- 验证方法：在 allowlist fixture 中创建 `scripts/lint/runtime_topology.py`，包含 lint 定义所需的 non-current pattern 字面量。
- 结果：确认需要显式例外。`runtime_topology.py` 自身必须允许保留匹配字面量，tests 继续由 `_test.py` 后缀排除。

## 3 根因分析

- 根本原因是 `runtime_topology.py` 的扫描面和 false-negative fixture 仍围绕上一轮暴露的 active docs 形态设计，没有把 tooling scripts 作为 active code handoff surface。
- 涉及路径：
  - `scripts/lint/runtime_topology.py`: `SCAN_PATHS` 缺少 `scripts`，`TEXT_SUFFIXES` 缺少 `.py`，`cmd/worker` 正则漏掉 `backend/cmd/worker` 和引号终止形式。
  - `scripts/lint/runtime_topology.py`: `worker producer enum` 只匹配反引号 / 斜杠包裹的 worker，不匹配 `producer: worker` 或 `"producer": "worker"`。
  - `scripts/lint/runtime_topology_test.py`: 旧 Red fixture 没覆盖 scripts tooling 或 raw producer 字段。
- 之前没有发现，是因为 `make lint-events` 能覆盖当前事件 truth source，但 `make lint-runtime-topology` 自身声称覆盖 generated contract handoff 和旧 producer 回流，却没有用同一 gate 证明这些 raw 字段形态。

## 4 修复方案

- 将 `scripts/` 纳入 `runtime_topology.py` 扫描范围，并把 `.py` 加入文本后缀。
- 对 `scripts/lint/runtime_topology.py` 本身增加显式例外，保留 lint 定义所需的 non-current pattern 字面量；测试文件仍按后缀排除。
- 扩展 `cmd/worker` 正则，覆盖 `backend/cmd/worker` 与引号终止形式。
- 扩展 `worker producer enum` 正则，覆盖 `producer: worker`、`"producer": "worker"` 和 worker-before-producer 两类语序。
- 在 `runtime_topology_test.py` 增加 scripts/raw producer Red-Green fixture，并原地修订 `backend-runtime-topology/001-worker-consolidation` v1.3 Phase 4.5。

## 5 验证

- `python3 -m unittest scripts.lint.runtime_topology_test`
- `make lint-runtime-topology`
- `make lint`

## 6 经验教训

- 负向 lint 的扫描面必须包括曾经承载旧 allowlist / drift gate 的 tooling scripts，否则代码门禁自身会成为旧口径回流点。
- 对 contract truth source 的负向搜索不能只覆盖自然语言 handoff；还要覆盖 YAML / JSON / generated artifact 的原始字段形态。
- 每次修 false-negative 时，Red fixture 应同时覆盖“扫描范围漏项”和“pattern 漏项”，并为 lint 自身保留显式、最小例外。
