# BUG-0194: Frontend redeploy omitted an unmerged resume preview fix

> 模块: frontend | 严重度: medium | 状态: resolved
> 发现日期: 2026-07-20 | 关联 Commit: `feat(resume): add drag and drop uploads (BUG-0194)`
> 关联日志: [2026-07-20](../work-journal/2026-07-20.md)

## 1 现象

- 在文件上传交互修订并重新部署前端后，PDF 简历详情再次出现超宽纸面和 canvas 周围大块空白。
- 上传页的新文案和交互存在，但上一轮已经验收过的 A4 PDF/Markdown 预览几何消失。
- 影响当前本地验收环境中的简历只读详情；数据、PDF source、Markdown 正文和后端接口没有损坏。

## 2 诊断过程

### 假设 1：上传页 CSS 修改覆盖了 A4 selector

- 验证方法：检查当前 `screens.css`、`PdfPageStackPreview.tsx` 和合并基线 diff。
- 结果：当前分支确实仍有 `1150px` 纸宽、Markdown `min-height` 和 canvas 内联展示尺寸，但上传 CSS 只修改 create-flow selector，没有直接覆盖详情 selector。

### 假设 2：A4 修复没有进入当前部署分支

- 验证方法：比较 `main`、当前 create-flow 分支与 `fix/resume-a4-preview-0720` 的提交祖先关系和文件差异。
- 结果：确认。A4 修复只存在于独立提交 `5a38e1938`，当前 create-flow 分支从其父提交派生；重新部署因此忠实加载了不含 A4 修复的旧详情实现。

## 3 根因分析

- 两项并行的用户可见改动分别停留在独立本地分支，上传分支在 A4 修复合并前就从 `main` 派生。
- 前端 redeploy 只保证部署当前 checkout，不会自动整合 sibling branch；因此部署成功反而暴露了分支集成缺口。
- 两个 owner 都有各自 focused gate，但当时没有在同一交付分支上同时检查 create dropzone 与 detail A4 geometry。

## 4 修复方案

- 将 `fix/resume-a4-preview-0720` 以 fast-forward 方式合并到当前 create-flow 分支，再恢复上传改动。
- 冲突解析后把 subject spec 升为 2.27：`D-20` 保留 A4 preview geometry，`D-21` 承接 create upload drag/drop；001/002 owner 和 INDEX 投影同时保留。
- 在同一工作树运行 Resume Workshop 全量 owner tests、typecheck/build、根 `make test`、两个 context 与 docs/index gates；真实 Chrome 同时读取上传区和 PDF/Markdown desktop/mobile 几何。

## 5 验证

- Resume Workshop：20 files / 124 tests PASS；其中 upload drag/drop 与 A4 CSS/PDF canvas regression 同时存在。
- 根 `make test`：Python 621 tests / 4628 subtests、Go 全包、frontend 137 files / 1125 tests PASS；typecheck/build PASS。
- Chrome desktop：上传区 legacy icon count 为 0、横向溢出为 0；PDF page `794×1123px`、canvas `792px` 且无 inline style；Markdown 为单个 `794×2003px` 连续页面。
- Chrome 390×844：上传区宽 `328px`，PDF page `358×506px`，Markdown 单页宽 `358px`，三者横向溢出均为 0；控制台 warning/error 为 0。
- Chrome 扩展的本地文件注入仍受权限限制，未把有效文件 live handoff 标记为 PASS；002 plan 保持 active 并保留未勾选验收项。

## 6 经验教训

- `redeploy PASS` 只证明当前 checkout 被正确部署，不证明 sibling feature/fix branch 已被整合。
- 同一页面族存在并行 UI 改动时，回到 `main` 前应在最终集成分支执行跨 owner 组合 gate，并明确检查每个已验收用户不变量仍存在。
- 生命周期和 INDEX 冲突必须合并为下一版本，不能选择任一分支版本覆盖另一边的决策或证据。
