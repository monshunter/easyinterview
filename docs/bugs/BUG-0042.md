# BUG-0042: resume tailor mode enum drifted across event consumers

> 模块: schema | 严重度: medium | 状态: resolved
> 发现日期: 2026-05-12 | 关联 Commit: `fix(events): align resume tailor mode contract`
> 关联日志: [2026-05-12](../work-journal/2026-05-12.md)

## 1 现象

- Resume Workshop additive plans 要求 `ResumeTailorMode` 使用 `gap_review` / `bullet_suggestions`。
- 当前事件契约仍保留旧值 `inline` / `rewrite` / `mirror`，并传播到 `shared/events.yaml`、event baseline、JSON Schema ref、Go generated events 与 TS generated events。
- OpenAPI 004 与 DB 002 需要把 B3 D-14 作为前置信号；若不先修复，后续 schema / persistence 会建立在旧事件语义上。

## 2 诊断过程

### 假设 1：只有文档措辞漂移

- 验证方法：检查 `shared/events.yaml` 与 generated events。
- 结果：排除。旧值已经存在于 truth source 和 generated artifacts。

### 假设 2：事件 truth source 与 generated consumers 同时漂移

- 验证方法：运行事件 inventory lint、Go/TS focused tests，并对旧字面量执行负向搜索。
- 结果：确认。修复需要同步 truth source、baseline manifest、JSON Schema ref、Go/TS generated files、lint expected values 与 type-narrowing tests。

## 3 根因分析

- 根本原因是 Resume Workshop 新语义已经在 plan/spec 中收敛，但 B3 event contract 的 enum truth source 仍停留在旧 tailor mode 口径。
- 涉及路径：
  - `shared/events.yaml`
  - `shared/events/baseline/events.v1.json`
  - `shared/events/refs/ResumeTailorMode.json`
  - `backend/internal/shared/events/events.go`
  - `frontend/src/lib/events/events.ts`
  - `scripts/lint/events_inventory.py`
- 之前没有发现，是因为旧 enum 值仍能通过生成器与类型检查；缺少针对 non-current 字面量的负向搜索和 type-narrowing 断言。

## 4 修复方案

- 将 `ResumeTailorMode` 从 `inline` / `rewrite` / `mirror` 改为 `gap_review` / `bullet_suggestions`。
- 同步 event baseline、JSON Schema ref、Go/TS generated artifacts、inventory lint expected values 与 Go/TS tests。
- 在 DB 002 与 OpenAPI 004 checklist 中记录 B3 D-14 已落地前置信号，防止下游继续引用旧语义。

## 5 验证

- `make lint-events`
- `make codegen-events && make lint-events`
- Go focused event tests
- frontend focused event tests
- 旧 `inline` / `rewrite` / `mirror` 字面量负向搜索
- `python3 scripts/lint/openapi_inventory.py openapi/openapi.yaml`
- `python3 .agent-skills/sync-doc-index/scripts/sync-doc-index.py --check`

## 6 经验教训

- Cross-layer enum 替换不能只依赖 codegen idempotency；需要同时做 non-current 字面量负向搜索。
- 对事件枚举这类 consumer-facing contract，新增 plan 应把 generated Go/TS type-narrowing test 作为完成证据。
