# BUG-0124: resume Phase 8 flat gate drifted on current profile fields

> 模块: test | 严重度: medium | 状态: resolved
> 发现日期: 2026-06-14 | 关联 Commit: `fix(test): close resume phase8 flat gate drift (BUG-0124)`
> 关联日志: [2026-06-14](../work-journal/2026-06-14.md)

## 1 现象

- `/plan-code-review frontend-resume-workshop/003-branch-rewrites-and-edit frontend --fix` 收口 D-20 Phase 8 时，P0.084 / P0.086 / P0.087 wrapper 的 non-current grep 仍把 generic `experiences` 当作旧口径。
- 当前 D-20 flat resume 允许 `structuredProfile.experiences[]` 作为 backend parse historical shape 的输入兼容，且 BUG-0123 regression 正在要求 accepted rewrites 写回 `experiences[].bullets`。
- P0.084 首次运行失败在 verify 阶段，命中 `ResumeDetailView.tsx` 的合法 `["sections", "experience", "experiences", "projects"]` merge allowlist，而不是旧 route / old API regression。
- Owner plan / BDD docs 仍保留 Phase 1-7 historical branch/version 口径，Phase 8 缺少 flat profile、route `targetJobId` 和 wrapper zero-reference 的当前验收项。

## 2 诊断过程

### 假设 1：runtime 又恢复了旧 branch/version API

- 验证方法：反查 `ResumeWorkshopScreen.tsx`、`ResumeDetailView.tsx`、`ResumeRewritesTab`、generated client consumer 和 P0.084 trigger log。
- 结果：排除。Runtime 仍走 flat `resumeId` detail，rerun request body 为 `{ resumeId, targetJobId, mode }`，P0.084 trigger log 中 5 files / 46 tests 已覆盖 no old ids、omitted `structuredProfile` fallback 和 flat bullets merge。

### 假设 2：scenario non-current grep 把当前合法 flat profile 字段误判为旧模块

- 验证方法：运行 P0.084 `setup -> trigger -> verify`，读取 verify 失败位置和命中的源文件行。
- 结果：确认。失败来自 verify 脚本中的 `experiences` literal；该字段现在是 accepted rewrites merge 的合法目标，不应被当成 non-current Experiences module marker。

### 假设 3：Phase 8 文档 gate 没有把 BUG-0123 类 profile/context regression 固化

- 验证方法：读取 `plan.md`、`checklist.md`、`bdd-plan.md`、`bdd-checklist.md` 和 `context.yaml`，比对 D-20 product-scope / current runtime。
- 结果：确认。文档仍过度强调 branch/version historical phase，缺少 Phase 8 current owner gate、operation matrix、BDD scenario semantics 和 context discovery keywords。

## 3 根因分析

- 根本原因：D-20 简历扁平化后，旧模块负向搜索没有从“宽泛词面”收敛到“精确 non-current identifier”。`experiences` 既可能是已丢弃的独立 Experiences module shorthand，也可能是当前合法 flat profile field；wrapper 没有区分二者。
- 涉及路径：
  - `test/scenarios/e2e/p0-084-resume-branch-flow-three-seed-strategies/scripts/verify.sh`
  - `test/scenarios/e2e/p0-086-resume-suggestion-accept-reject-edit-and-update-version/scripts/verify.sh`
  - `test/scenarios/e2e/p0-087-resume-detail-export-copy-consistency-and-parity/scripts/verify.sh`
  - `test/scenarios/e2e/p0-087-resume-detail-export-copy-consistency-and-parity/README.md`
  - `docs/spec/frontend-resume-workshop/plans/003-branch-rewrites-and-edit/`
- 之前没有发现，是因为 Phase 8 checklist 未把 BUG-0123 的 flat profile/context regressions 写成剩余验收 gate；historical completed phase 和旧 PASS 记录掩盖了当前 wrapper 语义 drift。

## 4 修复方案

- 将 P0.084 / P0.086 / P0.087 verify 脚本中的 non-current grep 从 generic `experiences` 收紧为 `ExperiencesScreen|experiences-route`，继续拒绝旧独立模块，但允许当前 `structuredProfile.experiences[]`。
- 更新 P0.087 README，明确 D-20 `structuredProfile.experiences[]` 是合法 flat profile field，不属于 non-current Experiences module。
- 重写 plan / BDD plan 的 Phase 8 current semantics，将 branch/version historical phase 与 D-20 flat route/detail/Rewrites/Edit owner gate 分开。
- 在 checklist / BDD checklist 中固化 BUG-0123 类验收：flat profile `sections` / `experience` / `experiences` / `projects` merge、omitted profile fallback、route `targetJobId` rerun body、accept-only save modal、non-current branch/version zero-reference 和 P0.084-P0.087 wrapper PASS。
- 更新 `context.yaml` discovery keywords / operation matrix，让后续 `/plan-code-review` 能命中 current flat operations 和 targetJob context，而不是旧 branch/version API。

## 6 经验教训

- Non-current grep 应使用精确旧 identifier / route marker；对于可能成为当前 schema 字段的普通名词，必须先按 current truth source 分类，不能直接加入零命中词表。
- D-20 flat profile gate 必须覆盖 `experience` / `experiences` / `projects` 等真实输入形态，否则容易把 BUG-0123 类用户可见保存回归漏到历史完成项之外。
- Phase 8 这种历史重塑收口必须把 current owner gate 写回 plan / BDD checklist，不能只依赖旧 Phase PASS 或旧 wrapper 文案。
