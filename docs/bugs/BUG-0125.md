# BUG-0125: resume archive persistence and P0.102 gate drift escaped review

> 模块: schema | 严重度: medium | 状态: resolved
> 发现日期: 2026-06-15 | 关联 Commit: `fix(review): persist resume archive and p0102 gate (BUG-0125)`
> 关联日志: `-`

## 1 现象

- Review 发现 `archiveResume` route 返回 `status=archived`，但 service 只修改响应 DTO，不写入 store。
- 后续 `GET /resumes/{resumeId}` / `listResumes` 仍能读取同一 active row，使用不同 IK 的第二次 archive 也无法产生 fixture 中声明的 already-archived `409`。
- `E2E.P0.102` README 已声明 retired `jd_match` 不再作为独立保护 route，但 `scripts/trigger.sh` / `scripts/verify.sh` 仍引用已删除的 `TestJDMatchRoutesRequireSessionOnAllRoutes`，场景会因旧 PASS marker 不存在而失败。

## 2 诊断过程

### 假设 1：archive endpoint 只缺 handler error mapping

- 验证方法：读取 `backend/internal/resume/handler/archive.go`、`backend/internal/resume/service.go` 与 store 接口。
- 结果：排除。handler 已调用 service，但 service 只通过 `ReadStore.Get` 读取 active row 后手动设置 response status，没有持久化调用点。

### 假设 2：当前 schema 已有独立 archive 状态列

- 验证方法：反查 `docs/spec/backend-resume/spec.md`、`migrations/000001_create_baseline.up.sql`、`migrations/000015_resume_flatten.up.sql` 与 `backend/internal/resume/store/assets.go`。
- 结果：排除。当前 active spec/store 字段只声明 `deleted_at` 作为 active filter，未声明 `archived_at` 或独立 resume status 列；本修复不新增 schema。

### 假设 3：P0.102 wrapper 只需改 README

- 验证方法：直接读取 `test/scenarios/e2e/p0-102-auth-gated-interview-routes/scripts/trigger.sh` 与 `verify.sh`，并运行 setup/trigger/verify/cleanup。
- 结果：排除。trigger/verify 同时引用旧测试名；修复后 trigger 真实前端测试数为 26，verify 也需要同步当前 runner marker。

## 3 根因分析

- `Service.ArchiveResume` 将 archive 当作响应投影处理，没有定义 archive store interface，也没有调用持久化更新。
- Store 层缺少“active row -> soft-hidden row”和“already archived -> conflict”的可执行测试，导致 endpoint success 与 DB state 脱节。
- JD Match 退役后，P0.102 的脚本级 gate 未同步到当前负向测试 `TestJDMatchRoutesAreGonePerD17`，owner checklist 也保留了旧测试名和旧 frontend test count。

## 4 修复方案

- 新增 `resumestore.ArchiveResumeInput`、`Repository.ArchiveResume` 与 `ErrAlreadyArchived`，通过现有 `deleted_at`/`updated_at` 持久化 soft-hide；active `Get`/`List` 继续借助既有 `deleted_at is null` filter 隐藏 archived resume。
- 将 `resume.Service.ArchiveResume` 改为调用 `ArchiveStore`，并把 store 的 `ErrAlreadyArchived` 映射到 service 层错误。
- 将 archive handler 的 already-archived 错误映射为 `409 TARGET_INVALID_STATE_TRANSITION`，message 对齐 fixture 的 `Resume is already archived`。
- 更新 `archiveResume` fixture 默认响应的 `deletedAt`，使 fixture 与当前持久化语义一致。
- 更新 P0.102 trigger/verify/README 与 frontend-shell owner checklist：使用 `TestJDMatchRoutesAreGonePerD17`，并把 frontend runner 证据更新为 `3 files / 26 tests`。

## 6 经验教训

- Side-effect endpoints must prove persistence with store-level tests, not only response DTO projection tests.
- Scenario wrappers must be treated as executable evidence assets: when a focused test is retired or renamed, both trigger and verify scripts must move to the current PASS marker and reject no-op evidence.
