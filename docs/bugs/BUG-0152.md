# BUG-0152: home and workspace quick-start dropped target context

> 模块: frontend | 严重度: medium | 状态: resolved
> 发现日期: 2026-07-09 | 关联 Commit: `fix(frontend): preserve quick-start target context (BUG-0152)`
> 关联日志: `-`

## 1 现象

- Review of `fix/frontend-round-data-0709` found that Home recent and Workspace list quick-start buttons started practice without the structured `roundId/roundName` selected from `TargetJob.summary.interviewRounds[]`.
- Home recent jobs also requested `listTargetJobs` without `analysisStatus=ready`, so queued / processing TargetJobs with a bound resume could appear as startable interview cards.
- Impact: users could start a practice session from shortcut cards with incomplete round context, or see non-ready TargetJobs in the Home recent surface.

## 2 诊断过程

### 假设 1：shared TargetJob round mapper was not available to quick-start paths

- 验证方法：Compared `ParseScreen` start handoff with Home and Workspace quick-start code.
- 结果：确认。Parse used `interviewContextFromTargetJob`, but Home and Workspace local param builders only returned `targetJobId`, optional `planId`, and optional `resumeId`.

### 假设 2：Home recent relied on backend default list admission

- 验证方法：Read `useRecentTargetJobs` and backend list semantics from the generated client call contract.
- 结果：确认。Home only sent `pageSize=12` and did not defensively filter non-ready or blank-title records, while Workspace already requested `analysisStatus=ready`.

## 3 根因分析

- Home and Workspace had duplicated local route-param builders instead of using a shared TargetJob practice-route projection.
- Existing quick-start tests asserted plan/resume/session IDs, but did not assert structured `roundId/roundName`.
- Workspace had ready admission tests; Home recent did not assert the same ready-only list contract.

## 4 修复方案

- Added shared helpers in `frontend/src/app/navigation/interviewContext.ts`:
  - detail route params for parse card-body navigation
  - practice route params with default practice settings and structured `roundId/roundName`
  - startable TargetJob guard requiring ready status, bound resume, and structured round context
- Updated Home recent to request `listTargetJobs(analysisStatus=ready)`, filter non-ready / blank-title records, and use the shared practice params for quick-start.
- Updated Workspace list quick-start to use the same shared practice params while keeping card-body navigation on detail params.
- Updated Parse start handoff to reuse the same practice-route helper.
- Added focused regression assertions for ready filtering and structured round params.
- Updated owner plan/checklist contracts for Home and Workspace.

## 6 经验教训

- Shortcut start paths must assert the same route context fields as the primary Parse start path, especially structured round context.
- When a list surface is used as a startable action source, tests should lock both backend query admission and UI-level defensive filtering.
