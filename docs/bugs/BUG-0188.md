# BUG-0188: practice launch waited for the opening LLM without transition feedback

> 模块: frontend | 严重度: medium | 状态: resolved
> 发现日期: 2026-07-18 | 关联 Commit: `fix(frontend): add interview launch transition (BUG-0188)`
> 关联日志: [2026-07-18](../work-journal/2026-07-18.md)

## 1 现象

- 用户从 Home 最近面试、Workspace 列表/详情或 Report 复练/下一轮进入面试时，入口会等待 opening LLM 请求完成后才导航到 `/practice`。
- 等待期间原页面只表现为按钮禁用，没有全屏进度反馈，数秒内看起来像页面卡死。
- 影响所有复用 `startPracticeFromParams` 的正式面试启动入口；简历解析和报告生成已有明确等待态，但该段 pre-session 生命周期没有等价反馈。

## 2 诊断过程

### 假设 1：Practice 页面 skeleton 可以覆盖等待时间

- 验证方法：反查四类 caller、共享 `startPracticeFromParams` 以及导航时序。
- 结果：排除。opening LLM 请求在导航前完成，Practice 页面尚未挂载，因此会话内 skeleton 无法覆盖这段等待。

### 假设 2：问题只存在于 Home 快捷入口

- 验证方法：搜索 `startPracticeFromParams` 的生产调用方，并逐一冻结 start Promise 观察 DOM。
- 结果：排除。Home recent、Workspace list、Workspace detail、Report replay/next-round 均复用同一启动链路，也都缺少统一等待反馈。

## 3 根因分析

- 现有 spec 只写了会话页面的 conversation skeleton，没有定义“点击启动后、导航发生前”的 pre-session 状态。
- 共享启动 helper 正确承接了 plan/session/opening message 的顺序，但视觉反馈仍由各 caller 自己管理；各入口只有局部 `starting`/`confirming` 状态和按钮禁用，没有共享的全屏 transition owner。
- 既有 caller tests 只断言重复点击被阻止和成功导航，没有冻结 opening 请求并断言 pending 期间的可见反馈与背景交互阻断。

## 4 修复方案

- 新增共享 `PracticeLaunchTransition`，使用 portal 提供全屏 indeterminate 动画、诚实等待文案、`role=status`、`aria-live=polite` 与 `aria-busy=true`。
- transition 展示时锁定 body 滚动，并为背景 sibling 设置 `inert` 与 `aria-hidden`；卸载时恢复原属性、滚动和焦点。
- Home recent、Workspace list/detail、Report replay/next-round 复用各自既有 in-flight 状态渲染同一 transition，不修改 OpenAPI、后端启动顺序或持久化契约。
- Report 启动失败补充安全的本地化错误提示；reduced-motion 下停止装饰动画，不伪造百分比或 opening 内容。

## 5 验证

- TDD RED 在四类 caller 上复现 transition 缺失，并覆盖 Report rejection；GREEN 后共享组件与 caller focused tests 为 5 files / 45 tests PASS，frontend typecheck PASS。
- 根 `make test` 通过 Python 584 tests/4583 subtests、Go 全包与 frontend 127 files/1035 tests；`make lint`、frontend production build、`make docs-check`、index/context/diff gates 通过。
- 使用 Chrome skill 连接正式本地 frontend/backend，在真实 opening LLM pending 期间验证 1440×900 与 390×844：transition 可见且 fixed/inset=0，移动端无横向溢出；桌面背景 sibling 均为 inert/aria-hidden，body scroll 被锁定。
- 等待结束后 transition 自动关闭并进入 `/practice`，面试官 opening prompt 正常显示。

## 6 经验教训

- 长耗时命令的等待态必须按真实生命周期放置；会话页内 skeleton 不能替代导航前的异步反馈。
- 共享 command helper 的 caller matrix 应同时覆盖 pending、success 和 failure，且 pending 测试要冻结 Promise，而不是只检查最终导航。
- 本次模式已固化到原 owner spec、Phase 30 plan/checklist 与 BDD gate，无需新增通用 Bug pattern。
