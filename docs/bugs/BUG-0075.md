# BUG-0075: resume branch rewrites L2 gates were false-green

> 模块: frontend | 严重度: medium | 状态: resolved
> 发现日期: 2026-05-18 | 关联 Commit: `fix(frontend-resume-workshop): close branch rewrites L2 gates (BUG-0075)`
> 关联日志: [2026-05-18](../work-journal/2026-05-18.md)

## 1 现象

- `frontend-resume-workshop/003-branch-rewrites-and-edit` 已标记 `completed`，但 L2 review 发现 `frontend/tests/pixel-parity/resume-workshop-branch-rewrites-edit.spec.ts` 不存在。
- P0.087 scenario 的 `trigger.sh` 只运行 Vitest，`verify.sh` 也只检查 Vitest marker 与 grep gate，未执行 frontend build、Playwright pixel parity 或 axe-core。
- Checklist 7.2 / 7.4 文本要求 Playwright / axe PASS，但 evidence 注释仍写 deferred；7.5 声称 Rewrites / Edit Tab 顶 header 保留 Export PDF / Copy Text，实际 runtime 只在 Preview tab 渲染这些按钮。

## 2 诊断过程

### 假设 1：Playwright spec 已存在但未被 scenario 调用

- 验证方法：检查 `frontend/tests/pixel-parity/` 与 P0.087 `scripts/trigger.sh` / `scripts/verify.sh`。
- 结果：排除。目标 spec 文件缺失；P0.087 trigger/verify 没有任何 Playwright runner marker。

### 假设 2：Export PDF / Copy Text 由 detail header 统一渲染

- 验证方法：阅读 `ResumeDetailView.tsx` 和 `ResumePreviewTab.tsx`，新增失败用例断言 Rewrites / Edit tabs 可见 `resume-detail-export-pdf` 与 `resume-detail-copy-text`。
- 结果：排除。按钮只在 `ResumePreviewTab` 内部渲染，Rewrites / Edit tab 没有对应 header actions。

### 假设 3：浏览器 parity gate 会暴露额外 UI 漂移

- 验证方法：补 Playwright spec 后运行 desktop + mobile。
- 结果：确认。补测先暴露 `ui-design` font token alias 缺失导致 `var(--ei-serif)` 回退，以及 mobile BranchFlow 两列/三列网格导致水平溢出。

## 3 根因分析

- 根本原因：plan/checklist completion 复用了 Vitest DOM anchor 和历史说明，把 Playwright / axe / mobile viewport gate 当作已完成证据，但 scenario wrapper 没有真实执行 browser runner。
- 涉及路径：
  - `docs/spec/frontend-resume-workshop/plans/003-branch-rewrites-and-edit/checklist.md`
  - `test/scenarios/e2e/p0-087-resume-detail-export-copy-consistency-and-parity/scripts/trigger.sh`
  - `test/scenarios/e2e/p0-087-resume-detail-export-copy-consistency-and-parity/scripts/verify.sh`
  - `frontend/src/app/screens/resume-workshop/components/ResumeDetailView.tsx`
  - `frontend/src/app/theme/themes.css`
  - `frontend/src/app/screens/resume-workshop/branch/ResumeBranchFlow.tsx`
- 之前没有发现，是因为 P0.087 verify 只确认 Vitest summary 和 grep gate，没有检查 Playwright runner marker、spec path、pass summary，也没有拒绝 deferred evidence。

## 4 修复方案

- 新增 Playwright pixel parity / axe spec，覆盖 BranchFlow / Rewrites / Edit 的 desktop + mobile DOM anchor、computed style、viewport bounding box、screenshot buffer、scoped axe-core，以及 Rewrites/Edit Export PDF + Copy Text 行为。
- 在 `ResumeDetailView` 的非 Preview tab header 补 Export PDF / Copy Text actions，复用 plan 001 的 export/copy handlers。
- 给 theme root 增加 `--ei-serif` / `--ei-sans` / `--ei-mono` aliases，保证从 `ui-design` 迁移来的 inline token 名称在正式前端可解析。
- 给 BranchFlow 补移动端网格折叠规则，消除 390px viewport 横向溢出。
- 升级 P0.087 trigger/verify：trigger 执行 Vitest + frontend build + Playwright；verify 检查 runner marker、spec path、pass summary，并继续保留 non-current/prototype grep gates。

## 6 经验教训

- Completed plan 的 Playwright / axe / scenario gate 必须直接读 `trigger.sh` 与 `verify.sh`，确认 runner 真实执行，而不是只读 checklist 的历史 PASS。
- Browser parity gate 在补齐后能发现 Vitest DOM anchor 覆盖不到的 token fallback 与 mobile overflow；后续 UI L2 review 应把 mobile viewport 作为必要证据。
- 本 Bug 归入 [模式 4](./PATTERNS.md#模式-4completed-checklist-掩盖未执行的-runner-gate)。
