# BUG-0049: backend upload follow-up review exposed MinIO race and no-op live gate

> 模块: platform | 严重度: medium | 状态: resolved
> 发现日期: 2026-05-13 | 关联 Commit: `fix(backend-upload): close follow-up L2 gaps`
> 关联日志: `-`

## 1 现象

- Review against `checkpoint/008` found that `MinIOStore.clientFor` lazily cached the HTTP client without synchronization.
- E2E.P0.033 was used as evidence for live presign -> object storage PUT -> register -> `DELETE /api/v1/me`, but `trigger.sh` only ran focused Go package tests and did not execute that live HTTP/object-storage roundtrip.
- The first issue could race during concurrent first use of upload presign paths; the second issue could let a BDD gate pass without proving the documented live workflow.

## 2 诊断过程

### 假设 1：`MinIOStore.clientFor` is only called serially

- 验证方法：added `TestMinIOStoreClientForConcurrentFirstUse` and ran it with `go test -race`.
- 结果：排除。The red test reported a data race between concurrent reads and writes of the cached client field.

### 假设 2：E2E.P0.033 already performs a live presign/upload/register/delete roundtrip

- 验证方法：reviewed `test/scenarios/e2e/p0-033-file-presign-register-roundtrip/scripts/trigger.sh` and `verify.sh`.
- 结果：排除。The trigger only ran focused package tests; verify accepted test output without requiring a live roundtrip test name and without rejecting no-op output.

## 3 根因分析

- `MinIOStore.clientFor` treated lazy singleton initialization as a plain pointer assignment, but the upload store can be reached by concurrent HTTP requests.
- The scenario script contract only checked that related packages were tested. It did not require a named integration test that exercised the live `createUploadPresign` HTTP handler, MinIO signed PUT, `RegisterFileObject`, privacy delete API, and privacy drainer.
- The prior skip guard prevented one false PASS shape, but did not prove that the documented end-to-end path was actually invoked.

涉及的代码路径：

- `backend/internal/upload/objectstore/minio.go`
- `test/scenarios/e2e/p0-033-file-presign-register-roundtrip/scripts/trigger.sh`
- `test/scenarios/e2e/p0-033-file-presign-register-roundtrip/scripts/verify.sh`

## 4 修复方案

- Added `sync.Mutex` protection around `MinIOStore.clientFor` lazy client cache initialization.
- Added a race regression test covering concurrent first use of the MinIO client cache.
- Added `TestUploadPresignRegisterPrivacyDeleteLiveRoundtrip` behind the `integration` build tag to exercise live HTTP presign, MinIO signed PUT, internal register, `DELETE /api/v1/me`, and privacy drainer cleanup.
- Updated E2E.P0.033 trigger and verify scripts so the scenario requires the live roundtrip test and rejects missing/no-op evidence.
- Updated the backend-upload plan, checklist, BDD plan, BDD checklist, scenario README, expected outcome, and plan index so the documented gate matches the fixed behavior.

## 5 验证

## 6 经验教训

- Lazy cached runtime clients need an explicit synchronization regression test when they can be reached from HTTP handlers.
- BDD evidence should name the live behavior it proves; a package-level `go test` command is not enough when the acceptance criterion is an end-to-end external dependency roundtrip.
- Scenario verify scripts should reject both skipped tests and no-op outputs.
