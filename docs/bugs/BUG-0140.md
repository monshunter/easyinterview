# BUG-0140: resume detail injected display name into Markdown body and split source preview surfaces

> 模块: materials | 严重度: medium | 状态: resolved
> 发现日期: 2026-07-08 | 关联 Commit: `fix(resume): close source preview and picker regressions (BUG-0139, BUG-0140, BUG-0141)`
> 关联日志: [2026-07-08](../work-journal/2026-07-08.md)

## 1 现象

- 用户复核简历详情截图时发现 Markdown 正文卡片顶部额外出现了详情 header 的 `displayName` / 简历名称。
- 同一详情正文区域里，PDF renderer 使用浅色背景板 + 白色页面栈，而 Markdown renderer 直接使用白底卡片，来源格式之间视觉割裂。
- 影响范围覆盖 `ResumePreviewTab` Markdown 详情、PDF 详情、`ui-design` 原型、CSS parity 和 pixel parity smoke。

## 2 诊断过程

### 假设 1：Markdown 内容本身包含重复标题

- 验证方法：为 `ResumePreviewTab.test.tsx` 构造 `displayName = "Injected Display Name Must Stay In Header"`，同时让 `parsedTextSnapshot` 只包含 `# Candidate Body` 和正文。
- 结果：排除。Red 阶段 DOM 中出现 `<h3 class="ei-text-title">Injected Display Name Must Stay In Header</h3>`，说明名称来自组件额外注入，不是 Markdown body。

### 假设 2：PDF / Markdown UI 差异只是截图缩放造成

- 验证方法：读取 `frontend/src/app/screens/screens.css` 与 CSS parity test。
- 结果：排除。旧 CSS 只有 `.ei-resume-detail-preview-card--pdf` 使用 `background: #f6f3ee; padding: 28px;`，Markdown 分支仍是 `background: #ffffff; padding: 44px 56px;`。

### 假设 3：UI truth source 已经统一，正式前端迁移漂移

- 验证方法：读取 `ui-design/src/screen-resume-workshop.jsx`。
- 结果：确认存在 truth source 与正式前端同步修订需求。原型 Markdown 分支同样把 `resume.name` / `resume.summary` prepend 到正文卡片，且没有与 PDF 共用背景板。

## 3 根因分析

- `ResumePreviewTab` 的 Markdown 分支在渲染 `react-markdown` 前额外输出 `uiResume.name`，把详情 header 元数据混入正文区域。涉及代码路径：`frontend/src/app/screens/resume-workshop/components/ResumePreviewTab.tsx`。
- 来源格式视觉合同只在 PDF 分支落地，PDF 通过 `ei-resume-detail-preview-card--pdf` 获得浅色背景板，Markdown 缺少对应 page surface。涉及代码路径：`frontend/src/app/screens/screens.css` 与 `ui-design/src/screen-resume-workshop.jsx`。
- 之前测试只覆盖 Markdown DOM 语义和 PDF page-stack，不覆盖“Markdown body 不得注入 header metadata”和“PDF / Markdown 共用阅读背景板”这两个回归点。

## 4 修复方案

- `ResumePreviewTab` 取消 Markdown 分支里的 `uiResume.name` / summary prepend，Markdown body 只渲染 `buildResumeBodyMarkdown(resume)` 输出；空正文只显示通用不可读状态，不再用 summary 填充正文。
- 移除 PDF 专属外层 modifier，统一 `ei-resume-detail-preview-card` 为浅色阅读背景板，并新增 `ei-resume-detail-markdown-page` 白色页面容器。
- 同步 `ui-design/src/screen-resume-workshop.jsx`、`docs/ui-design`、`frontend-resume-workshop` spec/plan/checklist/BDD/context 和 CSS/pixel parity tests。

## 6 经验教训

- 正文 renderer 的测试不能只断言内容可见和 Markdown DOM 语义，还必须负向断言 header metadata 不会进入 body 区域。
- 来源格式自适应 renderer 需要共享外层布局合同；否则 PDF 与 Markdown 会分别演进出不同的阅读面。
- UI truth source 与正式前端需要同时加 page-level anchor，避免只有 CSS 截图观感而没有可执行 DOM gate。
