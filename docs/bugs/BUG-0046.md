# BUG-0046: resume workshop hid failed loads and original-source pending state

> 模块: frontend | 严重度: medium | 状态: resolved
> 发现日期: 2026-05-12 | 关联 Commit: `fix(frontend-resume-workshop): harden resume workshop error states`
> 关联日志: `-`

## 1 现象

- L2 review 发现 `listResumes` 成功但 `listResumeVersions` 失败时，列表页仍渲染 StatsStrip / tree / flat，并把版本数量当作 0。
- `getResumeVersion` 非 404 失败时，详情页没有错误态，因为 `data` 为空且 `loading=false` 后仍命中 loading fallback。
- "查看原件" 弹层在 `getResume` pending 或失败时，会把 `structuredProfile` 的 headline/summary 当作原始简历来源展示。

## 2 诊断过程

### 假设 1：版本列表 hook 已暴露 error，但组件没有消费

- 验证方法：阅读 `useResumeVersions` 与 `ResumeListView`，并新增 `listResumeVersions` reject 后的组件测试。
- 结果：确认。hook 只返回 `loading/data/error`，组件用 `versionsQuery.data?.items ?? []` 投影，未检查 `versionsQuery.error`，也没有 retry。

### 假设 2：详情页错误分支顺序导致非 404 被当成 loading

- 验证方法：mock `getResumeVersion` 返回 `HTTP 500`，观察 `ResumeDetailView` 渲染。
- 结果：确认。`notFound` 只覆盖 404；其它 error 下 `data=null`，组件进入 `versionQuery.loading || !versionQuery.data` 分支并持续展示 loading。

### 假设 3：原件弹层 fallback 没有区分 source 请求状态

- 验证方法：mock `getResume` pending 和 reject，检查 modal 内容。
- 结果：确认。`originalSource` 为空时立即回退到 `versionFallbackOriginalText`，没有等待 source 成功，也没有 source error UI。

## 3 根因分析

- 根本原因：Resume Workshop 的 dependent data load 没有把 `pending / error / success-empty` 三种状态分开建模，组件把缺失数据直接折叠成空数组、loading fallback 或 version summary fallback。
- 涉及的代码路径：
  - `frontend/src/app/screens/resume-workshop/components/ResumeListView.tsx`
  - `frontend/src/app/screens/resume-workshop/components/ResumeDetailView.tsx`
  - `frontend/src/app/screens/resume-workshop/components/OriginalResumePreviewModal.tsx`
  - `frontend/src/app/screens/resume-workshop/hooks/useResumeVersions.ts`
  - `frontend/src/app/screens/resume-workshop/hooks/useResumeVersion.ts`
  - `frontend/src/app/screens/resume-workshop/hooks/useResumeAsset.ts`
- 为什么之前没有发现：既有测试覆盖了 happy path、404 和立即成功的 `getResume` 原件来源，但缺少版本列表失败、详情非 404 失败、原件 pending / failed 的负向断言；`listResumes.paginated` 测试还依赖了被吞掉的 missing version scenario failure。

## 4 修复方案

- 为 `useResumeVersions` / `useResumeVersion` / `useResumeAsset` 增加 retry trigger，保留 request sequence guard。
- `ResumeListView` 在已有 asset 且版本集合尚未落定时显示 loading；`listResumeVersions` 失败时显示 retryable error，不再把版本数渲染为 0。
- `ResumeDetailView` 在非 404 detail error 时显示 retryable error；404 仍走原有 NotFound UI。
- `OriginalResumePreviewModal` 增加 `ready / loading / error` 内容状态；`ResumeDetailView` 只有在 `getResume` 成功且 source text 为空时才使用 version summary fallback。
- 补充 en/zh i18n copy 和 focused regression tests。

## 6 经验教训

- Dependent API loads must not use `data ?? empty` unless the component also has an explicit error branch.
- Modal source fallback must be gated by successful source response, not by absence of source data.
- Fixture tests that intentionally exercise one operation should mock adjacent dependent operations when the selected scenario is not present for those operations.
