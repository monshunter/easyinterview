# BUG-0134: home JD source controls over-separated from input card

> 模块: frontend | 严重度: medium | 状态: resolved
> 发现日期: 2026-07-06 | 关联 Commit: `fix(frontend-home): integrate source controls into input card (BUG-0134)`
> 关联日志: [2026-07-06](../work-journal/2026-07-06.md)

## 1 现象

- 用户截图反馈 Home 的粘贴 JD、上传 JD 文件和 URL 导入不应拆成独立右侧上传 panel，而应回到更简约的旧式输入框内整合。
- `home-source-layout` 把 paste 与 upload/source 拆成两个区域后，页面视觉比需求更重，和首页作为“模拟面试新建规划快捷方式”的轻量入口不一致。
- 旧 Phase 10 的 source 分区 gate 已经把错误布局固化为通过条件，导致正式前端、`ui-design` 和 pixel parity 都会接受独立上传 panel。

## 2 诊断过程

### 假设 1：独立上传 source panel 是当前最终设计

- 验证方法：对照用户最新截图反馈、`ui-design/src/screen-home.jsx`、`HomeScreen.tsx`、owner spec Phase 10 与 Home pixel parity。
- 结果：排除。最新需求明确要求“粘帖JD和上传JD文件或者填写URL”在输入框中整合，独立 source panel 属于过度拆分。

### 假设 2：只需要删除右侧 panel，导入行为无需验证

- 验证方法：补 `HomeLayout.test.tsx` 红灯断言 `home-jd-source-controls` 必须位于 `home-jd-input-card` 内，同时保留 upload / URL trigger；运行 Home import、resume selection、auth gate 和 recent mocks 邻近测试。
- 结果：排除。布局回收不能破坏上传文件 presign、URL import modal、ready resume gate、真实 `resumeId` 透传和 `立即面试` 提交顺序。

### 假设 3：只改正式前端即可闭环

- 验证方法：同步检查 `ui-design/` 真理源、`docs/ui-design/`、owner spec / plan / checklist / BDD、P0.014 / P0.015 场景文档、UI contract 和 pixel parity。
- 结果：排除。Phase 10 已经把错误布局写进契约，必须原地修订 owner plan 并补负向断言，避免旧独立 panel 继续被测试接受。

## 3 根因分析

- Phase 10 对截图反馈中的“区分 source 和 submit”理解过重，把上传 JD 文件和 URL 导入拆成独立 source panel，没有保留用户偏好的“输入框内整合”简约形态。
- 测试只锁定了“上传不在 textarea footer 的提交区域”和“提交按钮在简历选择下方”，没有锁定 upload / URL source controls 应归属于 JD input card。
- Owner spec、BDD 和场景文档随 Phase 10 同步为错误结构后，后续实现自然复刻了错误布局，而不是暴露回归。

## 4 修复方案

- 更新 `ui-design/src/screen-home.jsx`：删除 `home-source-layout` 和 `home-upload-source-panel`，在 `home-jd-input-card` 底部新增 `home-jd-source-controls`，承载上传 JD 文件与 URL 导入。
- 更新正式 `HomeScreen.tsx`：复刻输入卡片内整合 DOM，保留 `JDAssistModal`、upload presign、URL import、ready resume gate、pending auth 和真实 `resumeId` handoff。
- 更新 `HomeLayout.test.tsx`、`ui-design` contract 和 Home pixel parity：正向断言 `home-jd-source-controls` 位于输入卡片内，负向断言旧 `home-source-layout` / `home-upload-source-panel` 不再出现。
- 原地修订 owner spec / plan / checklist / BDD、`docs/ui-design/*` 与 P0.014 / P0.015 场景文档，把 Phase 11 固化为当前设计契约。

## 6 经验教训

- 截图反馈里同时出现“简约”和“整合”时，不能只拆解为布局层级，应在 owner plan 中写清保留或删除哪些容器。
- UI gate 需要同时有正向归属断言和负向旧结构断言，否则错误设计一旦进入 plan，会被后续测试稳定放行。
- Home 新建规划入口的轻量感也是契约：JD source actions 可以区别于提交按钮，但不应默认升级成独立 panel。
