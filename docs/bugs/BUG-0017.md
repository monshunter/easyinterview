# BUG-0017: runtime topology lint missed structured producer and owner handoff forms

> 模块: platform | 严重度: medium | 状态: resolved
> 发现日期: 2026-05-07 | 关联 Commit: `fix(runtime): harden worker topology structured gate`
> 关联日志: -

## 1 现象

- L2 code review 发现 `make lint-runtime-topology` 仍会放过 event contract 中跨行 YAML / JSON producer 字段回流，例如 `producer` block、`producer.values` 或 JSON Schema `producer.enum` 中包含 `worker`。
- 同一 review 还发现 `docs/spec/backend-runtime-topology/**` 被整包排除，导致 owner plan/checklist 即使把 `cmd/worker` 写成 current handoff / build / runtime / verification 入口也不会失败。
- 影响范围包括 `shared/events.yaml`、`shared/events/schemas/*.json`、event baseline / fixture 等 contract artifacts，以及 `backend-runtime-topology/001-worker-consolidation` owner handoff 文档。

## 2 诊断过程

### 假设 1：逐行正则无法覆盖结构化 producer block

- 验证方法：创建临时 fixture，让 `shared/events.yaml` 写入 `producer:\n  - worker`，让 `shared/events/schemas/test.v1.json` 写入多行 `"producer": { "enum": ["worker"] }`。
- 结果：确认。旧 lint 返回 `runtime_topology: OK`，因为 `producer` 与 `worker` 分布在不同行，逐行正则没有上下文。

### 假设 2：owner docs 例外过宽导致 current handoff 回流不可见

- 验证方法：创建临时 owner plan，写入 `Current handoff: build backend/cmd/worker as the runtime entry.`。
- 结果：确认。旧 lint 返回 `runtime_topology: OK (0 active files scanned)`，因为 `docs/spec/backend-runtime-topology/**` 被整体排除。

### 假设 3：收窄 owner docs 例外可能误伤负向断言

- 验证方法：为 owner checklist 增加允许 fixture，包含 `删除 cmd/worker`、`WORKER_LISTEN_ADDR 不再作为 current config` 等负向断言；同时保留 current handoff / verification command 失败 fixture。
- 结果：确认需要语义例外。Green 实现只扫描 owner plan/checklist 的 current handoff/build/runtime/verification 句子，并对删除、不得、回流、负向、history/test 等证据词放行。

## 3 根因分析

- 根本原因是 `runtime_topology.py` 的 false-negative gate 仍主要依赖逐行文本正则，未对 YAML / JSON contract surface 使用结构化解析。
- 另一个根因是 owner docs 例外从“允许负向断言”实现成“整包排除”，导致当前 handoff 口径也被排除在 active scan 外。
- 涉及路径：
  - `scripts/lint/runtime_topology.py`: `scan_file` 原先只逐行匹配 `RETIRED_PATTERNS`。
  - `scripts/lint/runtime_topology.py`: `is_excluded_path` 原先整包排除 `docs/spec/backend-runtime-topology/**`。
  - `scripts/lint/runtime_topology_test.py`: 旧 fixture 只覆盖单行 raw producer 字段和 owner negative assertions，没有覆盖跨行结构化字段或 owner current handoff。
- 之前没有发现，是因为 `make lint-events` 能证明当前 truth source 正确，但不能证明 `runtime_topology` 自身会拦截未来跨行 producer 回流；而 owner plan/checklist 历史证据需要保留，导致例外过宽没有被测试压住。

## 4 修复方案

- 在 `runtime_topology.py` 中增加 YAML / JSON 结构化 producer 扫描，覆盖 `producer` key、`name: producer` + `values`、JSON Schema `producer.enum` 等跨行形态。
- 将 owner docs 例外收窄到 owner spec/history 等负向证据；owner plan/checklist 进入扫描，但只针对 current handoff/build/runtime/verification 语境触发，负向断言继续允许。
- 在 `runtime_topology_test.py` 中增加 Red-Green fixture：
  - `shared/events.yaml` producer block / list 中包含 `worker` 必须失败。
  - `shared/events/schemas/*.json` producer enum 中包含 `worker` 必须失败。
  - owner plan/checklist current handoff / verification command 使用 `cmd/worker` 必须失败。
  - owner negative assertions、history、tests、lint 定义继续通过。
- 原地修订 `backend-runtime-topology/001-worker-consolidation` 到 v1.4，新增 Phase 4.6 / 4.7 并同步 spec、history、context 与 INDEX。

## 6 经验教训

- 负向 gate 声称覆盖结构化 contract artifacts 时，Red fixture 必须覆盖 YAML / JSON 的真实多行字段形态，而不是只覆盖单行 grep 形态。
- 例外规则必须证明“允许历史/负向证据”与“拒绝当前执行口径”两个方向；整包排除会让 owner 文档成为 blind spot。
- 修复 false-negative 时，应把扫描范围、pattern 形态、例外语义分别纳入测试，避免上一轮补强只覆盖一个维度。
