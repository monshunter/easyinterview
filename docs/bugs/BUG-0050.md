# BUG-0050: resume workshop list drifted from ui-design tree controls

> 模块: frontend | 严重度: medium | 状态: resolved
> 发现日期: 2026-05-13 | 关联 Commit: `fix(frontend-resume-workshop): restore ui-design list parity`
> 关联日志: `-`

## 1 现象

- 用户对比 `http://localhost:5173/index.html` 的 `ui-design` 简历工坊页面与正式前端 `http://localhost:5173` 后指出：前端简历模块在样式、排版和功能按钮上与原型存在明显距离。
- 正式前端的 TopBar 仍显示 "简历版本"，而 `ui-design` 当前导航为 "简历"；列表页缺少右侧 "新建简历" 主按钮。
- StatsStrip 只展示单层数值，缺少 `2 / 3`、`3 个定制`、最高匹配来源、最近编辑版本说明等原型副文案。
- 树形列表把 "基于这棵树新建版本" 放在每个底稿卡片头部，和原型的 "选为底稿" / 选中后统一分叉操作关系不一致；版本行也缺少缩进、图标、tag、match badge 与 `打开 ->` 的源级排版。
- Locale catalog 仍保留旧 per-row toast key（`resumeWorkshop.tree.toastSelect` / `resumeWorkshop.tree.toastBranch`），使旧交互口径继续存在于 runtime 文案资产中。

## 2 诊断过程

### 假设 1：正式前端只是缺少最新 fixture 数据

- 验证方法：读取 `ui-design/src/screen-resume-workshop.jsx`、`docs/ui-design/resume-module.md`、`ResumeListView.tsx`、`ResumeTreeView.tsx`、`ResumeVersionRow.tsx` 与 i18n 文案。
- 结果：排除。差异不仅是数据数值，而是 DOM 层级、按钮位置、选中状态、行内 icon/tag/match badge、统计副文案和导航文案都偏离了 `ui-design` 源。

### 假设 2：已有 pixel parity / BDD gate 能覆盖这些差异

- 验证方法：运行新增 red tests 前的 focused Vitest，针对 TopBar 文案、header create button、tree per-row new-version button、selected helper、version indent/icon 等断言旧实现。
- 结果：排除。旧 gate 主要覆盖列表能渲染与树/平铺切换，未把功能按钮位置和源级行结构纳入可执行断言。

### 假设 3：现有实现有意把分叉入口设计为每行按钮

- 验证方法：反查 `ui-design` 页面截图和源码中的底稿卡片结构。
- 结果：排除。当前原型是先 `选为底稿`，再在列表上方呈现统一分叉入口；每行固定 "基于这棵树新建版本" 属于旧实现口径。

## 3 根因分析

- 根本原因一：`frontend-resume-workshop/001-listing-routing-and-detail-readonly` 的实现完成后，后续契约变更把导航与简历列表原型收敛到 "简历" / "新建简历" / "选为底稿" 的当前形态，但正式前端没有重新按 `ui-design` 源执行深度 parity。
- 根本原因二：P0.036 和组件测试只覆盖了列表主流程，没有断言关键控件是否出现在原型定义的位置，也没有把选中底稿后的分叉 helper 和旧 toast 文案清理作为行为门禁。
- 涉及代码路径：
  - `frontend/src/app/topbar/TopBar.test.tsx`
  - `frontend/src/app/i18n/locales/zh.ts`
  - `frontend/src/app/i18n/locales/en.ts`
  - `frontend/src/app/screens/resume-workshop/components/ResumeListView.tsx`
  - `frontend/src/app/screens/resume-workshop/components/ResumeTreeView.tsx`
  - `frontend/src/app/screens/resume-workshop/components/ResumeVersionRow.tsx`
  - `frontend/src/app/screens/resume-workshop/components/ResumeWorkshopIcon.tsx`
  - `frontend/src/app/screens/screens.css`
  - `frontend/src/app/scenarios/p0-036-resume-list-tree-flat-toggle.test.tsx`
- 为什么之前没有发现：历史测试把 "能显示列表、能打开详情、能切换视图" 当作主证据，没有把当前 `ui-design` 的功能按钮、行内结构和选中状态作为细粒度 DOM/行为断言。

## 4 修复方案

- 将 TopBar `resume_versions` 中文文案改为 "简历"，并恢复简历工坊 subtitle 的当前原型文本。
- 为 `ResumeListView` 增加 header 级 "新建简历" 主按钮、统计卡副文案、视图上下文文案，以及选中底稿后的统一 helper 和分叉入口。
- 重塑 `ResumeTreeView`：底稿卡片头部改为 `选为底稿` / `已选`，移除每行固定新建版本按钮，补齐 chevron、file icon、语言 pill、source type/date/summary/version count 元信息和上传另一个底稿 CTA。
- 重塑 `ResumeVersionRow`：补齐缩进层级、resume/briefcase icon、`MASTER` / `TARGETED` tag、日期、bullet 统计、采纳数、match badge 与 `打开 ->` 按钮。
- 新增 `ResumeWorkshopIcon` 复用本模块所需的小型图标，避免每个组件内重复手写 svg。
- 删除旧 per-row toast locale key，并用 locale file 负向断言阻止旧口径回流。
- 强化组件测试、TopBar 测试、locale 文件测试和 P0.036 BDD 测试，覆盖 header create button、selected helper、旧 per-row new-version 入口消失、旧 toast key 清理、版本行 icon/indent、导航文案和创建/分叉 navigation。

## 6 经验教训

- UI parity 修复不能只比较页面是否大致可用；功能按钮的所属层级、选中状态和行内结构需要落到组件测试或 BDD gate。
- 对 `ui-design` 已变更过的页面，L2 review 需要反向读取当前源码，而不能沿用历史 completed checklist 的视觉结论。
- 本次已在 P0.036 和组件测试中固化 "选为底稿后统一分叉入口" 与版本行结构断言，暂不更新 `PATTERNS.md`。
