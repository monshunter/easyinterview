# BUG-0123: resume detail accepted rewrites missed flat profile and JD context

> 模块: materials | 严重度: high | 状态: resolved
> 发现日期: 2026-06-14 | 关联 Commit: `fix(resume): close detail rewrite review regressions (BUG-0123)`
> 关联日志: [2026-06-14](../work-journal/2026-06-14.md)

## 1 现象

- Reviewer 指出 `ResumeDetailView` 在 D-20 flat resume detail / rewrites flow 中存在三类用户可见回归。
- 当 flat `getResume` response 省略 `structuredProfile`，且没有 `parsedTextSnapshot` / `originalText` 时，detail 原件 fallback 直接读取 `profile.headline`，页面会因为 `undefined` 访问崩溃。
- 用户采纳改写后选择覆盖或另存，UI 可显示成功 toast，但保存 payload 只改 `structuredProfile.sections[].bullets`，对当前 flat profile 常见的 `experience` / `projects` 以及历史 backend parse 输出 `experiences` 无效。
- 从携带 `targetJobId` 的 detail route 触发 tailor rerun 时，请求 body 只包含 `{ resumeId, mode }`，新的 `resume_tailor` run 丢失 JD 上下文。

## 2 诊断过程

### 假设 1：detail fallback 未对 omitted `structuredProfile` 做默认值

- 验证方法：新增 `ResumeDetailView.test.tsx` 场景，mock `getResume` 返回没有 `parsedTextSnapshot`、没有 `originalText` 且省略 `structuredProfile` 的 flat resume。
- 结果：确认。Red 阶段抛出 `TypeError: Cannot read properties of undefined (reading 'headline')`。

### 假设 2：accepted rewrite 保存只覆盖旧 sections shape

- 验证方法：新增 overwrite 场景，profile 同时包含 `experience`、`experiences`、`projects`，每个 entry 都有匹配原 bullet。
- 结果：确认。Red 阶段 `updateResume` payload 保留原 bullet，未写入 accepted rewrite。

### 假设 3：route `targetJobId` 没有进入 rerun container

- 验证方法：把 rerun 测试 route params 扩展为 `{ resumeId, targetJobId, tab: "rewrites" }`，断言 `requestResumeTailor` body 包含 `targetJobId` 且不恢复 `resumeAssetId` / `resumeVersionId`。
- 结果：确认。Red 阶段 request body 缺少 `targetJobId`。

## 3 根因分析

- `frontend/src/app/screens/resume-workshop/components/ResumeDetailView.tsx` 在原件 fallback 中把 `resume.structuredProfile` 直接 cast 为 record，没有用 `{}` 兜底。
- `ResumeDetailView.tsx` 的 accepted rewrite merge helper 仍基于 pre-D-20 / legacy `sections[].bullets` 形态，没有覆盖 D-20 flat profile 的 `experience` / `projects` 和 backend parse historical `experiences`。
- `frontend/src/app/screens/resume-workshop/ResumeWorkshopScreen.tsx` 已解析并保留 route `targetJobId`，但 `DetailWrapper` 没有传给 `ResumeDetailView` / `ResumeRewritesTabContainer`，导致 `useRequestResumeTailor` 调用无法携带 JD-aware context。
- 之前测试只覆盖了 `sections[].bullets` 和无 `targetJobId` rerun body，缺少 D-20 flat profile shape 与 route context preservation regression。

## 4 修复方案

- 在 `ResumeWorkshopScreen.tsx` 中把 parsed `targetJobId` 透传给 `DetailWrapper` 和 `ResumeDetailView`，并在 rerun request body 中仅当 present 时携带 `targetJobId`。
- 在 `ResumeDetailView.tsx` 原件 fallback 中把 `resume.structuredProfile` 默认成 `{}` 后再读取 headline / summary。
- 将 accepted rewrite merge 扩展为精确处理 `sections`、`experience`、`experiences`、`projects` 中的 `bullets: string[]`，避免宽泛递归误改非 bullet 文本字段。
- 在 `ResumeDetailView.test.tsx` 新增三条 regression coverage，锁定 omitted profile、flat bullet arrays 和 targetJobId rerun。

## 6 经验教训

- D-20 flat resume 前端测试不能只用 `sections` legacy projection；保存类测试必须覆盖当前 `experience` / `projects` shape 和 historical `experiences` 输入。
- 当 route parser 已保留业务上下文 key，detail 子容器 rerun / regenerate 类操作也必须有 body-level context preservation test，避免只在 auth gate 或 root data attribute 层通过。
