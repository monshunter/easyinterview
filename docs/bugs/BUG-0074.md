# BUG-0074: resume create flow retry and L2 gates were false-green

> 模块: frontend | 严重度: medium | 状态: resolved
> 发现日期: 2026-05-17 | 关联 Commit: `fix(frontend-resume-workshop): close create flow L2 gaps (BUG-0074)`
> 关联日志: [2026-05-17](../work-journal/2026-05-17.md)

## 1 现象

- `plan-code-review frontend-resume-workshop/002-create-flow-and-onboarding --fix` 反查 Phase 4 / Phase 6 时发现，`ResumeParseFlow` 的失败态 "retry" 按钮只把状态重置为 `polling`，没有重新触发 `getResume` 轮询。
- P0.081/P0.082/P0.083 场景 wrapper 的 `verify.sh` 只检查了宽泛 passed summary 和文件名，没有显式拒绝 no-test / failed summary。
- checklist 6.11 / 6.12 记录的 raw grep gate 声称 0 命中，但测试文件自身包含 retired module 和 prototype path 字面量，导致真实 raw grep 非零。
- `frontend/tests/pixel-parity/resume-workshop-create.spec.ts` 头部声明覆盖 Upload / Paste / Guided / Parse / Preview 5 个屏幕，但实际只覆盖了前三个 input screen。

## 2 诊断过程

### 假设 1：retry 失败是组件未接线

- 验证方法：读取 `ParsingStage.tsx`、`ResumeParseFlow.tsx` 和 `useResumeParsingPolling.ts` 的调用链，并新增 `retry() restarts polling after a failed terminal state` 回归测试。
- 结果：确认 `ResumeParseFlow` 已把按钮接到 `retry()`，但 hook 内 `retry()` 只重置 snapshot，未改变 `useEffect` 依赖，也没有安排新的 poll；新增测试 Red 阶段失败在 `polling`。

### 假设 2：BDD wrapper 能证明当前 runner 非 no-op

- 验证方法：直接读取 P0.081/P0.082/P0.083 的 `trigger.sh` / `verify.sh`。
- 结果：`trigger.sh` 使用 `set -euo pipefail`，不会吞 runner exit code；但 `verify.sh` 缺少 runner marker、非零 tests marker、no-test / failed summary 的负向检查。

### 假设 3：legacy negative grep gate 与 checklist 命令一致

- 验证方法：执行 `git grep -nE "welcome|mistake|growth|drill|followup|STAR|experiences|voice|OnboardingScreen|onboarding=true" -- frontend/src/app/screens/resume-workshop/create/` 和 prototype import grep。
- 结果：两条 raw grep 均命中 `CreateFlowLegacyNegative.test.ts` 自身，说明测试实现污染了 checklist 要求的 raw gate。

### 假设 4：pixel parity spec 覆盖了计划声明的 5 屏矩阵

- 验证方法：读取并运行 `frontend/tests/pixel-parity/resume-workshop-create.spec.ts`。
- 结果：首次 Playwright 只运行 6 tests，即 Upload / Paste / Guided × desktop/mobile；ParseFlow 和 PreviewConfirm 缺失。

## 3 根因分析

- `useResumeParsingPolling.retry()` 使用 ref 和 snapshot 重置表达 "重试"，但没有把 retry 作为可触发 effect 重启的 state，导致 failed terminal state 后没有新的轮询 session。
- L2 完成状态过早依赖 checklist / BDD 勾选与 runner summary，缺少对实际 wrapper 语义和 Playwright screen matrix 的反向审计。
- negative grep 测试用字面量实现待禁止词和 prototype path，导致测试本身与 raw grep gate 冲突。

涉及代码路径：

- `frontend/src/app/screens/resume-workshop/create/hooks/useResumeParsingPolling.ts`
- `frontend/src/app/screens/resume-workshop/create/hooks/useResumeParsingPolling.test.tsx`
- `frontend/src/app/screens/resume-workshop/create/CreateFlowLegacyNegative.test.ts`
- `frontend/tests/pixel-parity/resume-workshop-create.spec.ts`
- `test/scenarios/e2e/p0-081-resume-create-flow-upload-paste-guided-happy/scripts/verify.sh`
- `test/scenarios/e2e/p0-082-resume-create-flow-parsing-failure-and-retry/scripts/verify.sh`
- `test/scenarios/e2e/p0-083-resume-create-flow-preview-confirm-and-cta-handoff/scripts/verify.sh`

## 4 修复方案

- 为 `useResumeParsingPolling` 增加 `retryEpoch` state，`retry()` 递增该 state 以触发 effect 启动新的 polling session。
- 新增 hook 回归测试，覆盖 failed → retry → processing → ready，确认只重新轮询，不重新注册简历。
- 加强 P0.081/P0.082/P0.083 `verify.sh`：要求 Vitest `RUN` marker、非零 passed test files、非零 passed tests，并拒绝 no-test / failed summary。
- 将 legacy negative test 内的待禁止词和 prototype path 通过片段拼接生成，确保 raw grep 仍为 0。
- 扩展 `resume-workshop-create.spec.ts`，新增 ParseFlow 和 PreviewConfirm logical tests，desktop/mobile 共 10 项覆盖 5 屏矩阵。

放弃的方案：

- 不把 `retry()` 改成由 `ParsingStage` remount 组件实现，因为 retry 语义属于 hook 自身，放在 hook 内更容易被单测锁定。
- 不新增新的 workflow pattern；本问题已落在 `PATTERNS.md` 模式 4 的 false-green runner / pixel gate 检查范围内。

## 5 验证

- Green：`pnpm --filter @easyinterview/frontend exec vitest run --reporter=verbose src/app/screens/resume-workshop/create/hooks/useResumeParsingPolling.test.tsx`：5 passed。

  - P0.083 `setup.sh` + `trigger.sh` + `verify.sh` + `cleanup.sh`：5 files / 31 tests PASS。
- UI parity：`pnpm --filter @easyinterview/frontend test:pixel-parity tests/pixel-parity/resume-workshop-create.spec.ts`：10 passed。
- Static gates：`pnpm --filter @easyinterview/frontend typecheck`、`pnpm --filter @easyinterview/frontend build`、`make docs-check`、`git diff --check` PASS。
- Negative grep：retired module grep and prototype import grep under `frontend/src/app/screens/resume-workshop/create/` returned zero matches.

## 6 经验教训

- Completed checklist is not enough for L2 closure; scenario wrappers and Playwright specs must be read as first-class evidence.
- Negative grep tests must avoid storing the exact forbidden strings in the scanned tree unless the gate explicitly excludes test files.
- Existing Pattern 4 already describes this class: completed checklist plus under-asserted runner / pixel gate evidence can hide real behavior drift.
