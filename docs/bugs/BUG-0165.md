# BUG-0165: empty report retry focus encoded as null and blocked ready dashboards

> 模块: review | 严重度: high | 状态: resolved
> 发现日期: 2026-07-13 | 关联 Commit: `fix(report): ground report semantics and reliability (BUG-0164, BUG-0165, BUG-0166)`
> 关联日志: [2026-07-13](../work-journal/2026-07-13.md)

## 1 现象

- 真实 provider 已把中文 `needs_practice` 与英文 `well_prepared` 报告持久化为 `ready`，直接 API 读取也返回完整摘要、维度、证据与行动。
- 正式前端打开这两份报告时均 fail-close 到“报告内容未通过校验”，因此合法 ready 报告无法进入 dashboard，P0.099 六图验收被阻断。
- 受影响的是 `retryFocusDimensionCodes` 合法为空的报告，包括 `well_prepared`、`basically_ready` 和两个 generic retry exception；严重度为 `high`。

## 2 诊断过程

### 假设 1：模型报告内容仍违反 24/64 或完整语义合同

- 验证方法：只读比较 DB 与 API 的状态、preparedness、维度/证据/行动数量、行动长度与内部 anchor；运行后端完整 validator 与前端 focused contract tests。
- 结果：排除。报告在后端为 `ready`，中文行动为 32 code points、英文行动为 5 words，事实到判断再到建议的链路完整。

### 假设 2：前端因响应中的空集合形态而拒绝

- 验证方法：读取 API 的脱敏结构摘要并与 `frontend/src/app/screens/report/reportContract.ts` 的 exact ready contract 对照。
- 结果：确认。响应中的 `retryFocusDimensionCodes` 为 `null`，而 OpenAPI 与前端要求必填数组；同一报告在 DB 中为合法空数组。

### 假设 3：存储读层直接返回了错误的空值

- 验证方法：反查 store scanner 与 API mapper 的空集合初始化路径，并对 queued/generating/ready/failed 四种 projection 增加 non-nil regression。
- 结果：排除 store、确认 mapper。store 先初始化 `[]`，mapper 随后用 nil 目标 slice 复制，重新产生 `nil`。

## 3 根因分析

- `backend/internal/api/reports/mapper.go` 使用 `append([]string(nil), report.RetryFocusDimensionCodes...)` 复制 focus。
- 当源 slice 长度为零时，Go 保留 `nil`；JSON 编码因此输出 `null`，违反 required array wire contract。
- 既有 mapper 测试只覆盖非空 focus，并且 status projection 测试没有断言 empty collection 的 JSON/non-nil 语义，所以该跨层漂移没有被发现。

## 4 修复方案

- 将 mapper 的目标改为 non-nil empty slice，使空 focus 稳定编码为 `[]`，非空 focus 的复制语义不变。
- 在四种 report status 的 projection 测试中统一断言 `RetryFocusDimensionCodes != nil`，锁定 wire 语义。
- 未放宽前端 validator，也未把 `null` 加入 OpenAPI；空集合是当前 required contract，保持一个真理源更符合奥卡姆原则。

## 5 验证

- RED：新增断言在 queued/generating/ready/failed 四个子测试全部失败，错误为 empty focus 仍编码成 `null`。
- GREEN：`go test ./internal/api/reports -count=1` 通过；reports/store/review focused 三包通过；`git diff --check` 通过。
- backend 无数据清理重部署后，原两份 ready 报告均进入正式 dashboard；桌面和 390px viewport 的 `scrollWidth == clientWidth`。
- E2E.P0.099 run `e2e-p0-099-20260713T095144Z-12381` 的 exact 六图、DB/API content digest、内容审计、raw-debug redline、trigger 与 verify 全部 PASS。

## 6 经验教训

- required collection 的 empty 与 null 是跨语言 wire 合同的一部分；Go slice copy helper 必须显式保留 non-nil empty 语义。
- 结构测试不能只覆盖有元素的 happy path；每个 required array 至少应有 empty JSON projection regression。
- 本次不新增 PATTERNS 条目；该经验已由当前 mapper regression 和 owner checklist 固化。
