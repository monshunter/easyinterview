# BUG-0096: prompt examples rendered as minimal placeholder JSON

> 模块: eval | 严重度: medium | 状态: resolved
> 发现日期: 2026-05-24 | 关联 Commit: `fix(prompt-rubric): render complete prompt example outputs (BUG-0096)`
> 关联日志: [2026-05-24](../work-journal/2026-05-24.md)

## 1 现象

- 用户检查当前 prompt 后指出：每个 prompt 的 example 过于简陋，高效 example 应当是包含完整 JSON output 的描述和示例，而不是 OpenAPI schema。
- 当前 renderer 生成的是 schema-valid 但 required-only 的最小 JSON；常见值是 `"string"`、`1`、`0.5` 这类占位。
- 影响范围：13 个 chat feature_key × 2 language coordinates 的 output contract block，以及 `config/prompts/README.md`、`scripts/lint/prompt_lint.py` 和 seed migration 中的 prompt body/hash。

## 2 诊断过程

### 假设 1：现有 plan 已要求完整示例，只是个别 prompt 漏更新

- 验证方法：读取 `docs/spec/prompt-rubric-registry/plans/002-output-schema-contract/plan.md`、`checklist.md` 与 `config/prompts/README.md`。
- 结果：排除。文档明确写过 `minimal example JSON` / `minimal valid JSON`，说明缺口来自 gate 设计，而不是单个 prompt 漏改。

### 假设 2：renderer 只输出 required 字段

- 验证方法：读取 `scripts/lint/prompt_lint.py` 的 `example_for_schema`，并新增 focused regression `test_rendered_example_includes_optional_properties_and_representative_values`。
- 结果：确认。Red 阶段测试失败，`example_for_schema` 返回值只包含 `questionText` / `questionIntent`，遗漏 optional `focusDimension`、`expectedSignals`、`timeBudgetSeconds`。

### 假设 3：只改 prompt body 足够

- 验证方法：检查 YAML `template_hash`、`migrations/000002_seed_baseline_prompt_rubric_versions.up.sql` 和 `make lint-prompts` 的 seed hash gate。
- 结果：排除。prompt body 改动必须刷新 YAML hash；已有 seed rows 也需要同步 body/hash，否则 DB seed 会保留旧示例或 hash 漂移。

## 3 根因分析

- 根本原因：002 plan 初始把 example 定义为 minimal schema-valid JSON，renderer 因此只遍历 `required` 字段并使用通用占位值。
- 涉及路径：
  - `scripts/lint/prompt_lint.py`
  - `scripts/lint/prompt_lint_test.py`
  - `config/prompts/**/v0.1.0*.md`
  - `config/prompts/**/v0.1.0*.yaml`
  - `migrations/000002_seed_baseline_prompt_rubric_versions.up.sql`
- 为什么之前没有发现：`make lint-prompts` 只证明 example JSON schema-valid 与 contract block 可重渲染，没有断言 optional 字段完整性、业务形态值或 “not OpenAPI schema” 负向语义。

## 4 修复方案

- 将 renderer 改为生成 complete representative JSON output：按 schema 顺序包含 required + optional properties，并为常见业务字段生成代表性值。
- 在 output contract block 中增加明确要求：返回 complete JSON value，不是 JSON Schema 或 OpenAPI schema。
- 重渲染 26 个 prompt body，刷新 YAML `template_hash`，同步 seed migration 里已有 prompt rows 的 body/hash。
- 补充 `resume.parse` project/education optional schema properties，避免完整示例出现 `{}`。
- 原地修订 `prompt-rubric-registry` spec v2.6、`002-output-schema-contract` plan/checklist v1.3、context discovery 和 `config/prompts/README.md`。

## 6 经验教训

- Prompt example gate 不能只证明 JSON schema-valid；对于 LLM 输出契约，示例还必须足够完整、业务化，并明确区分 output value 与 schema 文档。
- schema-rendered prompt block 的测试矩阵应同时覆盖 required path、optional path、示例值质量和负向语义提示，避免 lint 把低信号示例固定成 “green”。
