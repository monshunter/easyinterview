# BUG-0073: resume tailor ready provenance was incomplete after DB roundtrip

> 模块: schema | 严重度: medium | 状态: resolved
> 发现日期: 2026-05-17 | 关联 Commit: `fix(backend-resume): persist tailor run provenance (BUG-0073)`
> 关联日志: `-`

## 1 现象

- `$plan-code-review backend-resume/002-versions-tailor-runs-and-save-v1 --fix` 发现 `getResumeTailorRun` ready 响应可能返回不完整 `GenerationProvenance`。
- `ResumeTailorRun.provenance` 在 OpenAPI 中引用 `GenerationProvenance`，其 required 字段包括 `promptVersion / rubricVersion / modelId / language / featureFlag / dataSourceVersion`。
- 现有 unit / fixture / fake-store gates 均能通过，但真实 DB roundtrip 后 `language / featureFlag / dataSourceVersion` 为空。

## 2 诊断过程

### 假设 1：OpenAPI 允许 ready tailor run 只返回部分 provenance

- 验证方法：读取 `openapi/openapi.yaml` 中 `GenerationProvenance` 和 `ResumeTailorRun` schema，以及 backend-resume plan / BDD 对 ready run 的要求。
- 结果：排除。`GenerationProvenance` 六个字段均 required，`ResumeTailorRun` 描述要求 `status == ready` 时携带 provenance。

### 假设 2：AI job 已经产生完整 provenance，但 store 没有完整持久化

- 验证方法：读取 `backend/internal/resume/jobs/tailor.go`、`backend/internal/resume/store/tailor_runs.go` 和 `backend/internal/resume/service.go`。
- 结果：确认。`tailorProvenance` 已产生完整字段，但 `resume_tailor_runs` 只写入 `prompt_version / rubric_version / model_id / provider`，`scanTailorRun` 后续也只恢复这些字段。

### 假设 3：现有测试可以覆盖该缺口

- 验证方法：跑 focused gates，并补充 `CompleteTailorRunSuccess -> GetTailorRun` integration RED 断言。
- 结果：排除。现有 handler fixture parity、service fake store、cmd/api drainer fake store 都通过；新增 DB roundtrip 断言稳定失败于空 `language / featureFlag / dataSourceVersion`。

## 3 根因分析

- `resume_tailor_runs` 表缺少 `language / feature_flag / data_source_version` typed columns，导致 ready run 只持久化了部分 provenance。
- `CompleteTailorRunSuccess` 和 `MarkTailorRunReady` 写库路径没有完整保存 `VersionProvenance`。
- `tailorRunRecordToAPI` 会把 store record 中的空字段原样映射到 `api.GenerationProvenance`，因此 fake store 测试无法暴露真实 DB read-after-write 漏洞。
- `resume_version_suggestions` 的 provenance 也只从 joined tailor run 覆盖了 prompt/rubric/model/provider，未从 run 侧覆盖 language/feature flag/data source。

## 4 修复方案

- 新增 migration `000008_resume_tailor_run_provenance`，为 `resume_tailor_runs` 添加 `language / feature_flag / data_source_version`，并提供非空默认值。
- 更新 `MarkTailorRunReady` / `CompleteTailorRunSuccess`，将完整 provenance 写入 run row。
- 更新 `scanTailorRun` / `tailorRunSelectSQL`，确保 `GetTailorRun` 从 DB 恢复完整 provenance。
- 更新 `loadVersionSuggestions`，让 suggestion provenance 从 joined tailor run 同步覆盖六个 required 字段。
- 加固 live store integration，覆盖 ready run DB roundtrip 和 suggestion decision response provenance。

## 6 经验教训

- AI provenance contract 不能只用 fake store 或 fixture byte parity 证明；只要 schema required 字段需要从 DB 返回，就必须有 write-after-read 的 store integration 断言。
- 若 API resource 的 ready response 与 async job completion 之间隔着持久化层，L2 review 应把 "job produced complete data" 和 "GET restored complete data" 分开验证。
